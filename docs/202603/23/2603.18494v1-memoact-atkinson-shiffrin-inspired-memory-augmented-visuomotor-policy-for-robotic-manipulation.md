## 摘要
记忆增强的机器人策略在处理依赖记忆的任务中至关重要。然而，现有方法通常依赖于简单的观测窗口扩展，难以同时实现精确的任务状态追踪和鲁棒的长时程记忆保留。为了克服这些挑战，受 Atkinson-Shiffrin 记忆模型的启发，我们提出了 MemoAct，这是一种基于分层记忆的策略，利用不同的记忆层级来解决特定的瓶颈。具体而言，无损短期记忆确保了精确的任务状态追踪，而压缩长期记忆则实现了鲁棒的长时程记忆保留。为了丰富评估维度，我们基于 RoboTwin 2.0 构建了 MemoryRTBench，专门用于评估策略在任务状态追踪和长时程记忆保留方面的能力。在模拟和真实场景中进行的大量实验表明，与现有的马尔可夫基准和历史感知策略相比，MemoAct 实现了更优越的性能。项目页面已发布。

## Abstract
Memory-augmented robotic policies are essential in handling memory-dependent tasks. However, existing approaches typically rely on simple observation window extensions, struggling to simultaneously achieve precise task state tracking and robust long-horizon retention. To overcome these challenges, inspired by the Atkinson-Shiffrin memory model, we propose MemoAct, a hierarchical memory-based policy that leverages distinct memory tiers to tackle specific bottlenecks. Specifically, lossless short-term memory ensures precise task state tracking, while compressed long-term memory enables robust long-horizon retention. To enrich the evaluation landscape, we construct MemoryRTBench based on RoboTwin 2.0, specifically tailored to assess policy capabilities in task state tracking and long-horizon retention. Extensive experiments across simulated and real-world scenarios demonstrate that MemoAct achieves superior performance compared to both existing Markovian baselines and history-aware policies. 在机器人操作中，许多任务具有**记忆依赖性**（如：多次敲击不同颜色的方块，中间步骤的视觉观测可能完全相同）。传统的马尔可夫（Markovian）策略仅依赖当前观测，容易陷入“感知混淆”；而现有的记忆增强方法存在两难困境：
*   **简单窗口法（FIFO）**：虽然保留了近期细节，但早期信息会被快速覆盖，缺乏长程记忆。
*   **有损压缩法**：虽然能延长记忆跨度，但会丢失关键的细粒度状态信息，导致任务步骤跳跃或出错。
**研究动机**：受人类认知科学中的 Atkinson-Shiffrin 记忆模型启发，探索如何同时实现**精确的任务状态追踪**（短期）和**稳健的长程信息保留**（长期）。

#### 2. 方法论：核心思想与关键技术
MemoAct 提出了一个分层记忆架构，包含三个核心模块：
*   **感官蒸馏模块 (Sensory Distillation)**：
    *   利用预训练的 DINOv2 提取图像特征，通过可学习的查询 Token 压缩冗余视觉信息。
    *   将视觉特征与机器人本体状态（Proprioception）融合，生成“感官记忆” Token。
*   **长短期记忆模块 (Long Short-Term Memory Module)**：
    *   **短期记忆库 (STMB)**：存储最近的无损 Token（容量 $T_s=6$），用于精确追踪当前任务进度。
    *   **长期记忆库 (LTMB)**：存储压缩后的历史信息（容量 $T_k=8$）。
    *   **记忆整合机制**：当 STMB 满时，最旧的 Token 通过因果 Transformer 编码器提取摘要并移入 LTMB；LTMB 满时，通过计算相邻 Token 相似度进行合并，以极低的存储代价保留极长的时间跨度。
*   **动作解码器 (Action Decoder)**：
    *   采用基于扩散模型（Diffusion Policy）的架构。
    *   通过门控网络（Gating Network）自适应融合当前感官记忆与检索到的历史记忆，引导动作序列生成。
*   **训练范式**：采用**流式训练（Streaming Training）**，按时间顺序加载样本以维持记忆库的连续性。
