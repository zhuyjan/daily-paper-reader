## 摘要
大语言模型（LLM）智能体在封闭世界具身环境中经常失败，因为动作必须满足严格的前置条件（如位置、物品清单和容器状态），且失败反馈非常稀疏。我们识别出两种结构耦合的失败模式：(P1) 无效动作生成和 (P2) 状态漂移，两者在退化循环中相互放大。我们提出了 RPMS，这是一种冲突管理架构，它通过结构化规则检索强制执行动作的可行性，通过轻量级信念状态管控记忆的适用性，并通过规则优先的仲裁机制解决两个来源之间的冲突。在 ALFWorld（134 个未见任务）上，RPMS 使用 Llama 3.1 8B 实现了 59.7% 的单次尝试成功率（比基线提升 23.9 个百分点），使用 Claude Sonnet 4.5 实现了 98.5%（提升 11.9 个百分点）；在 8B 模型的提升中，仅规则检索就贡献了 +14.9 个百分点（具有统计学显著性），使其成为主导因素。一个关键发现是，情境记忆是有条件有用的：在缺乏接地的情况下使用时，它会损害某些任务类型的性能，但一旦经过当前状态过滤并受到显式动作规则的约束，它就会产生稳定的正向收益。将 RPMS 适配到 ScienceWorld 并结合 GPT-4，在所有消融条件下均获得了一致的提升（平均得分 54.0，而 ReAct 基线为 44.9），提供了核心机制在结构迥异的环境中依然有效的迁移证据。

## Abstract
LLM agents often fail in closed-world embodied environments because actions must satisfy strict preconditions -- such as location, inventory, and container states -- and failure feedback is sparse. We identify two structurally coupled failure modes: (P1) invalid action generation and (P2) state drift, each amplifying the other in a degenerative cycle. We present RPMS, a conflict-managed architecture that enforces action feasibility via structured rule retrieval, gates memory applicability via a lightweight belief state, and resolves conflicts between the two sources via rules-first arbitration. On ALFWorld (134 unseen tasks), RPMS achieves 59.7% single-trial success with Llama 3.1 8B (+23.9 pp over baseline) and 98.5% with Claude Sonnet 4.5 (+11.9 pp); of the 8B gain, rule retrieval alone contributes +14.9 pp (statistically significant), making it the dominant factor. A key finding is that episodic memory is conditionally useful: it harms performance on some task types when used without grounding, but becomes a stable net positive once filtered by current state and constrained by explicit action rules. Adapting RPMS to ScienceWorld with GPT-4 yields consistent gains across all ablation conditions (avg. score 54.0 vs. 44.9 for the ReAct baseline), providing transfer evidence that the core mechanisms hold across structurally distinct environments.

### 1. 核心问题与研究动机
*   **核心问题**：LLM 智能体在封闭世界具身环境（如 ALFWorld）中经常失败，主要源于两个耦合的失败模式：
    *   **(P1) 无效动作生成**：动作违反了环境隐含的严格前提条件（如：未到冰箱前就尝试打开冰箱），而环境反馈往往极其稀疏（仅提示“Nothing happens”）。
    *   **(P2) 状态漂移**：智能体对环境状态的认知（Belief）与真实状态脱节，导致错误累积。
*   **研究背景**：现有的方法（如 ReAct、Reflexion）通常只针对其中一个问题，且在面对“静默失败”时缺乏有效的约束机制。作者认为，知识增强（规则和记忆）只有在**可执行**且**状态一致**时才真正有效。

### 2. 方法论：RPMS 架构
RPMS 采用了一种“冲突管理”架构，核心思想是：**规则强制执行可行性，状态过滤确保记忆适用性，规则优先解决冲突。**

*   **信念状态跟踪 (Belief State Tracking)**：
    *   维护一个轻量级的任务相关状态 $b_t$，记录位置、手部状态（是否持物）、容器开关状态等。
    *   更新过程是确定性的（基于正则表达式解析），不依赖 LLM 调用，从而保证状态认知的准确性。
*   **分层规则手册 (Hierarchical Rule Manual)**：
    *   **通用规则 (Tier 1)**：环境无关的搜索策略和失败恢复逻辑。
    *   **领域规则 (Tier 2)**：针对特定任务类型（如“加热物品”）的标准操作流程。
    *   **环境规则 (Tier 3)**：具体的动作语义（前提条件与效果），纠正 LLM 的常识偏差（例如：在 ALFWorld 中，加热动作是原子的，不需要把物品放入微波炉内部）。
*   **状态一致性过滤 (State-Consistent Filtering, SCF)**：
    *   从情节记忆（Episodic Memory）中检索经验时，不仅匹配目标，还匹配“状态签名”（如手部占用情况）。如果过去成功的经验是在“手空”时发生的，而当前智能体“手中有物”，则该记忆会被过滤掉。
*   **规则优先仲裁 (Rules-First Arbitration)**：
    *   当检索到的记忆建议与显式规则冲突时，强制执行规则。通过硬过滤（删除违规动作）和软标注（向 LLM 发出警告）来解决冲突。

### 6. 主要结论与发现
*   **性能大幅提升**：在 Llama 3.1 8B 上，RPMS 将成功率从 35.8% 提升至 59.7%（+23.9%）；在 Claude 4.5 上达到了 98.5%。
*   **规则是主导因素**：在 8B 模型的提升中，规则检索贡献了 14.9%，是性能增长的核心。
*   **记忆的“条件有用性”**：这是一个重要发现——**未经地（Grounding）的记忆可能有害**。在某些任务中，单纯增加记忆反而降低了成功率，只有在规则约束和状态过滤下，记忆才能转化为稳定的正向收益。
*   **跨环境泛化**：在 ScienceWorld 上，RPMS 同样取得了一致的提升，证明了其分层规则和过滤机制的通用性。

### 7. 优点与亮点
*   **即插即用**：无需训练或微调，通过纯 Prompting 方式显著增强了小参数模型（8B）的能力。
*   **结构化分析**：深入分析了具身规划中 P1 和 P2 的恶性循环，并提出了针对性的架构解决方案。
*   **鲁棒性**：通过状态过滤和仲裁机制，解决了 RAG（检索增强生成）中常见的“检索到不相关或冲突信息”的问题。

### 8. 不足与局限
*   **规则依赖**：Tier 3 环境规则仍需人工编写。虽然 Tier 1 和 Tier 2 具有通用性，但迁移到全新环境时仍有一定的人工成本。
*   **记忆冷启动**：情节记忆依赖于先前的交互经验，无法在完全没有任何先验轨迹的情况下生成指导。
*   **状态跟踪简化**：目前的信念状态跟踪主要基于手部占用等关键变量，对于更复杂、细粒度的环境状态（如化学反应的中间产物）可能需要更强大的解析器。
