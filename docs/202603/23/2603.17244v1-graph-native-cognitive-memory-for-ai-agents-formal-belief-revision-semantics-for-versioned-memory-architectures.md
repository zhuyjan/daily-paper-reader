## 摘要
尽管先前的系统中存在 AI 智能体记忆的单个组件，但其架构综合与形式化基础仍有待深入探索。我们提出了 Kumiho，这是一种基于形式化信念修正语义的图原生认知记忆架构。认知记忆所需的结构原语——不可变修订版本、可变标签指针、类型化依赖边、基于 URI 的寻址——与管理智能体产出的可版本化资产所需的结构原语完全一致，从而实现了一个服务于双重目的的统一图原生架构。核心形式化贡献在于建立了 AGM 信念修正框架与属性图记忆系统操作语义之间的对应关系，证明了其满足基本 AGM 公理（K*2--K*6）以及 Hansson 的信念基公理（相关性、核心保留性）。该架构实现了一种双存储模型（Redis 工作记忆，Neo4j 长期图谱），并结合了混合全文检索与向量检索。在 LoCoMo（词元级 F1）基准测试中，Kumiho 实现了 0.565 的总 F1 分数（n=1,986），其中包括 97.5% 的对抗性拒绝准确率。在测试隐式约束召回的二级认知记忆基准 LoCoMo-Plus 上，Kumiho 达到了 93.3% 的评判准确率（n=401）；基准作者的独立复现结果在 80% 中段，仍显著优于所有已发表的基准模型（表现最好的是 Gemini 2.5 Pro，为 45.7%）。三项架构创新推动了这一结果：前瞻性索引（在写入时对 LLM 生成的未来场景影响进行索引）、事件提取（在摘要中保留结构化的因果事件）以及客户端 LLM 重排序。该架构是模型解耦的：将回答模型从 GPT-4o-mini（约 88%）切换到 GPT-4o（93.3%）可在不改变流水线的情况下提高端到端准确率，401 条条目的总评估成本约为 14 美元。

## Abstract
While individual components for AI agent memory exist in prior systems, their architectural synthesis and formal grounding remain underexplored. We present Kumiho, a graph-native cognitive memory architecture grounded in formal belief revision semantics. The structural primitives required for cognitive memory -- immutable revisions, mutable tag pointers, typed dependency edges, URI-based addressing -- are identical to those required for managing agent-produced work as versionable assets, enabling a unified graph-native architecture that serves both purposes. The central formal contribution is a correspondence between the AGM belief revision framework and the operational semantics of a property graph memory system, proving satisfaction of the basic AGM postulates (K*2--K*6) and Hansson's belief base postulates (Relevance, Core-Retainment). The architecture implements a dual-store model (Redis working memory, Neo4j long-term graph) with hybrid fulltext and vector retrieval. On LoCoMo (token-level F1), Kumiho achieves 0.565 overall F1 (n=1,986) including 97.5% adversarial refusal accuracy. On LoCoMo-Plus, a Level-2 cognitive memory benchmark testing implicit constraint recall, Kumiho achieves 93.3% judge accuracy (n=401); independent reproduction by the benchmark authors yielded results in the mid-80% range, still substantially outperforming all published baselines (best: Gemini 2.5 Pro, 45.7%). Three architectural innovations drive the results: prospective indexing (LLM-generated future-scenario implications indexed at write time), event extraction (structured causal events preserved in summaries), and client-side LLM reranking. The architecture is model-decoupled: switching the answer model from GPT-4o-mini (~88%) to GPT-4o (93.3%) improves end-to-end accuracy without pipeline changes, at a total evaluation cost of ~$14 for 401 entries.

随着 AI 智能体从简单的聊天机器人转向能够执行多步任务、产生数字资产（代码、设计、文档）的自主工作者，现有的记忆机制暴露了两个核心瓶颈：

*   **资产管理脱节**：智能体产生的中间产物缺乏版本控制和依赖追踪，导致多智能体协作时难以定位正确的输入版本。
*   **研究动机**：作者发现认知记忆所需的结构原语（不可变版本、类型化边、URI 寻址）与资产管理系统高度一致。因此，本论文旨在构建一个统一的图原生架构，既能作为智能体的“大脑”存储记忆，又能作为“基础设施”管理其工作产出，并为其提供严谨的形式化逻辑支撑。

### 2. 方法论：核心思想与关键技术
Kumiho 架构的核心思想是将**属性图（Property Graph）**的操作语义与 **AGM 信念修正理论**相结合。

*   **双存储模型**：
    *   **工作记忆**：基于 Redis，提供极低延迟（<5ms）的会话缓冲区。
    *   **长期记忆**：基于 Neo4j，存储不可变的修订版本（Revisions）和可变的标签指针（Tags）。
*   **形式化语义**：
    *   建立了图操作与 AGM 公设（K\*2–K\*6）的对应关系。通过 `Supersedes` 边连接新旧信念，利用标签重定向实现信念更新，确保逻辑一致性。
    *   引入了基于 URI 的全局寻址方案（`kref://`），使每个记忆单元都可被精确引用和回溯。
*   **三项关键创新技术**：
    1.  **前瞻性索引 (Prospective Indexing)**：在写入记忆时，利用 LLM 生成该记忆在未来可能被用到的 3-5 个假设场景并进行索引，解决了查询与存储内容之间的语义鸿沟。
    2.  **事件提取 (Event Extraction)**：从对话中提取结构化的“事件-后果”链条，防止叙述性压缩导致因果细节丢失。
    3.  **客户端 LLM 重排序 (Client-side Reranking)**：由调用智能体自身的 LLM 根据结构化元数据选择最相关的修订版本，实现零额外推理成本的精度提升。
*   **Dream State（梦境状态）**：一个异步整合流水线，负责在后台对记忆进行去重、丰富和清理。引入了“断路器”（防止过度删除）和“发布保护”等分布式系统安全机制。

### 7. 优点
*   **理论与工程结合**：首次将经典的 AGM 信念修正理论应用于现代 AI 智能体记忆架构，提供了坚实的形式化基础。
*   **隐私与效率平衡**：采用 **BYO-storage（自带存储）** 模式，云端仅存元数据和索引，原始数据保留在本地，既保护隐私又降低了存储成本。
*   **结构化推理**：通过类型化边（Depends_On 等）实现了显式的因果和依赖追踪，这是扁平向量数据库无法做到的。

### 8. 不足与局限
*   **逻辑表达力受限**：为了保证计算可行性，目前仅支持简单的命题逻辑，无法处理复杂的描述逻辑（如类继承、角色组合）。
*   **Dream State 依赖性**：异步整合的效果高度依赖于 LLM 的评估质量，虽然有安全闸，但仍存在长期运行下记忆质量逐渐退化的风险。
*   **超大规模验证缺失**：目前的实验集中在数十万节点的规模，在千万级甚至亿级节点下的检索精度和性能表现尚待验证。
*   **评估偏差风险**：LoCoMo-Plus 基准本身由 LLM 辅助生成，可能存在某种程度的“模型家族偏好”。
