## 摘要
我们提出了 MemX，这是一种面向 AI 助手的本地优先长期记忆系统，采用了面向稳定性的检索设计。MemX 基于 Rust 语言实现，构建于 libSQL 和兼容 OpenAI 的嵌入 API 之上，为对话智能体提供持久化、可搜索且可解释的记忆。其检索流水线应用了向量召回、关键词召回、倒数排名融合（RRF）、四因子重排序，以及一种低置信度拒绝规则，用于在存储中不存在答案时抑制伪召回。我们从两个维度对 MemX 进行了评估。首先，通过两个自定义中文基准测试集（43 个查询，不超过 1,014 条记录）验证了流水线设计：在默认场景下 Hit@1 达到 91.3%，在高混淆场景下达到 100%，并具有保守的未命中查询抑制能力。其次，利用 LongMemEval 基准测试（500 个查询，最高 220,349 条记录）量化了系统在四种能力类型和三种存储粒度下的边界。在事实级粒度下，系统达到 Hit@5=51.6% 和 MRR=0.380，性能是会话级粒度的两倍，而时间推理和多会话推理仍具挑战性（Hit@5 <= 43.6%）。FTS5 全文索引在 10 万条记录规模下将关键词搜索延迟降低了 1,100 倍，使端到端搜索保持在 90 毫秒以内。与针对端到端智能体基准测试的 Mem0 及相关工作不同，MemX 专注于一个更窄、可复现的基准：本地优先部署、结构简单性、可解释检索以及面向稳定性的设计。

## Abstract
We present MemX, a local-first long-term memory system for AI assistants with stability-oriented retrieval design. MemX is implemented in Rust on top of libSQL and an OpenAI-compatible embedding API, providing persistent, searchable, and explainable memory for conversational agents. Its retrieval pipeline applies vector recall, keyword recall, Reciprocal Rank Fusion (RRF), four-factor re-ranking, and a low-confidence rejection rule that suppresses spurious recalls when no answer exists in the memory store.   We evaluate MemX on two axes. First, two custom Chinese-language benchmark suites (43 queries, <=1,014 records) validate pipeline design: Hit@1=91.3% on a default scenario and 100% under high confusion, with conservative miss-query suppression. Second, the LongMemEval benchmark (500 queries, up to 220,349 records) quantifies system boundaries across four ability types and three storage granularities. At fact-level granularity the system reaches Hit@5=51.6% and MRR=0.380, doubling session-level performance, while temporal and multi-session reasoning remain challenging (<=43.6% Hit@5). FTS5 full-text indexing reduces keyword search latency by 1,100x at 100k-record scale, keeping end-to-end search under 90 ms.   Unlike Mem0 and related work that targets end-to-end agent benchmarks, MemX focuses on a narrower, reproducible baseline: local-first deployment, structural simplicity, explainable retrieval, and stability-oriented design.

### 2. 方法论：核心思想与技术细节
MemX 采用 Rust 语言开发，基于 libSQL（SQLite 的分支）构建，其核心检索流水线包含以下关键步骤：
*   **双路召回（Hybrid Recall）**：同时进行基于向量（Vector）的语义搜索和基于 FTS5 的关键词（Keyword）匹配。
*   **倒数排名融合（RRF）**：使用 RRF 算法将两路召回的结果进行初步融合。
*   **四因子重排序（Four-Factor Re-ranking）**：根据四个维度计算综合得分：
    1.  **语义相似度**（权重 0.45）。
    2.  **新鲜度（Recency）**：采用指数半衰期衰减函数，优先考虑最近检索过的记忆。
    3.  **频率（Frequency）**：对检索次数进行对数归一化处理。
    4.  **重要性（Importance）**：用户或系统预设的显式权重。
*   **低置信度拒绝规则（Rejection Rule）**：如果关键词召回为空且最大向量相似度低于阈值 $\tau$（默认 0.5），系统将返回空结果，以抑制伪召回。
*   **去重机制**：通过内容去重和“标签签名”去重，防止相似主题的记忆占据所有检索位。
*   **访问与检索分离**：显式区分“查看记忆”和“检索成功”的计数，避免管理性操作污染排序信号。

### 6. 主要结论与发现
*   **存储粒度至关重要**：在 LongMemEval 中，**事实级（Fact-level）**存储的检索质量是会话级的两倍（Hit@5 达到 51.6%）。

*   **能力边界**：系统在处理**时间推理**和**多会话推理**时表现较弱（Hit@5 < 44%），说明仅靠单次向量检索难以解决复杂逻辑问题。

### 7. 优点
*   **本地优先与隐私**：完全基于本地文件系统，不依赖云端服务。
*   **稳定性导向**：通过拒绝规则和四因子排序，极大地降低了 AI 助手产生幻觉的风险。
*   **可解释性**：排序逻辑透明，且通过标签和类型系统使记忆结构化。
*   **高性能**：在 Rust 和 libSQL 的加持下，即使在 20 万级数据量下也能实现亚秒级响应。

### 8. 不足与局限
*   **多主题覆盖不足**：对于需要同时检索多个不同主题的复杂查询，系统往往只能找回最显著的一个。
*   **推理能力限制**：缺乏处理时间线排序和跨会话逻辑合成的机制。
*   **去重策略的副作用**：在纯事实级数据（无标签）中，现有的标签去重机制可能会误删相关结果。
*   **缺乏闭环反馈**：目前无法根据 AI 助手是否真正采用了检索到的记忆来自动优化排序权重。
