# Abstract 

Single-object tracking (SOT) on edge devices is a criti-cal computer vision task, requiring accurate and contin-uous target localization across video frames under oc-clusion, distractor interference, and fast motion. How-ever, recent state-of-the-art distractor-aware memory mech-anisms are largely built on segmentation-based trackers and rely on mask prediction and attention-driven memory up-dates, which introduce substantial computational overhead and limit real-time deployment on resource-constrained hardware; meanwhile, lightweight trackers sustain high throughput but are prone to drift when visually simi-lar distractors appear. To address these challenges, we propose EdgeDAM, a lightweight detection-guided track-ing framework that reformulates distractor-aware mem-ory for bounding-box tracking under strict edge con-straints. EdgeDAM introduces two key strategies: (1) Dual-Buffer Distractor-Aware Memory (DAM), which integrates a Recent-Aware Memory to preserve temporally consistent target hypotheses and a Distractor-Resolving Memory to explicitly store hard negative candidates and penalize their re-selection during recovery; and (2) Confidence-Driven Switching with Held-Box Stabilization, where tracker reli-ability and temporal consistency criteria adaptively acti-vate detection and memory-guided re-identification during occlusion, while a held-box mechanism temporarily freezes and expands the estimate to suppress distractor contamina-tion. Extensive experiments on five benchmarks, including the distractor-focused DiDi dataset, demonstrate improved robustness under occlusion and fast motion while main-taining real-time performance on mobile devices, achiev-ing 88.2% accuracy on DiDi and 25 FPS on an iPhone 15. Code will be released.

# 1. Introduction 

Visual object tracking (VOT) is a fundamental video un-derstanding task that requires continuously localizing a tar-get across frames given only its initial bounding box. De-spite recent progress with transformer-based and memory-augmented trackers [26, 28, 32], real-world deployment still suffers from two persistent failure modes: occlusion-induced drift and distractor confusion . In cluttered scenes, partial or full occlusions weaken the target evidence, and vi-sually similar objects become competing hypotheses, lead-ing to identity switches that subsequently propagate and compound over time. These challenges are further amplified on mobile and edge platforms, where strict latency and energy bud-1

gets limit the use of dense attention, high-resolution fea-ture fusion, and mask-propagation pipelines commonly adopted by top-performing trackers. This creates a clear robustness–efficiency gap: state-of-the-art methods such as SAM2.1++ [26] and SAMURAI [28] improve distrac-tor resilience by maintaining segmentation-based memory banks with dense per-frame representations and retriev-ing them via cross-frame attention, but the resulting com-pute and memory overhead often prevents real-time on-device tracking. However, as shown in Fig. 1, these meth-ods operate at 2–8 FPS even on high-end GPUs far be-low the real-time threshold for mobile applications. At the other extreme, efficient trackers such as EdgeTAM [32] and OSTrack [29] sacrifice robustness for speed: both report approximately 0.78 IoU on occlusion-heavy benchmarks while running at 10–15 FPS. This exposes a fundamental gap in existing VOT research: heavy memory-based designs 

improve robustness under occlusion and distractors but are often too slow for real-time edge deployment, whereas ef-ficient box-based designs achieve high throughput yet re-main fragile in the same challenging conditions. To date, no method achieves both robustness and efficiency simul-taneously without relying on segmentation supervision or large pretrained encoders. In practice, the computational bottleneck of memory-based trackers stems mainly from two factors: (i) dense mask propagation across frames, and (ii) global cross-attention over high-dimensional memory tokens. We argue that for bounding-box tracking, neither mech-anism is necessary. Distractor-aware reasoning can be per-formed directly on axis-aligned boxes using lightweight ge-ometric and appearance cues, eliminating the need for seg-mentation supervision. This motivates a different design paradigm: a detection-driven pipeline in which a compact 

Distractor-Aware Memory (DAM) operates at the bounding-box level, explicitly separating temporally stable target hy-potheses from distractor candidates through geometric gat-ing rather than attention. Based on this insight, we introduce EdgeDAM, alightweight single-object tracking framework tailored for real-time deployment on mobile and edge devices. EdgeDAM combines a flexible YOLO-based detector with the classical CSRT tracker [21], and introduces a bounding-box DAM module that is inspired by, yet substantially re-designed from, the mask-based DAM in SAM2.1++ [26]. A key distinction is that our DAM replaces dense feature memory with a dual-buffer structure: a Recent-Aware Mem-ory (RAM) that stores geometrically verified target states using IoU and area-consistency gating, and a Distractor-Resolving Memory (DRM) that maintains stable appear-ance anchors to support recovery after occlusion. Un-like SAM2.1++, which updates memory through attention over segmentation features, EdgeDAM updates both buffers using lightweight geometric gating criteria, requiring no mask supervision, no transformer inference, and no GPU-intensive cross-frame attention. To enable category-agnostic tracking, the detector is trained as a single-class object detector by unifying all cate-gory labels into a single target index, making the framework compatible with standard tracking benchmarks regard-less of object category. Moreover, EdgeDAM is detector-agnostic by design: any Ultralytics YOLO variant from YOLOv8 onward can be substituted without architectural changes. Our main contributions are: • We propose a bounding-box Distractor-Aware Memory (DAM) with dual RAM/DRM buffers that separates veri-fied target states from distractor hypotheses via geometric gating, requiring no segmentation masks or attention lay-ers. • We introduce a confidence-driven switching criterion and a held-box recovery mechanism that suppresses distractor contamination during occlusion and enables reliable re-identification via DRM-based scoring. • We present a detector-agnostic, edge-deployable tracking framework using a single-class YOLO backbone, achiev-ing real-time performance at 25 FPS on an iPhone 15 Pro Max with strong accuracy across five benchmarks. 

# 2. Related Work 

Tracking arbitrary objects in unconstrained video remains a longstanding challenge, particularly under long-term occlu-sion, distractor interference, and appearance changes. Clas-sical approaches such as Siamese networks [2, 13, 14, 25, 30] and regression-based trackers [6] match features be-tween a template and a search region but are prone to model drift during occlusion. Transformer-based trackers [5, 19] improve robustness via cross-frame attention but rely on high-dimensional memory tokens and deep encoders, which limit their applicability to resource-constrained platforms. Memory-based tracking has emerged as a dominant paradigm for robust tracking under occlusions. Such meth-ods maintain temporal buffers of features or masks and re-trieve contextual cues for re-identification [24, 26, 32]. For example, SAM2 [24] extends the Segment Anything Model with memory attention to propagate masks across frames. While effective for lengthy video segmentation [7, 11, 17, 23, 27], it suffers from high inference latency due to dense cross-attention over memory banks, making it unsuitable for real-time applications. Edge focused variants such as EdgeTAM [32] and SAM2.1++ [26] reduce latency through architectural op-timizations. EdgeTAM compresses spatial memory us-ing Perceiver modules, while SAM2.1++ introduces adistractor-aware memory structure (DAM). Both methods still rely on segmentation masks and global memory fusion, which impose non-negligible computational overhead and 2Figure 2. EdgeDAM framework overview. reduce flexibility. Recent work on efficient, detection-driven trackers such as KeepTrack [22], TransT [3], and MixFormer [4] seeks to improve speed and generalization. Yet, many fail to ex-plicitly handle occlusion or suffer from slow inference due to dense transformer blocks. SAMURAI [28], though pre-trained on large-scale video datasets, faces similar latency bottlenecks due to its hierarchical memory design. In con-trast, our proposed method integrates a YOLOv11s-based detection backbone with a lightweight DAM module. In-stead of relying on global memory fusion or segmenta-tion supervision, we introduce a post-processing strategy that applies spatial freezing and bounding box scaling to preserve tracking coherence under uncertainty, without in-creasing inference cost.

3.1. Distractor-Aware Memory for Bounding Boxes 

DAM4SAM [26] introduced a RAM/DRM decomposition for SAM2-based segmentation tracking, but is redesigned for bounding-box, where RAM stores dense per-frame mask-conditioned memory tokens and DRM retains hard-negative mask features retrieved via cross-frame attention. Our DAM adopts the same two-buffer philosophy but fundamentally redesigns both components for bounding-box operation, as summarized in Table 1. Concretely, we replace (i) dense mask features with compact geomet-ric descriptors, (ii) attention-based memory retrieval with IoU and area-consistency gating, and (iii) mask-conditioned promotion with a cosine-similarity stability test over ap-pearance descriptors. These changes reduce memory and compute requirements by orders of magnitude while pre-serving the core distractor-rejection capability of the dual-buffer design. # 4. Experiments 

4.1. Implementation Details 

4.1.1. Training Setup 

YOLOv11s is fine-tuned as a single-class detector by map-ping all object categories in each benchmark’s training split to a unified target label, making the backbone category-agnostic at test time. Benchmarks with mask or polygon annotations are normalized to axis-aligned bounding boxes prior to fine-tuning: tight enclosing rectangles for mask an-notations and vertex extrema for polygon annotations. Vali-dation and test splits are never used during training. All data augmentations are applied during fine-tuning only and dis-abled at evaluation. The framework is detector-agnostic and supports any YOLO variant (YOLOv8–YOLOv26) without architectural changes. 

4.1.2. Inference Configuration 

Detector inputs are resized to 640 ×640 with confidence threshold τs=0 .45 , NMS IoU threshold 0.50 , detection stride ∆=3 , and ROI crop scale κ=2 .0. Half-precision in-ference with model fusion is applied to reduce operator overhead. DAM uses |RAM |=10 , |DRM |=10 with FIFO eviction; each entry stores a 16 ×16 grayscale patch con-catenated with a 16 ×16 HSV histogram, ℓ2-normalised. Switching thresholds are τconf =0 .35 and τjump =0 .30 .DRM scoring weights are λiou =0 .4, λapp =0 .3, λmot =0 .2,

λtime =0 .1, with decay α=0 .05 and distractor penalty 

γ=0 .25 . RAM admission uses τin =0 .50 and area tolerance 

τa=0 .20 ; occlusion detection uses τocc =0 .40 ; DRM pro-motion requires cosine similarity τsim =0 .85 over a window of W =5 frames with mmin =3 .

4.1.3. Mobile Deployment 

Detector weights are exported to ONNX and converted to CoreML for execution on an iPhone 15 Pro Max. Throughput optimizations include fixed 640 ×640 inputs, half-precision inference, model fusion, adaptive detection stride scheduling, and ROI-crop inference during stable tracking, collectively achieving real-time performance at 

25 FPS on-device. 

4.2. Datasets 

We benchmark EdgeDAM against SOTA trackers using di-verse public datasets characterized by high occlusion rates, distractor interference, and tracking difficulty. These in-clude DiDi [26], VOT2020 [16], VOT2022 [15], LaSOT [10], LaSOText [9], and GOT-10k [12]. All datasets were pre-processed uniformly for compatibility with EdgeDAM. Quantitative results in the following subsections demon-strate EdgeDAM’s robustness and accuracy across varied occlusion scenarios. Qualitative results in Fig. 3 illustrate EdgeDAM’s performance in real-time tracking under occlu-sion on a standard SOTA benchmarks. We also conducted ablations to assess the impact of DAM, memory manage-ment, and post-processing on tracking performance.

4.4. Results on VOT Benchmarks 

The VOT challenges [15, 16] are standard benchmarks for evaluating single-object trackers under real-world condi-tions. We report EdgeDAM performance on VOT2020 and VOT2022 using three standard metrics: Expected Average Overlap (EAO), accuracy (mean overlap), and robustness (successfully tracked frames). We present the results on the VOT2020 benchmark in-cludes 60 challenging video sequences with frequent occlu-sion, abrupt motion, and distractor objects. As shown in Ta-ble 3, EdgeDAM achieves an EAO of 0.849, outperforming all compared methods, including the prior best SAM2.1++ [26] (EAO 0.729). It also improves accuracy from 0.799 to 0.849 and robustness from 0.961 to 0.992, reflecting relative gains of +12.0% in EAO and +5.0% in accuracy. Also, we extend our experiments on VOT2022 bench-mark extends difficulty by introducing 62 sequences with motion blur, deformable targets, and long-term occlusions. As reported in Table 4, EdgeDAM achieves an EAO of 0.790, surpassing the challenge winner MSAOT [8] (0.673) and SAM2.1++ (0.753). While accuracy matches SAM2.1++ (0.790), EdgeDAM exhibits slightly lower ro-bustness (0.941 vs. 0.969). Nonetheless, its higher EAO highlights improved temporal consistency and fewer track-ing interruptions.

4.6. Ablation Study 

We ablate four aspects of EdgeDAM: (1) the contribution of each system component, (2) DAM buffer capacity and its ef-fect on recovery accuracy and throughput, (3) bounding-box optimization under multi-distractor occlusion, and (4) sen-sitivity to key hyperparameters. All ablations are conducted on the custom domain-specific dataset of 35,000 frames fea-turing horse-rider sequences with frequent partial and full occlusions, unless stated otherwise. The framework is ro-bust to moderate hyperparameter variations: perturbing all default thresholds and weights by ±20% produces an IoU fluctuation of less than 0.015 on the DiDi benchmark, con-firming that EdgeDAM does not require fine-grained per-dataset tuning to maintain competitive performance.

4.8. DAM Buffer Capacity 

Table 8 reports the effect of varying RAM and DRM buffer sizes on IoU, recovery latency (RecF), and FPS. Smaller buffers (5–5) achieve high IoU (0.981) with low latency and real-time throughput (24.89 FPS). The 10–10 configuration yields the best IoU (0.991) and highest FPS (25.81), repre-senting the optimal trade-off for edge deployment. Larger buffers (15–15 and 20–20) cause a notable IoU drop to 0.870 and 0.880, respectively, despite wider memory capac-ity. This counter-intuitive degradation arises because FIFO eviction retains stale or partially corrupted entries as the buffer grows; under FIFO, older hypotheses that no longer reflect the current target appearance accumulate in DRM and compete with valid anchors during recovery scoring, increasing the probability of selecting an incorrect candi-date. Larger buffers also increase per-frame matching cost, reducing FPS below the real-time threshold. The 10–10 set-ting balances anchor diversity against staleness and is used as the default throughout.

Recovery degrades in multi-distractor scenes due to in-creased candidate ambiguity: with three horses and a 15–15 buffer, only 2 of 3 occlusions are recovered, as stale DRM entries compete with the correct anchor. # 5. Conclusion 

We presented EdgeDAM, a lightweight and efficient vi-sual object tracker designed for robust occlusion handling in resource-constrained environments. Our model integrates a DAM module within a YOLOv11s-based detection and CSRT tracking pipeline, enabling reliable recovery from long-term occlusions, abrupt appearance changes, and dis-tractor interference. To further enhance temporal stabil-ity, we introduced a post-processing strategy that adap-tively freezes and enlarges the bounding box during un-certain states, maintaining localization continuity until re-identification. Our results validate the effectiveness of struc-tured memory and hybrid detection-tracking integration for real-time, occlusion-resilient performance. In future work, we plan to extend EdgeDAM to multi-object and 3D occlusion-aware tracking, and explore its integration with vision-language models for more context-aware tracking.
