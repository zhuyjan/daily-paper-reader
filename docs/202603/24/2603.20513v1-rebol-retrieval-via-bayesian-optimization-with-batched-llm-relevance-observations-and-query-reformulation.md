## 摘要
LLM 重排序受限于通过向量相似度检索到的前 k 个文档，这既无法实现查询与文档之间的上下文 Token 交互，也无法捕捉多模态相关性分布。虽然 LLM 查询重构尝试通过生成改进的或额外的查询来提高召回率，但其后续仍采用向量相似度检索。因此，我们提出了 ReBOL 来解决这些前 k 阶段检索失败的问题。ReBOL 1) 利用 LLM 查询重构来初始化关于文档相关性的多模态贝叶斯优化（BO）后验分布，2) 迭代地获取文档批次进行 LLM 查询-文档相关性评分，随后更新后验分布以优化相关性。在探索了查询重构和文档批次多样化技术后，我们在五个 BEIR 数据集上，使用两种 LLM（Gemini-2.5-Flash-Lite, GPT-5.2）将 ReBOL 与 LLM 重排序基准进行了评估。ReBOL 一贯地实现了更高的召回率和具有竞争力的排名，例如在 Robust04 数据集上，与最佳 LLM 重排序器相比，其 recall@100 为 46.5% 对 35.0%，NDCG@10 为 63.6% 对 61.2%。我们还表明，ReBOL 可以达到与 LLM 重排序器相当的延迟。

## Abstract
LLM-reranking is limited by the top-k documents retrieved by vector similarity, which neither enables contextual query-document token interactions nor captures multimodal relevance distributions. While LLM query reformulation attempts to improve recall by generating improved or additional queries, it is still followed by vector similarity retrieval. We thus propose to address these top-k retrieval stage failures by introducing ReBOL, which 1) uses LLM query reformulations to initialize a multimodal Bayesian Optimization (BO) posterior over document relevance, and 2) iteratively acquires document batches for LLM query-document relevance scoring followed by posterior updates to optimize relevance. After exploring query reformulation and document batch diversification techniques, we evaluate ReBOL against LLM reranker baselines on five BEIR datasets and using two LLMs (Gemini-2.5-Flash-Lite, GPT-5.2). ReBOL consistently achieves higher recall and competitive rankings, for example compared to the best LLM reranker on the Robust04 dataset with 46.5% vs. 35.0% recall@100 and 63.6% vs. 61.2% NDCG@10. We also show that ReBOL can achieve comparable latency to LLM rerankers.

### ReBOL: 基于贝叶斯优化与 LLM 反馈的迭代式检索框架总结

#### 1. 核心问题与整体含义（研究动机和背景）
传统的“检索-重排序”（Retrieve-and-Rank）流水线存在严重的**召回率瓶颈**。LLM 重排序器虽然强大，但只能处理向量相似度检索返回的前 $k$ 个文档。向量检索存在两个主要缺陷：
*   **交互受限**：仅依赖查询和文档的向量点积，缺乏 Token 级别的上下文交互。
*   **分布单一**：通常诱导单模态的相关性分布，而实际相关文档往往分布在多个不同的语义簇中（多模态）。
即使引入 LLM 查询重构（Query Reformulation），最终仍需回归向量检索。本文提出 **ReBOL**，旨在将检索过程建模为一个**贝叶斯优化（BO）问题**，通过 LLM 的实时反馈动态引导搜索，从而突破固定 top-$k$ 列表的限制。

#### 2. 方法论：核心思想与技术细节
ReBOL 的核心是将文档检索视为对未知相关性函数 $f$ 的优化过程：
*   **多模态后验初始化**：利用原始查询 $q_0$ 和 LLM 生成的多个重构查询 $\{q_1, \dots, q_Q\}$ 在嵌入空间中初始化高斯过程（GP）先验。这使得模型在搜索初期就能捕捉到多个潜在的相关性峰值。
*   **迭代式批处理观测**：
    1.  **采集函数（Acquisition Function）**：使用 Greedy（后验均值）、UCB（置信上限）或 Random 策略选择下一批文档。
    2.  **批次多样化**：引入了一种受 **MMR（最大边界相关性）** 启发的创新方法，在选择文档批次时平衡“采集函数值”与“文档间的语义多样性”，防止采样陷入局部最优。
    3.  **LLM 评分**：调用 LLM 对选定的文档批次进行绝对相关性评分（采用 0-3 分制，如 UMBRELA 准则）。
    4.  **后验更新**：将 LLM 的评分作为观测值更新 GP 后验分布，从而调整对整个文档库相关性的估计。
*   **随时检索（Anytime Retrieval）**：在任何迭代阶段，ReBOL 都可以根据当前的 GP 后验均值对所有文档进行排名并返回结果。

#### 7. 优点（亮点）
*   **范式创新**：打破了“先检索后排序”的静态框架，将检索转变为一个动态的、由 LLM 反馈驱动的优化过程。
*   **多模态建模**：通过非线性核函数（RBF）有效解决了向量检索难以处理多语义簇的问题。
*   **效率平衡**：通过批处理观测和 MMR 采样，在保证检索质量的同时，将延迟控制在与现有重排序技术相当的水平。

#### 8. 不足与局限
*   **数据集覆盖面**：虽然涵盖了 5 个数据集，但对于更大规模（如百万级以上）或更极端的长尾分布数据集的验证仍有限。
*   **LLM 依赖性**：系统性能高度依赖于 LLM 评分的准确性，且 LLM 调用成本依然是实际部署中的主要障碍。

*   **偏差风险**：LLM 评分可能存在社会偏见或对特定文档格式的偏好，这些偏好会被 BO 框架放大。
