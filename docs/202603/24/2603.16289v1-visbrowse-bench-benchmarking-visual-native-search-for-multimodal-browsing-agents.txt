## Abstract 

The rapid advancement of Multimodal Large Language Models (MLLMs) has enabled browsing agents to acquire and reason over multimodal information in the real world. But existing benchmarks suffer from two limitations: insufficient evaluation of vi-sual reasoning ability and the neglect of native visual information of web pages in the reasoning chains. To address these challenges, we introduce a new benchmark for visual-native search, VisBrowse-Bench. It contains 169 VQA instances covering multiple domains and evaluates the models’ visual reasoning capabilities during the search process through multimodal evidence cross-validation via text-image re-trieval and joint reasoning. These data were constructed by human experts using a multi-stage pipeline and underwent rigorous manual verification. We additionally propose an agent workflow that can effectively drive the browsing agent to actively collect and reason over visual information during the search process. We compre-hensively evaluated both open-source and closed-source models in this workflow. Experimental results show that even the best-performing model, Claude-4.6-Opus only achieves an accuracy of 47.6%, while the proprietary Deep Research model, o3-deep-research only achieves an accuracy of 41.1%. The code and data can be accessed at: https://github.com/ZhengboZhang/VisBrowse-Bench

VisBrowse-Bench relies on two core design principles: the integration of multimodal information within the reasoning chain, and an inherent dependence on visual capabilities. We constructed our benchmark through a multi-stage, expert-guided pipeline, resulting in 169 VQA instances that cover seven distinct domains. Based on a seed entity with inherent visual ambiguity, domain experts recursively construct multi-hop reasoning chains, which require locating and reasoning novel visual evidence that cannot be paraphrased textually, alongside textual sources that provide complementary but insufficient information. Furthermore, we propose an efficient agentic workflow for visual reasoning and visual information retrieval during the search process. In this workflow, the agent is driven to actively perform visual reasoning and visual information retrieval using a rich set of tools. The preview performance of the MLLMs using our workflow on VisBrowse-Bench is shown in Figure 2. The best performance in VisBrowse-Bench is achieved by Claude-4.6-Opus, with an accuracy of 47.6%, while most models achieve an accuracy of around 30%. Our key contributions can be summarized as follows: • We formalize the task of multimodal browsing, identifying two critical challenges in existing benchmarks: insufficient evaluation of visual reasoning ability and the neglect of visual-native information in the reasoning chains. • We propose VisBrowse-Bench, a new benchmark comprising 169 rigorously validated instances constructed by human experts. This benchmark aims to jointly evaluate the search and visual reasoning capabilities of multimodal browsing agent systems. • We introduce a multi-turn visual information retrieval-reasoning agentic workflow to solve multimodal browsing problems in the real world. Compared to direct answer, the models’ performance are significantly improved using our workflow, but performance limitations still exist. The results demonstrate that the agentic workflow is effective and our benchmark is challenging. 32. Related Work 

2.1. Multimodal Browsing Agents 

Early systems built upon LLMs demonstrated the feasibility of tool-augmented web navigation, employing search engines and web browsing tools to retrieve and integrate textual information in response to user queries [ 11 , 12 , 13 , 14 , 15 ]. These agent systems established the foundational architecture for iterative retrieval and reasoning, but remained fundamentally constrained by their inability to perceive and process visual content, reducing rich multimodal web environ-ments to text-only representations. The advent of MLLMs drives a paradigm shift toward multimodal browsing. MMSearch-R1 [16 ] is the first end-to-end reinforcement learning framework that drives MLLMs to perform multi-turn searches on demand in real-world internet environments. WebWatcher [ 8] can leverage more external tools and design a data synthesis pipeline to produce high-quality multimodal data for training. DeepMMSearch-R1 [ 17 ], Skywork-R1V4 [ 18 ] and SenseNova-MARS [ 19 ] introduce the image cropping tool and enhance the reasoning ability for query images. Despite their strong performance on current multimodal deep research benchmarks, a rigorous benchmark dedicated to fine-grained image understanding is still missing. 

2.2. Multimodal Browsing Benchmarks 

Multimodal browsing benchmarks are crucial for evaluating the performance of multimodal browsing agents, but existing benchmarks still have limitations. MMSearch [ 7] pioneers the evaluation of multimodal browsing agents by introducing visual content into the search queries but the semantic information of query images can be extracted through reverse image search tool, reducing visual reasoning to text-only tool invocation. Furthermore, the problem complexity remains constrained, with most instances resolvable within limited interaction rounds that do not test sustained multi-hop reasoning capability. BrowseComp-VL [ 8 ] extends this paradigm by increasing the required search iterations, thereby amplifying the depth of information retrieval. VDR-Bench [ 10 ] employs a multi-round cropping strategy to avoid the model taking text shortcuts and using the entire image for searching. MMSearch-Plus [ 9] and MM-BrowseComp [20 ] require the agents to think with the query image and to search using fine-grained visual features, but the core evidence in the search process often includes videos or special files that cannot be accessed from publicly available sources by using tools. To address these limitations, we propose VisBrowse-Bench, a benchmark constructed by human experts from publicly available data. This benchmark ensures that visual evidence is incorporated into the search process and activates the model’s visual reasoning capabilities.

3.1. Data Design Principles 

Our goal is to jointly evaluate the agent’s capabilities in deep search and thinking-with-image, requiring the agent to search for specific visual information during the process and reason with the visual information to obtain evidence. This emphasizes that two key principles must be included in the data design process: 

Multimodal Information Integration: To evaluate genuine multi-modal deep search capa-bilities, we enforce that both the query formulation and evidence acquisition phases require active processing of visual and textual modalities. Each question is paired with a set of ref-erence images containing task-critical cues, ensuring that query understanding necessitates visual perception. And the search space comprises interleaved text documents and visual assets. Successful evidence gathering requires the model to issue multi-modal queries and rank mixed-modality results. 

Visual Competency Enforcement: All visual information is designed to be structurally indis-pensable, and it cannot be substituted by textual captions or semantic descriptions, compelling the model to invoke core visual competencies, including spatial grounding, attribute perception, and relational parsing. The problem uses spatial information to describe visual elements, re-quiring the model to perform precise visual localization. Furthermore, the visual information involves perceptual attributes missing from the text, prompting the model to perform pixel-level analysis. The final information integration for the answer requires cross-image reasoning, which necessitates visual alignment rather than text matching.

The collection workflow proceeds through an iterative entity-event-visual information traversal. The experts first retrieve real-world events associated with the identified entity, then locate complementary visual information through these events. And the visual information must implicate a distinct related entity, distinct from the previous hop. This process of visual retrieval, event bridging, and cross-modal entity transition repeats recursively to construct multi-hop evidence chains with no fewer than two visual evidences, ensuring that both the query formulation and evidence acquisition phases require active processing of visual and textual modalities. The accumulated evidence chains undergo expert synthesis into unified questions through careful annotation, with each question designed to require integration of all retrieved evidence and explicit grounding of intermediate reasoning steps in specific visual and textual blocks, thereby guaranteeing complete instantiation of multimodal integration and visual competency enforcement principles across the entire deep search trajectory. 

3.3. Quality Validation 

Our quality validation ensures strict adherence to both design principles through a multi-layer verification framework. At the instance level, each collected question undergoes the review of two experts to confirm that visual information is structurally indispensable. We additionally enforce that the solution of each question must span a minimum of three hops and at least two visual evidence blocks, with traversal checks ensuring no single-hop shortcut can yield correct answers. For corpus-level quality assurance, we implement solvability and answer uniqueness veri-fication procedures. Each question is independently solved by two domain experts with full reasoning traces documented to ensure that sufficient information exists for deterministic reso-lution. Instances where experts fail to reach consistent answers or identify critical evidence gaps are flagged for reconstruction or elimination. 63.4. Dataset Statistics

3.5. Agentic Framework 

To instantiate the multimodal deep search paradigm, we design a comprehensive agentic workflow equipped with five core tools: text search, image search, reverse image search, image crop and webpage visit. This agentic workflow enables the MLLMs to interact with real-world web environments through iterative tool invocation, dynamically acquiring and integrating heterogeneous evidence across modalities. The detail of agentic workflow is shown in Figure 2. The workflow operates as a closed loop: the MLLM parses the multimodal queries to formulate initial search strategies, issues appropriate tool calls based on current evidence gaps, processes returned results to update its internal state, and refines subsequent queries until sufficient evidence is gathered for answer synthesis. This design ensures that visual and textual evidence acquisition are interleaved throughout the reasoning trajectory, preventing premature collapse into unimodal search patterns. The tool’s detailed description is as follows: • text_search : The agent sends a natural language query, the SerperAPI * returns in-formation about the ranked candidate web pages (including page titles, snippets, and URLs).

3.6. Discussion 

As shown in Table 1, compared to representative multimodal deep search benchmarks, VisBrowse-Bench introduces several key advancements that collectively enhance the evaluation of mul-timodal reasoning and visual-aware search capabilities. VisBrowse-Bench is an important benchmark for evaluating visual-native search. Specifically, VisBrowse-Bench incorporates visual evidence into the reasoning and searching process and requires MLLMs to think with images and reason across multiple images. Furthermore, our data has been verified by experts, and we guarantee that the evidence required for the searching process is publicly accessible.

Implement Details. To quantify the impact of tool usage on performance, we evaluate each model under three progressively enhanced tool-use methods: • Direct Answer: Models answer the question relying on internal parametric knowledge, without external tool access. • + Text Search (+ TS): Models can only use text_search and webpage_visit tools to support evidence acquisition. • + Image Search (+ IS): Models can use all the tools in the Section 3.5 framework to collect visual and textual evidence. 

Evaluation Metrics. We use accuracy (%) as the metric to evaluate the model’s performance on VisBrowse-Bench. Through the regular expression matching method, we extract the model’s final answer. We use GPT-5.1 as the judge model and employ the LLM-as-Judge method to compare the model’s answer with the ground truth and determine its correctness. 5. Conclusion 

We identify two key gaps in existing multimodal browsing agent benchmarks: they inadequately test visual reasoning and overlook multimodal information integration in the reasoning process. 10 To address this, we introduce VisBrowse-Bench, a benchmark constructed by human experts with 169 questions covering seven categories. It is designed so that multimodal information fusion is required during search and visual understanding is essential for task completion. We implement an agentic workflow with tools including text search, image search, reverse image search, image cropping, and webpage browsing to encourage proactive collection and reasoning over visual content.
