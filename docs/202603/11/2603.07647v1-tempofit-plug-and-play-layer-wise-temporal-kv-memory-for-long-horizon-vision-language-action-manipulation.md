## 摘要
预训练的视觉-语言-动作（VLA）策略在单步操纵方面已取得显著进展，但其推理过程在很大程度上仍是无记忆的，这在存在遮挡、状态混淆和细微动作后变化的非马尔可夫长程设置中表现得非常脆弱。先前的方法要么通过堆叠帧来注入历史信息，这会增加视觉 token 数量和延迟，同时引入近乎重复的像素；要么通过学习额外的时间接口，这需要（重新）训练，并可能破坏原始的单帧推理图。我们提出了 TempoFit，这是一种无需训练的时间改造方案，通过状态级记忆升级冻结的 VLA。我们的核心见解是，前缀注意力 K/V 已经形成了一种模型原生的、可内容寻址的运行时状态；跨时间步重用它们可以在不增加新 token 或可训练模块的情况下引入历史信息。TempoFit 在选定的中间层存储逐层 FIFO 前缀 K/V，利用受 NLP 位置偏差启发的固定近期偏差——帧间隙时间偏差（FGTB）执行无参数的 K-to-K 检索，以保持决策以当前为主导，并通过带有保范数重缩放的注意力前残差加载注入检索到的上下文，从而避免冻结权重下的分布偏移。在 LIBERO-LONG 上，TempoFit 将强大的预训练骨干网络的平均成功率提高了多达 4.0%，同时保持了近乎实时的延迟，并能一致地迁移到 CALVIN 和真实机器人长程任务中。

## Abstract
Pretrained Vision-Language-Action (VLA) policies have achieved strong single-step manipulation, but their inference remains largely memoryless, which is brittle in non-Markovian long-horizon settings with occlusion, state aliasing, and subtle post-action changes. Prior approaches inject history either by stacking frames, which scales visual tokens and latency while adding near-duplicate pixels, or by learning additional temporal interfaces that require (re-)training and may break the original single-frame inference graph. We present TempoFit, a training-free temporal retrofit that upgrades frozen VLAs through state-level memory. Our key insight is that prefix attention K/V already form a model-native, content-addressable runtime state; reusing them across timesteps introduces history without new tokens or trainable modules. TempoFit stores layer-wise FIFO prefix K/V at selected intermediate layers, performs parameter-free K-to-K retrieval with Frame-Gap Temporal Bias (FGTB), a fixed recency bias inspired by positional biases in NLP, to keep decisions present-dominant, and injects the retrieved context via pre-attention residual loading with norm-preserving rescaling to avoid distribution shift under frozen weights. On LIBERO-LONG, TempoFit improves strong pretrained backbones by up to +4.0% average success rate while maintaining near-real-time latency, and it transfers consistently to CALVIN and real-robot long-horizon tasks.

### 1. 核心问题与整体含义（研究动机和背景）
当前的视觉-语言-动作（VLA）模型虽然在单步操作上表现出色，但大多遵循“单帧输入、单步预测”的马尔可夫决策范式。这种**“无记忆”推理**在处理长程任务（Long-horizon）时存在严重缺陷：
*   **非马尔可夫挑战**：在存在遮挡、状态混淆（如两个相似物体）或动作后视觉变化细微的情况下，模型容易出现重复操作、步骤遗漏或阶段性中断。
*   **现有方案的局限**：
    *   **帧堆叠（Frame Stacking）**：显著增加了视觉 token 数量，导致推理延迟激增，且引入了大量冗余像素。
    *   **额外时间接口**：需要重新训练或微调模型，破坏了原始预训练模型的推理图，难以实现“即插即用”。

**TempoFit 的目标**是为冻结的预训练 VLA 模型提供一种**免训练、即插即用**的时间记忆增强方案，在不增加输入长度和不改变模型参数的前提下提升长程操作的连贯性。

### 2. 方法论：核心思想与技术细节
TempoFit 的核心思想是将 Transformer 内部的 **Key-Value (KV) 缓存**视为模型原生的、可内容寻址的记忆载体。

*   **逐层 FIFO KV 存储（Layer-Wise FIFO Cache）**：
    *   在模型的特定中间层（而非所有层）维护一个先进先出（FIFO）缓冲区，存储过去 $C$ 个时间步的前缀 KV 状态。
    *   选择中间层是因为这些层通常捕捉更具迁移性的特征，避免了深层任务特定表示的干扰。
*   **K-to-K 检索（Address-Space Matching）**：
    *   利用当前步的 Key 作为查询向量，与历史 Key 进行相似度匹配，从而检索出相关的历史 Value。这种方式完全复用了模型原有的注意力几何空间，无需引入新的投影层。
*   **帧间隙时间偏差（FGTB）**：
    *   为了防止陈旧记忆的干扰，引入受 NLP 中 ALiBi 启发的固定偏置。检索分数会随时间步间隙的增大而线性衰减，确保决策以“当前”为主导，同时保留对关键历史的访问。
*   **保模长残差加载（Norm-Preserving Residual Loading）**：
    *   将检索到的历史上下文通过残差方式注入当前 KV 状态。
    *   **关键细节**：为了防止加法操作导致 KV 量级偏移（进而影响 Softmax 稳定性），对融合后的张量进行重缩放，使其保持原始的 $\ell_2$ 范数。

### 3. 实验设计
*   **数据集与场景**：
    *   **LIBERO-LONG**：包含 10 个复杂的多子目标操纵任务。
    *   **CALVIN**：评估在 D-D（同域）和 ABC-D（跨域）设置下的连续指令执行能力。
    *   **真实机器人**：使用 Realman RM-65B 机械臂执行 3 个长程任务（如清理桌面、按序摆放物体）。
*   **Benchmark 与对比方法**：

    *   **对比方法**：包括 Seer (预测逆动力学)、UniVLA、MemoryVLA (学习型记忆)、HiF-VLA (基于运动表示) 等。

### 4. 资源与算力
*   **推理算力**：论文在 **NVIDIA RTX 5090 GPU** 上测量了推理延迟和显存占用。
*   **训练算力**：**未提及训练时长**，因为该方法的核心卖点是 **Training-free（免训练）**。它直接作用于冻结权重的预训练模型推理过程，因此不需要额外的训练算力。

### 5. 实验数量与充分性
*   **实验规模**：
    *   在 LIBERO-LONG 上进行了 **500 次试验**（10 个任务 $\times$ 50 次），样本量足以支撑统计显著性。
    *   在 CALVIN 上评估了连续 5 条指令的平均完成长度。
    *   真实机器人实验包含 3 个任务，每个任务 20 次试验。
*   **消融实验**：非常详尽，涵盖了组件贡献（KV 记忆 vs FGTB）、检索策略（Q-to-K vs K-to-K）、注入策略（连接 vs 残差 vs 保模长残差）、层选择（顶层、底层、中间层）以及缓存容量 $C$ 的影响。
*   **客观性**：通过在两个不同的强大骨干网络（$\pi_{0.5}$ 和 QwenGR00T）上验证，证明了方法的通用性。

### 6. 主要结论与发现
*   **性能提升**：在 LIBERO-LONG 上，TempoFit 将 $\pi_{0.5}$ 的成功率从 92.6% 提升至 **96.6%**（+4.0%），优于许多需要训练的时间模型。
*   **长程鲁棒性**：在 CALVIN 实验中，提升主要体现在后续指令（第 3-5 步）上，说明记忆机制有效缓解了长程执行中的状态漂移。
*   **高效性**：相比于帧堆叠（8 帧时延迟增加 2.48 倍），TempoFit 仅增加了约 **2-4% 的推理延迟**和极少的显存开销，能够满足实时控制需求。
*   **层选择至关重要**：实验发现仅在中间层开启记忆效果最好，全层开启反而会因干扰导致性能下降。

### 7. 优点
*   **即插即用**：无需微调参数，无需改变输入 Token 结构，直接适配现有 VLA 模型。
*   **极低开销**：利用模型原生 KV 空间进行检索，避免了视觉编码器的重复计算。
*   **分布稳定**：通过保模长残差加载技术，巧妙解决了在冻结权重模型中注入外部信息导致的分布偏移问题。
*   **可解释性**：FGTB 偏置提供了一种直观的近期记忆优先机制。

### 8. 不足与局限
*   **超参数敏感性**：层选择（$L_{mem}$）、衰减斜率（$\beta$）和缓存容量（$C$）目前仍需手动指定，不同架构的模型可能需要微调这些超参数以达到最优效果。
*   **记忆深度有限**：虽然提升了长程连贯性，但对于需要极长时间跨度（如数分钟前的信息）或复杂逻辑推理的任务，简单的 FIFO 缓存可能仍显不足。
*   **应用限制**：目前主要在基于 Transformer 架构的 VLA 上验证，对于非 Transformer 架构（如纯 CNN 或 RNN 策略）不适用。
