## 摘要
本文介绍了 IronEngine，这是一个围绕统一编排核心构建的通用 AI 助手平台。该核心连接了桌面用户界面、REST 和 WebSocket API、Python 客户端、本地与云端模型后端、持久化记忆、任务调度、可重用技能、24 类工具执行、兼容 MCP 的扩展性以及面向硬件的集成。IronEngine 引入了一个三阶段流水线——讨论（规划者-审查者协作）、模型切换（显存感知的转换）和执行（工具增强的动作循环）——从而将规划质量与执行能力分离。该系统具有多级整合的分层记忆架构、由 ChromaDB 支持的向量化技能库、支持 92 个模型配置且具备显存感知上下文预算的自适应模型管理层，以及具有 130 多个别名规范化和自动错误纠正功能的智能工具路由系统。我们展示了在文件操作基准测试上的实验结果，在四个异构任务中实现了 100% 的任务完成率，平均总耗时为 1541 秒，并与包括 ChatGPT、Claude Desktop、Cursor、Windsurf 及开源智能体框架在内的代表性 AI 助手系统进行了详细对比。在不披露专有提示词或核心算法的情况下，本文分析了该平台的架构分解、子系统设计、实验性能、安全边界以及比较工程优势。本研究将 IronEngine 定位为面向系统的通用个人助手、自动化框架以及未来以人为中心的智能体平台的基础。

## Abstract
This paper presents IronEngine, a general AI assistant platform organized around a unified orchestration core that connects a desktop user interface, REST and WebSocket APIs, Python clients, local and cloud model backends, persistent memory, task scheduling, reusable skills, 24-category tool execution, MCP-compatible extensibility, and hardware-facing integration. IronEngine introduces a three-phase pipeline -- Discussion (Planner--Reviewer collaboration), Model Switch (VRAM-aware transition), and Execution (tool-augmented action loop) -- that separates planning quality from execution capability. The system features a hierarchical memory architecture with multi-level consolidation, a vectorized skill repository backed by ChromaDB, an adaptive model management layer supporting 92 model profiles with VRAM-aware context budgeting, and an intelligent tool routing system with 130+ alias normalization and automatic error correction. We present experimental results on file operation benchmarks achieving 100\% task completion with a mean total time of 1541 seconds across four heterogeneous tasks, and provide detailed comparisons with representative AI assistant systems including ChatGPT, Claude Desktop, Cursor, Windsurf, and open-source agent frameworks. Without disclosing proprietary prompts or core algorithms, this paper analyzes the platform's architectural decomposition, subsystem design, experimental performance, safety boundaries, and comparative engineering advantages. The resulting study positions IronEngine as a system-oriented foundation for general-purpose personal assistants, automation frameworks, and future human-centered agent platforms.

### 1. 核心问题与整体含义（研究动机和背景）
论文指出，当前的 AI 助手（如 ChatGPT、Claude、Cursor 等）虽然强大，但在工程实践中面临五个核心挑战：
*   **碎片化问题**：AI 助手散落在 Web、CLI、IDE 等不同端点，缺乏统一的编排核心。
*   **单模型瓶颈**：依赖单一模型处理所有任务，导致大模型在简单任务上浪费算力，小模型在复杂推理上能力不足。
*   **短期记忆问题**：大多数系统是无状态的，缺乏跨会话的知识积累和技能学习。
*   **本地部署挑战**：在消费级硬件上运行多个模型时，显存（VRAM）管理和模型切换效率低下。
*   **工具集成不可靠**：缺乏工具调用的别名规范化、错误纠正和回退机制。

**研究动机**：构建一个名为 **IronEngine** 的通用 AI 助手平台，通过系统级的架构设计（而非单纯提升模型参数）来解决上述问题，实现规划质量与执行能力的解耦。

### 2. 方法论：核心思想与关键技术
IronEngine 的核心是一个**三阶段流水线（Three-Phase Pipeline）**：
1.  **讨论阶段（Discussion）**：由 **Planner（规划者）** 生成任务分解方案，**Reviewer（审查者）** 进行质量评估并给出评分（0.0-1.0）和反馈。若评分未达标，则进入迭代修改。
2.  **模型切换阶段（Model Switch）**：具备显存感知能力。根据当前 GPU 状态，自动卸载讨论模型并加载 **Executor（执行者）** 模型，优化 VRAM 使用。
3.  **执行阶段（Execution）**：Executor 根据批准的计划调用工具。

**关键技术细节**：
*   **分层记忆架构（MemoMap）**：将记忆分为会话、流水线、每日摘要和精炼知识四个层级，利用闲时（Pulse 系统）进行自动整合。
*   **向量化技能库（SkillStore）**：基于 ChromaDB，支持“固有技能”和从成功任务中自动提取的“学习技能”。
*   **智能工具路由**：支持 24 类工具，具备 130+ 别名规范化和自动纠错功能（例如将错误的 CLI 调用重定向到文件操作）。
*   **自适应模型管理**：根据模型参数量（≤10B, 10-25B, >25B）提供三级 Tiered Prompt，并根据剩余显存动态计算上下文预算。

### 6. 主要结论与发现
*   **规划与执行分离有效**：通过 Reviewer 的反馈，14B 小模型的规划质量能从 0.1-0.2 提升至 0.8 以上。
*   **高成功率**：在复杂文件操作基准测试中实现了 **100% 的任务完成率**。
*   **工具路由至关重要**：自动纠错机制显著降低了小模型因格式错误导致的执行失败。
*   **显存管理可行**：通过顺序加载模型，可以在 24GB 显存上运行总参数量远超显存限制的多模型流水线。

### 7. 优点与亮点
*   **工程落地性强**：提供了完整的 4.6 万行代码架构说明，涵盖了从 UI 到硬件接口的全栈设计。
*   **创新的 Reviewer 机制**：引入了防幻觉检测、记忆回收检测和评分-文本矛盾检测，增强了系统的可靠性。
*   **本地化与隐私**：实现了完全不依赖云端的“本地优先”架构，且具备 MCP 协议的扩展性。
*   **可视化程度高**：桌面工作站设计让 AI 的推理过程（Thinking Blocks）和工具调用透明化。

### 8. 不足与局限
*   **延迟较高**：由于涉及频繁的模型切换和多轮讨论，响应速度（分钟级）不适合实时交互场景。
*   **平台依赖**：GUI 自动化目前主要针对 Windows 10 进行了优化，跨平台（macOS/Linux）支持尚不完善。
*   **单用户限制**：当前架构假设为单用户环境，缺乏多用户并发下的资源调度和权限隔离机制。
*   **基准测试范围**：虽然在文件操作上表现优异，但尚未在 WebArena 或 SWE-bench 等更广泛的标准智能体基准上进行全面测试。
