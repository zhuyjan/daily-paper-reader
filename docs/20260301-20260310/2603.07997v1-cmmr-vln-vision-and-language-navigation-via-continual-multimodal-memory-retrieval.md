## 摘要
尽管大语言模型（LLMs）已被引入视觉语言导航（VLN）以提升指令理解和泛化能力，但现有的基于 LLM 的 VLN 缺乏选择性召回和利用相关先验经验来辅助导航任务的能力，这限制了它们在长时程和陌生场景中的表现。在这项工作中，我们提出了 CMMR-VLN（基于持续多模态记忆检索的 VLN），这是一个赋予 LLM 智能体结构化记忆和反思能力的 VLN 框架。具体而言，CMMR-VLN 构建了一个以全景视觉图像和显著地标为索引的多模态经验记忆库，以便在导航过程中检索相关经验；引入了检索增强生成流水线，以模拟经验丰富的人类导航员如何利用先验知识；并结合了基于反思的记忆更新策略，选择性地存储完整的成功路径以及失败案例中的关键初始错误。综合测试表明，在仿真和真实测试中，CMMR-VLN 的平均成功率分别比 NavGPT、MapGPT 和 DiscussNav 提高了 52.9%、20.9%、20.9% 以及 200%、50%、50%，阐明了 CMMR-VLN 作为骨干 VLN 框架的巨大潜力。

## Abstract
Although large language models (LLMs) are introduced into vision-and-language navigation (VLN) to improve instruction comprehension and generalization, existing LLM- based VLN lacks the ability to selectively recall and use relevant priori experiences to help navigation tasks, limiting their performance in long-horizon and unfamiliar scenarios. In this work, we propose CMMR-VLN (Continual Multimodal Memory Retrieval based VLN), a VLN framework that endows LLM agents with structured memory and reflection capabilities. Specifically, the CMMR-VLN constructs a multimodal experi- ence memory indexed by panoramic visual images and salient landmarks to retrieve relevant experiences during navigation, introduces a retrieved-augmented generation pipeline to mimick how experienced human navigators leverage priori knowledge, and incorporates a reflection-based memory update strategy that selectively stores complete successful paths and the key initial mistake in failure cases. Comprehensive tests illustrate average success rate improvements of 52.9%, 20.9% and 20.9%, and 200%, 50% and 50% over the NavGPT, the MapGPT, and the DiscussNav in simulation and real tests, respectively eluci- dating the great potential of the CMMR-VLN as a backbone VLN framework.

### 2. 方法论
CMMR-VLN 框架由三个核心模块组成：
*   **多模态经验记忆 (MEM)**：
    *   **构建**：在导航前，为每个视点存储全景 SkyBox 图像和通过 Detic 模型提取的显著地标文本。
    *   **索引**：利用 CLIP 模型生成图像-文本混合嵌入，并使用 FAISS 进行高效的向量索引，实现视点级别的双向查找。
*   **检索增强生成流水线 (RAGP)**：
    *   **指令感知注意力**：在每个导航步骤，利用注意力机制融合当前候选视点的图像特征，重点关注与指令语义相关的方向，生成观测嵌入。
    *   **经验检索**：计算观测嵌入与记忆库中嵌入的余弦相似度，检索最相关的先验经验。
    *   **导航规则转化**：将检索到的经验转化为显性的“导航规则（R）”，作为高优先级约束输入给 LLM（如 GPT-4o），引导其进行 Chain-of-Thought（思维链）推理（分析、规划、行动）。
*   **反思与记忆更新模块**：
    *   **成功案例**：存储完整的成功路径，强化高效路线。
    *   **失败案例**：识别并蒸馏“第一步错误”，将其分类为：中途偏离（MRD）、错误目标识别（FGR）或目标后过度移动（PGC），并存储错误原因以供未来规避。
    *   **经验过滤**：通过比较路径效率或重复性，动态更新记忆库，避免冗余。

### 7. 优点
*   **模拟人类认知**：通过“检索-反思-更新”闭环，模拟了人类从经验中学习并形成导航直觉的过程。
*   **失败蒸馏机制**：不仅记录成功，更重点记录“第一次犯错”的原因，这种负反馈机制显著降低了重复错误的概率。
*   **即插即用**：作为一种检索增强的框架，它可以作为多种 LLM 导航器的骨干增强模块，具有较好的通用性。

### 8. 不足与局限
*   **依赖初始经验**：系统的有效性高度依赖于记忆库中存储经验的质量和覆盖范围；在完全没有任何先验的纯粹新环境中，初始表现可能受限。
*   **检索效率挑战**：随着导航经验的不断累积，记忆库规模会迅速扩大，如何在海量经验中保持极高的检索精度和低延迟是未来需要解决的问题。
*   **API 成本**：虽然比多智能体讨论（如 DiscussNav）成本低，但频繁调用 GPT-4o 等高级模型在长程任务中仍存在一定的经济成本。
