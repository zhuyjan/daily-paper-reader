## 摘要
当前的智能体机器人系统能够编写可执行的代码即策略程序、观察反馈并在多次尝试中修正行为，但它们仍然主要是任务驱动的：只有在明确指令后才会获取可复用技能。我们研究游戏化智能体机器人学习，其中具身编码智能体在下游任务到来之前，将自我导向的玩耍作为持续技能学习阶段。我们引入了RATs（机器人智能体团队），专为玩耍时间的技能获取而设计。在玩耍过程中，RATs提出新颖但可学习的探索性任务，规划并执行机器人代码策略，验证中间进展，诊断失败，基于密集的步骤级反馈重试，并将成功执行提炼为持久化代码技能库。在测试时，智能体从该冻结库中复现相关技能以帮助解决新任务。在LIBERO-PRO和MolmoSpaces上的实验表明，与无玩耍和随机玩耍基线相比，玩耍学习的技能在保留的下游任务上分别提升了20.6和17.0个百分点（相对于LIBERO-PRO和MolmoSpaces上的CaP-Agent0）。此外，只需将学习到的技能检索到上下文中，即可将其嵌入其他推理时的代码即策略智能体中，无需微调底层模型，即可在RoboSuite和真实世界迁移中分别提升8.9和8.8个百分点。

## Abstract
Current agentic robot systems can write executable Code-as-Policy programs, observe feedback, and revise behavior across multiple attempts, but they remain largely task-driven: reusable skills are acquired only after explicit instructions. We study Playful Agentic Robot Learning, where an embodied coding agent uses self-directed play as a continual skill-learning stage before downstream tasks arrive. We introduce RATs, Robotics Agent Teams designed for play-time skill acquisition. During play, RATs proposes novel yet learnable exploratory tasks, plans and executes robot-code policies, verifies intermediate progress, diagnoses failures, retries with dense, step-level feedback, and distills successful executions into a persistent code skill library. At test time, the agent reuses relevant skills from this frozen library to help solve new tasks. Experiments in LIBERO-PRO and MolmoSpaces show that play-learned skills improve held-out downstream tasks over no-play and random-play baselines, with 20.6 and 17.0 percentage-point gains over CaP-Agent0 on LIBERO-PRO and MolmoSpaces, respectively. Moreover, the learned skills can be plugged into other inference-time Code-as-Policy agents by simply retrieving them into the context, improving RoboSuite and real-world transfer by 8.9 and 8.8 points, respectively, without finetuning the underlying model.

- **核心思想**：将玩耍视为显式的技能获取过程，通过自提议任务、执行代码策略、验证/诊断/重试、蒸馏成功行为为持久代码技能库，并在测试时检索重用。
- **系统架构**：**RATs（Robotics Agent Teams）**，包含三个协作团队：
  - **任务提议团队（Task Proposer）**：基于当前场景、技能库、失败记忆，生成候选任务池，利用 **Goldilocks原则** 选择既非太易也非太难的任务：最大化 \( N(τ) \cdot F(τ) \)，其中 \( N(τ) \) 为物体-技能组合新颖度（鼓励未尝试组合），\( F(τ) \) 为能力前沿（偏好在成功率约0.5的“甜区”）。
  - **执行团队（Execution Team）**：由规划器、策略编写器、验证器（规划验证、质量检查、目标验证、逐步骤验证）、失败诊断器组成，形成 **Write-Execute-Verify-Diagnose** 循环。每个任务有多轮重试预算，失败时诊断器定位失败类型（如抓取失败、导航错误）并给出修复建议。对持续卡住的子行为可启动 **子代理（SubAgent）** 独立练习。
  - **记忆管理团队（Memory-Management Team）**：维护两个持久存储——**技能库 L**（实验性→已验证→弃用三级可靠性）和**失败记忆 M**（存储失败教训）。成功时提取参数化代码技能；失败时蒸馏为“条件+错误+修复”形式的教训。每 K 轮执行一次记忆整理（合并重复、删除过时）和技能提议（为反复出现的缺陷自动生成候选函数）。

- **玩耍学习显著提升下游性能**：RATs 在 LIBERO-PRO 上平均成功率从 23.2%（CaP-Agent0）提升至 43.8%（+20.6 pp）；在 MolmoSpaces 上从 21.0% 提升至 38.0%（+17.0 pp）。

### 7. 优点（方法与实验设计亮点）
- **新颖的玩耍范式**：将自我导向玩耍引入Code-as-Policy代理，实现了持续技能积累，而非被动等待任务。
- **多代理团队设计**：通过分层验证（规划验证、逐步骤验证）、失败诊断和子代理，提供密集反馈，使得从稀疏成功/失败信号中提取可重用技能成为可能。
- **即插即用的技能库**：技能库可轻松插入其他Code-as-Policy基线，无需微调底层模型，具有良好的可迁移性和兼容性。
- **Goldilocks任务选择**：平衡新颖性与可学习性，符合发展心理学中“儿童自发练习能力边界附近技能”的观察。
- **计算匹配消融设计巧妙**：确保了玩耍带来的增益不是简单来自更多计算，而是来自技能知识的结构化积累。
- **跨环境和真实世界迁移验证**：不仅评估了域内泛化，还测试了从模拟到另一个未见模拟环境以及到真实硬件的迁移，增加了结论的实用性。

### 8. 不足与局限
- **实验主要在模拟**：虽然有初步真实世界测试，但任务简单、数量有限，需要更大规模物理部署来验证鲁棒的 sim2real 迁移。
- **环境多样性受限**：玩耍技能的学习受限于可用模拟环境的物体、动力学和可操作性的范围，可能限制习得技能的泛化广度。
- **技能复用可能有害**：当检索到的技能不适合下游任务时，反而会降低性能，需要更好的上下文感知选择机制。
- **推理成本高**：RATs 严重依赖 VLM 进行验证和诊断，增加了每次执行的token消耗和延迟。
- **依赖原始控制API**：系统受限于原始级控制原语，难以处理精细操作（如拧螺丝、柔软物体操纵），灵活性受限。
- **缺乏对持续学习灾难性遗忘的讨论**：技能库不断增长，但论文未探讨如何避免旧技能被干扰或遗忘。
