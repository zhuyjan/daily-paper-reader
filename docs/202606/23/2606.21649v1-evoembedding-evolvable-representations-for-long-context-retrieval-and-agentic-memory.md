## 摘要
现有的嵌入模型本质上是静态的：它们孤立地对文本片段进行编码，忽略了周围上下文和时间顺序。本文提出EvoEmbedding，一种生成可演化表示用于检索的新型嵌入模型。它专为长上下文场景设计，其中信息是动态的、顺序的，并需要持续的状态跟踪。我们的设计简单：EvoEmbedding在顺序处理输入时维护一个持续更新的潜在记忆，并利用它与原始内容共同生成可演化的嵌入。因此，对于相同的查询，我们的模型会根据演化的上下文调整其表示以检索不同的目标，超越了静态语义搜索。为了使模型具备这种能力，我们构建了EvoTrain-180K，一个用于潜在记忆和检索联合优化的多样化数据集。此外，我们引入了一个记忆队列来防止循环编码期间的表示崩溃，以及处理显著长度变化并将训练加速3.8倍的段落批处理技术。大量实验表明，我们的模型不仅在多个长上下文检索基准上优于更大规模的专家模型（如Qwen3-Embedding-8B和KaLM-Embedding-Gemma3-12B），而且在下游任务（如个性化）中也能良好泛化，其上下文长度比训练窗口长10倍。值得注意的是，EvoEmbedding无缝集成到智能体工作流中以提升性能。例如，配备我们模型的简单RAG流水线超越了专用的智能体记忆系统。项目页面：https://clare-nie.github.io/EvoEmbedding。

## Abstract
Existing embedding models are inherently static: they encode text segments in isolation, ignoring their surrounding context and temporal order. This paper introduces EvoEmbedding, a novel embedding model that generates evolvable representations for retrieval. It is tailored for long-context scenarios, where information is dynamic, sequential, and requires continuous state tracking. Our design is simple: EvoEmbedding maintains a continuously updated latent memory as it sequentially processes inputs, and uses it alongside the raw content to jointly generate evolvable embeddings. Consequently, for the same query, our model adapts its representation to retrieve distinct targets based on the evolving context, going beyond static semantic search. To equip the model with this capability, we construct EvoTrain-180K, a diverse dataset for the joint optimization of latent memory and retrieval. Furthermore, we introduce a memory queue to prevent representation collapse during recurrent encoding, alongside segment-batching techniques that tackle significant length variance and accelerate training by 3.8$\times$. Extensive experiments show that our model not only outperforms larger-scale specialists (e.g., Qwen3-Embedding-8B and KaLM-Embedding-Gemma3-12B) across a range of long-context retrieval benchmarks, but also generalizes well to downstream tasks (e.g., personalization) with contexts 10$\times$ longer than its training window. Notably, EvoEmbedding seamlessly integrates into agentic workflows to boost performance. For instance, a naive RAG pipeline equipped with our model surpasses dedicated agentic memory systems. ## 1. 核心问题与整体含义（研究动机和背景）

- **静态嵌入模型的内在缺陷**：现有嵌入模型（如 Sentence-BERT、E5、Qwen3-Embedding 等）是静态的，它们孤立地对文本片段进行编码，完全忽略周围上下文和时间顺序。当面对长上下文场景（信息动态、顺序化、需要持续状态跟踪）时，静态模型无法捕捉演化语义，例如用户先安排会议后又推迟，同一查询“会议几点开始”会错误地检索到旧信息。
- **现有方法的不足**：传统 RAG 优化（索引阶段的指代消解、结构化记忆、生成元数据；检索阶段的查询重写、重排序）以及新兴的智能体 RAG（动态路由、多步迭代检索）虽然缓解了部分问题，但带来大量计算开销、推理延迟，且仍不够优化。它们本质上是在静态表示基础上打补丁，而非从根本上解决问题。
- **本文目标**：在表示层面直接克服静态缺陷，通过生成**上下文可演化的嵌入**（evolvable embeddings），使相同查询能根据不同上下文检索出不同目标，实现超越静态语义搜索的动态匹配，并支持长上下文检索和智能体记忆。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：维护一个固定容量的**潜在记忆**（latent memory），顺序处理输入时持续更新该记忆，并联合原始内容生成可演化嵌入。
- **整体架构**（如图 3 所示）：
  - 输入被分割成若干片段（segment），模型顺序处理每个片段。
  - 在每个时间步 \(t\)，对当前输入 \(x_t\) 和先前记忆 \(M_{t-1}\) 并行执行两个任务：
    1. **记忆演化**（Memory Evolution）：通过 LLM + 记忆 LoRA 适配器生成新的潜在记忆 tokens \(\tilde{M}_t\)，并利用 FIFO 队列更新记忆 \(M_t\)。
    2. **表示生成**（Representation Generation）：通过 LLM + 检索 LoRA 适配器，结合历史记忆和当前片段生成嵌入 \(v_t\)。
  - 查询阶段仅执行表示生成任务。

- **关键技术细节**：
  - **潜在记忆队列（Latent Memory Queue）**：采用 FIFO 队列存储最近 \(L\) 步生成的潜在 tokens（总容量 \(C = L \times K\)，默认 \(C=512\)，每步生成 \(K=16\) 个 tokens）。好处：①有界循环，避免重复编码导致的表示崩溃；②有界容量，约束计算复杂度并迫使模型学习融合新旧知识。
  - **动态分段批处理（Dynamic Segment-Batching）**：将 \(k\) 个连续片段拼接后并行处理，动态确定 \(k\) 使总 tokens 不超过阈值（如 2048）。训练加速 3.8×，且性能略有提升。
  - **多 LoRA 设计**：将记忆演化与表示生成的参数解耦，分别使用不同 LoRA 适配器，推理时可灵活切换，避免灾难性遗忘。
  - **联合训练目标**（公式 3-5）：
    - 记忆生成损失 \(L_{\text{mem}}\)：以查询和最终记忆为上下文，用冻结的骨干 LLM 预测答案，交叉熵损失反向传播至记忆模块，迫使记忆状态与 LLM 原生语义空间兼容。
    - 对比表示损失 \(L_{\text{con}}\)：长度加权的多正例对比损失，正例为支持证据的片段，负例为其他片段。加入 \(\log(N+1)\) 长度权重因子，平衡不同长度样本的优化。
  - **EvoTrain-180K 数据集**：自动三阶段构建：①原始上下文构建（网页文本、对话、记忆）；②动态 QA 生成（40+模板、不同规模 LLM 生成简单/复杂问题）；③检索标注与验证（使用 Gemini 3.1 Pro 标注正负样本、过滤幻觉和无关问题）。最终 184,137 个样本，最大 12K tokens、256 片段。

## 6. 主要结论与发现

- **检索性能最优**：EvoEmbedding-4B 在 8 个长上下文检索基准上平均 Recall@10 达 80.5，NDCG@10 达 65.2，超越 KaLM-12B（+7.8%、+8.1%）和 Qwen3-8B（+11.1%）。
- **生成任务泛化强**：用简单 RAG 流水线（Top-4/8/16 检索）即超越专用智能体记忆系统。在 LongMemEval 上准确率 77.6%，远超 LightMem（70.2%）和 A-MEM（65.2%），且零额外记忆构建 tokens。
- **即插即用提升现有系统**：集成至 A-MEM、LightMem、MemoryOS 后分别带来 +19.2%、+13.5%、+20.5% 的绝对提升，优于重排序和推理策略。
- **时序检索能力突出**：能准确区分“firstly”“lastly”等时间意图，相似度曲线和 t-SNE 表明表示空间按时间位置清晰聚类。
- **高效扩展**：仅在 12K tokens 内训练，却能泛化到 128K（10×训练窗口长度），且训练数据量＜同类 1%。

## 7. 优点

- **创新性**：首次提出“可演化嵌入”概念，将潜在记忆与检索表示深度耦合，从根本上克服静态嵌入局限。
- **设计优雅简洁**：FIFO 记忆队列、分段批处理、多 LoRA 等组件简单高效，避免复杂课程学习或链式推理。
- **数据高效**：仅 18.4 万合成样本即超越海量数据训练的模型，体现架构优势。
- **泛化性与可扩展性**：训练上下文短，泛化到长 10 倍的下游任务；模型家族覆盖 0.8B~4B，适用不同场景。
- **即插即用兼容性**：可无缝嵌入现有 RAG 和智能体记忆系统，无需修改模型或额外训练。
- **可控性与可部署性**：潜在记忆仅用于检索而非生成，避免幻觉和灾难性遗忘；无需白盒访问内部状态，兼容 API 黑盒。

## 8. 不足与局限

- **域外退化风险**：受限于计算资源和训练数据规模（仅 180K 样本），在强域外场景下可能性能下降。
- **缺乏多模态支持**：当前架构仅支持文本，扩展到视觉、音频模态需要更大队列容量和更复杂的维度对齐，本文未探索。
- **编码速度与显存权衡**：相比并行编码的静态模型，EvoEmbedding 需要顺序处理上下文（表 6 显示上下文编码时间 22.08s vs Qwen3-4B 的 3.80s），但在峰值显存和性能上更优。
- **未覆盖极长上下文（>128K）**：虽泛化到 128K，但未测试更极端场景（如 1M tokens）下的表现。
- **对生成任务的具体影响分析有限**：虽然显示了检索性能提升，但生成任务中如何与生成模型交互（如注意力掩盖、输入长度限制）未深入讨论。
