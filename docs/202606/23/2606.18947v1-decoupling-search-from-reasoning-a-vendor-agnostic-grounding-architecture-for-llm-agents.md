## 摘要
生产级LLM智能体日益依赖实时搜索，但原生搜索接地将检索策略、提供商选择、证据注入、成本、延迟和生成行为捆绑在单个模型-提供商边界之后。这种耦合使得接地难以检查、调优、重用或移植，并可能触发搜索诱导的冗长（Search-Induced Verbosity），从而破坏严格的输出契约。我们提出了解耦搜索接地（DSG），这是一种与供应商无关的边界，通过兼容MCP的网关将接地移出推理模型，并将提供商路由、源感知上下文渲染、配置回退、检索深度控制以及精确和语义缓存作为一等控制暴露。在SimpleQA、FreshQA和HotpotQA上对五个前沿模型进行测试时，原生搜索在对时效性敏感的FreshQA上领先，但DSG在控制重要时展现出更强的前沿能力：在SimpleQA上，它以91%更低的搜索成本接近原生准确率（86.1% vs 87.7%），保持简洁的答案契约，并达到99.4%的热缓存命中率，延迟降低68%。作为大规模可互换模型代理工作负载的共享生产接地层部署时，DSG在电商查询理解（QIU）工作负载上匹配或略超原生搜索准确率，同时将搜索成本降低超过98%。实时接地最好被视为可优化的接口边界，而非固定的模型特征。

## Abstract
Production LLM agents increasingly depend on real-time search, yet native search grounding bundles retrieval policy, provider choice, evidence injection, cost, latency, and generation behavior behind a single model-provider boundary. This coupling makes grounding hard to inspect, tune, reuse, or port, and can trigger Search-Induced Verbosity that breaks strict output contracts. We present Decoupled Search Grounding (DSG), a vendor-agnostic boundary that moves grounding outside the reasoning model through an MCP-compatible gateway, exposing provider routing, source-aware context rendering, configured fallback, retrieval-depth control, and exact plus semantic caching as first-class controls. Across five frontier models on SimpleQA, FreshQA, and HotpotQA, native search leads on recency-sensitive FreshQA, but DSG exposes a stronger frontier when control matters: on SimpleQA it nearly matches native accuracy (86.1% vs. 87.7%) at 91% lower search cost, preserves concise answer contracts, and reaches a 99.4% warm-cache hit rate with 68% lower latency. Deployed as a shared production grounding layer for large-scale agentic workloads with interchangeable models, DSG matches or slightly exceeds native-search accuracy on an e-commerce query-understanding (QIU) workload while cutting search cost by over 98%. Real-time grounding is best treated as an optimizable interface boundary, not a fixed model feature.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：生产级LLM智能体依赖实时搜索，但**原生搜索接地**将检索策略、提供商选择、证据注入、成本、延迟和生成行为全部捆绑在模型-提供商边界之后，导致：(a) 接地难以检查、调优、复用或移植；(b) 容易引发**搜索诱导的冗长（Search-Induced Verbosity）**——即使给出严格的输出指令（如“只提供最终答案实体”），原生搜索仍会输出解释性段落，破坏下游解析器或精确匹配契约；(c) 提供商锁定、成本固定、延迟不透明。
- **整体含义**：实时搜索接地应被视为一个可优化的系统接口边界，而非固定的模型特性。作者主张将搜索从推理模型中解耦，暴露可独立控制的检索策略、缓存、提供商选择等操作面。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出**解耦搜索接地（Decoupled Search Grounding, DSG）**，通过一个兼容MCP（Model Context Protocol）的网关将搜索接地置于推理模型之外，使搜索成为结构化的工具层。
- **关键技术细节**：
  - **源感知工具格式**：DSG将搜索响应规范化为结构化内部对象（标题、URL、内容片段），以编号片段和显式来源URL的形式呈现给模型，而非合并到不透明的上下文块中。
  - **提供商抽象与回退**：提供Provider Registry，可配置提供商选择顺序、成本元数据；支持YAML适配器快速接入新提供商；提供商超时或错误时自动按回退链切换。
  - **搜索智能层**：实现**精确缓存 + 语义缓存**的双层缓存机制。语义缓存使用嵌入向量相似度匹配，满足阈值（τ）则复用先前结果；否则执行配置的提供商回退链。缓存键按提供商隔离，避免不同提供商缓存污染；支持按领域/时效性设置TTL。
  - **算法流程**（Algorithm 1）：收到查询q → 归一化 → 精确缓存查找 → 命中则直接渲染上下文；否则嵌入查询 → 语义缓存最近邻匹配（cosine相似度）→ 若≥τ则返回缓存 → 否则按回退链依次执行搜索 → 归一化结果并存入精确缓存和语义缓存 → 渲染上下文。
  - **可观测性**：每次请求发出结构化遥测（提供商、检索深度、缓存结果、延迟、成本），支持归因和退化检测。

## 6. 主要结论与发现

1. **成本-准确率Pareto提升**：在SimpleQA上，DSG+BrightData以86.1%准确率接近原生搜索87.7%，但搜索成本从$20/千次降至$1.8（降低91%）；更低的DSG+Serper以83.3%准确率和$0.67/千次提供额外选择。
2. **FreshQA上原生搜索领先**：由于原生提供商（Google）具有紧密集成的时效性索引优势，在所有模型上准确率最高（72.6% vs DSG最佳68.0%）。
3. **电商QIU生产工作负载**：DSG+Serper在零售场景达93.90%准确率（vs 原生93.40%），在长尾场景达87.79%（vs 原生87.62%），同时搜索成本降低超过98%。
4. **缓存效率**：热启动缓存命中率可达99.4%，平均延迟从4570ms降至1465ms（降低68%），边际搜索成本趋近于零。
5. **搜索诱导的冗长（Search-Induced Verbosity）**：在HotpotQA上，Claude Sonnet 4的原生搜索输出78.1%以“Based on the search results...”开头，而DSG的输出保持简洁；正确答案作为子串被包含但输出格式违规的比例达62.3%。
6. **DSG使搜索接地成为可配置的系统控制**：提供商选择、缓存策略、回退、检索深度、输出格式约束均可在网关层独立管理，模型可互换。

## 7. 优点

- **系统设计清晰**：将搜索接地从模型内部解耦到独立网关层，提供了具体可实现的架构（MCP兼容），使得工程团队能够灵活切换提供商、加入缓存、控制成本。

- **开源友好**：承诺发布非专有提示词、聚合结果和配置模板，促进可复现性（文中未明确承诺代码完全开源，但提供配置模板）。

## 8. 不足与局限

- **依赖第三方API**：DSG的性能受限于第三方搜索服务（Serper、BrightData等）的稳定性、索引质量和计价模式。不同时期定价可能变化，结果不具备永久性。

- **模型选择范围**：虽然覆盖了主流前沿模型，但未测试开源模型（如Llama、Mistral）或较小模型，可能限制泛化性。
