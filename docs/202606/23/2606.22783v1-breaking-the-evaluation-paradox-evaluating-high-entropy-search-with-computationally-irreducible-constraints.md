## 摘要
大型语言模型（LLM）的穷举搜索能力评估面临一个根本性悖论：验证完整性需要完整的真实标注，然而高熵枚举任务使得人类无法创建这样的真实标注。这导致基准测试系统性地惩罚那些表现优于人类标注者的模型。尽管在网络搜索和深度研究智能体方面取得了快速进展——它们现在能发出数百次查询、遍历不同网站并综合生成长报告——但评估仍然主要依赖于部分标注的答案集、基于LLM的评判者或避免真正穷举搜索场景的单一答案问题。我们通过将评估范式从模拟混乱的现实转变为构建计算上纯粹的任务来打破这一悖论。我们引入了VERITAS（可验证的遍历评估框架），这是一个基于计算不可约约束原则构建的框架。通过引入新颖的、不可优化的约束，我们创建了可验证的稀疏答案搜索任务，这些任务在计算上等同于穷举枚举。这些约束易于验证，但LLM或搜索引擎无法优化，迫使智能体真正遍历整个搜索空间。VERITAS可以自动生成几乎无限数量的测试用例，具有完美的真实标注和精确的难度控制，每个实例的边际成本主要由哈希计算决定。这不仅为评估不确定性下的系统探索提供了一个稳健的基准，还为生成训练数据以提升这些关键但尚未充分发展的能力提供了一种可扩展的方法。

## Abstract
Evaluating the exhaustive search capabilities of large language models (LLMs) is plagued by a fundamental paradox: verifying completeness requires complete ground truth, yet high-entropy enumeration tasks make such ground truth impossible for humans to create. This causes benchmarks to systematically penalize models for outperforming their human annotators. Despite rapid progress in web-search and deep research agents -- which now issue hundreds of queries, traverse diverse sites, and synthesize long reports -- evaluation still largely relies on partially annotated answer sets, LLM-based judges, or single-answer questions that avoid genuinely exhaustive search scenarios. We break this paradox by shifting the evaluation paradigm from simulating a messy reality to constructing computationally pure challenges. We introduce VERITAS (Verifiable Traversal Assessment for Search), a framework built on the principle of computationally irreducible constraints. By introducing novel, non-optimizable constraints, we create verifiable, sparse-answer search tasks that are computationally equivalent to exhaustive enumeration. These constraints are easy to verify but impossible for LLMs or search engines to optimize, forcing agents to genuinely traverse the entire search space. VERITAS can automatically generate a virtually infinite number of test cases with perfect ground truth and precise difficulty control, with marginal instance cost dominated by hash computations. This provides not only a robust benchmark for evaluating systematic exploration under uncertainty but also a scalable method for generating training data to improve these crucial, yet underdeveloped, capabilities.

## 1. 核心问题与整体含义（研究动机和背景）

- **评估悖论**：现有大语言模型（LLM）在“高熵查询”（HEQ）上的穷举搜索能力评估面临根本性矛盾——要验证搜索的完整性需要完整的地面真值，但高熵枚举任务（如“列出某艺术家的所有歌曲”）本身导致人类无法创建完整的地面真值。任何人工标注的答案集几乎必然不完整。
- **问题后果**：这导致基准测试系统性惩罚那些超出人类标注者发现结果的模型（正确项被误判为幻觉），而奖励仅仅复制人类遗漏项的平庸模型。当前主流基准（GAIA、BrowseComp、WideSearch等）依赖部分标注答案集或LLM评判器，无法准确测量系统在真正穷举搜索场景下的能力。
- **研究目标**：提出一种新评估范式，打破该悖论，使评估能够自动获得完整地面真值，同时保持任务的计算复杂性与原始高熵枚举问题等价。

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 核心思想

- **计算等价性**：将“列出所有X”的穷举搜索任务转换为“找到满足不可优化约束的k个稀疏目标”的问题。两者的最坏情况计算复杂度均为O(N)，但后者允许使用确定性验证（无需人工标注）。
- **计算不可约约束**：使用密码学哈希函数（如MD5）作为约束条件。哈希具有单向性（无法逆转）和语义零效性（输出与输入语义无关），使得搜索引擎无法利用哈希值直接定位目标，智能体必须遍历整个候选空间。

### 2.2 技术细节

- **问题形式化**：
  - 设搜索空间大小为N，k个目标满足非可优化约束。
  - 在随机搜索下，找到所有k个目标的期望搜索代价为 E[L_k] = (N+1)·k/(k+1) ≈ (k/(k+1))·N ，即线性于N。
  - 在有限预算T下成功概率：P_success ≈ (T/N)^k（当T≪N）。该公式揭示了通过调整N和k可精确控制任务难度。
- **语义约束缺陷**：传统语义约束（如“2003年发行的抒情歌曲”）可通过搜索引擎直接优化，退化为O(1)查询，无法测试真正的枚举能力。哈希约束完全阻断此捷径。
- **四种难度层级**：
  - **Easy**：单源枚举（如列出HuggingFace上某个作者的所有模型，找到MD5匹配项）。
  - **Medium**：复合搜索——7个独立枚举子任务，需综合结果找到匹配项。有效搜索空间扩大7倍。
  - **Hard**：深度属性提取——遍历项目后提取隐藏属性（如Git SHA）再哈希，增加单次访问成本。
  - **Extra Hard**：嵌套多任务——7个高复杂度子任务，构建完整格式化字符串再哈希，进一步扩大有效空间。
- **验证**：智能体对候选值计算MD5并与目标哈希比较，验证O(1)且无歧义。

### 2.3 算法流程（文字描述）

1. 系统生成目标哈希值（对真实答案ID或其派生属性计算MD5）。
2. 智能体接收任务：“在所有候选对象中找到满足MD5(标识符) == target_hash 的项”。
3. 智能体使用搜索、浏览、执行Python等工具遍历候选空间（例如访问HuggingFace API获取所有模型ID）。
4. 对每个候选计算MD5并与目标比较，匹配则提交答案。
5. 评估标准：所有目标项必须全部找到（所有k个）。部分匹配视为失败。

## 3. 实验设计

### 3.1 Benchmark与任务

- **VERITAS基准**：基于四个难度层级（Medium/Hard/Extra Hard），含平衡的多语言查询（英语、中文、日语）。
- 排除Easy层级以聚焦现实挑战。每个层级包含多个具体任务（如搜索歌曲、依赖列表等）。
- 评价指标：Pass@4（4次尝试中成功至少一次的比率）和Avg@4（平均成功率），严格全或无标准。

### 3.2 对比模型

- **开源模型**：GLM-4、MiniMax-M2、DeepSeek-v3.1、Qwen3-235B-A22、Kimi-K2
- **闭源模型**：GPT-4o、Gemini-2.5-Pro、Gemini-3-Pro、GPT-5

### 3.3 智能体工具配置

- 所有模型使用相同的四种工具：Search（Google搜索返回前10结果）、Visit（页面内容提取，基于Qwen3-235B-A22）、Exec Python（MD5哈希计算）、Answer（提交答案）。
