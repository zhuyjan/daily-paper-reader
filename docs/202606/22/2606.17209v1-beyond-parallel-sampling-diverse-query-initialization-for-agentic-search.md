## 摘要
测试时扩展对于智能体搜索通常增加深度（即每轨迹更多轮次和令牌）或广度（即更多并行展开）。这里我们关注广度扩展，表明标准并行采样收益递减，将其归因于第一轮查询冗余。当模型在多次展开中发出相似的首次查询时，线程检索重叠证据，后续轮次基于此共享检索。我们通过DivInit解决了这一限制，这是一种在第一轮无需训练的干预方法。DivInit不是采样k个独立的首次查询，而是从单次调用中抽取n个候选，选取k<n个多样化种子，并将它们作为并行轨迹运行。在五个开放权重模型和八个基准测试中，DivInit一致优于标准并行采样，在匹配计算量的多跳问答上平均提升五到七个百分点。代码可在https://github.com/cxcscmu/diverse-query-initialization获取。

## Abstract
Test-time scaling for agentic search typically increases depth (i.e., more turns and tokens per trajectory) or breadth (i.e., more parallel rollouts). Here we focus on breadth scaling, showing that standard parallel sampling yields diminishing returns, tracing this to query redundancy at the first turn. When models issue similar first queries across rollouts, the threads retrieve overlapping evidence, and subsequent turns are conditioned on this shared retrieval. We address this limitation with DivInit, a training-free intervention at the first turn. Rather than sampling k independent first queries, DivInit draws n candidates from a single call, picks k < n diverse seeds, and runs them as parallel trajectories. Across five open-weight models and eight benchmarks, DivInit consistently improves over standard parallel sampling, with average gains of five to seven points on multi-hop QA at matched compute. Code available at https://github.com/cxcscmu/diverse-query-initialization

## 一、核心问题与整体含义（研究动机与背景）
智能体搜索（Agentic Search）的测试时扩展通常有两种方式：**深度**（增加每轨迹的推理轮次）和**广度**（并行运行多个轨迹并聚合结果）。本文聚焦于广度扩展，发现**标准并行采样（Standard Parallel Sampling）存在收益递减问题**。具体而言，当并行运行k条轨迹时，模型在首轮生成的查询高度相似（即“锚点坍缩”现象），导致各轨迹检索到重叠的证据，后续推理轮次基于共享检索，最终失败模式相关化。这一现象浪费了测试时计算资源，限制了广度扩展的实际效果。作者提出**DivInit**，一种无需训练的干预方法，通过多样化首轮查询来打破锚点坍缩，从而以相同计算量显著提升性能。

## 二、方法论：核心思想与关键技术细节

### 核心思想
- **识别问题**：标准并行采样中，k条独立轨迹的首轮查询（turn-1 query）高度相似（低QPD，Query Pairwise Distance），导致文档重叠度高（高Inter-thread document overlap），失败关联性强。
- **解决方案**：在首轮**不等价地选取k个多样化查询**，而非独立采样。即从单次LLM调用生成n>k个候选查询，挑选出最大程度多样化的k个作为各轨迹的起点，后续推理过程不变。

### 技术细节
1. **候选池生成**：单次调用LLM（温度τ=1.0）生成n=16个候选查询，提示要求从不同角度或约束出发。
2. **多样化选择**：使用**最大边际相关性（MMR）** 从候选池中选取k个查询。MMR公式：
   \[
   c^* = \arg\max_{c\in C\setminus S} \left[ (1-\lambda)\min_{s\in S} d_J(c,s) - \lambda d_J(c,q) \right]
   \]
   其中\(d_J\)为令牌级Jaccard距离，\(q\)为原始问题。实验中取\(\lambda=0\)，即最大化所选集合内最小成对距离。
3. **轨迹运行**：每个被选中的查询作为一条独立轨迹的首轮查询，后续按标准ReAct循环（最多T=8轮）运行，聚合方式可选用投票、best-of-N或AggAgent。

### 计算开销
- 标准并行采样：首轮k次独立LLM调用，共需k⋅T次调用。
- DivInit：首轮1次共享调用（输出n个候选），之后k条轨迹各运行T-1轮，总调用次数为1 + k(T-1)，比标准方法节省k-1次首轮调用，实际速度更快（尤其在模型较大时，见附录B.4）。

## 六、主要结论与发现
1. **锚点坍缩广泛存在**：标准并行采样因首轮查询冗余导致收益递减，QPD低、文档重叠高的问题在多模型多基准上一致。
2. **DivInit显著提升性能**：在相同计算预算下，平均提升5~7个百分点（多跳QA上），开放网络推理也有稳定提升（2~6点）。
3. **多样化无需后续轮次干预**：仅首轮多样化足够，扩展到后续轮次无额外收益（表2），因为早期检索决策锚定后续探索。
4. **模型规模与增益正相关**：较小模型（如1.7B）收益有限，存在能力下限，即模型无法有效利用多样化查询。
5. **MMR最大化最小距离最优**：λ=0（纯多样化）优于λ>0（倾向贴近原始问题）或随机选择。
6. **聚合准确率同样受益**：使用AggAgent聚合后，DivInit的pass@1也优于标准采样，但仍有差距到pass@k上限。

## 七、优点
1. **方法简洁有效**：仅修改首轮查询采样方式，无需训练或修改后续搜索循环，即插即用。
2. **计算开销低**：首轮从k次独立调用缩减为1次调用+MMR选择，实际端到端时间更快（尤其大模型）。
3. **实验全面严谨**：覆盖多种模型、基准、消融、案例，统计可靠，公平匹配计算量。
4. **深入分析机制**：通过QPD、ATD等指标揭示了锚点坍缩与性能的关联，提供了清晰的因果解释。
5. **可迁移性**：与任意智能体搜索循环兼容，未来可在RL训练（如GRPO）中用于增加轨迹多样性。

## 八、不足与局限
1. **指标局限**：pass@k反映的是轨迹池的上限，而非实际部署中的单答案准确率。虽然报告了AggAgent pass@1，但与pass@k仍有差距，聚合本身的提升是开放问题。
2. **应用范围限制**：仅在搜索代理场景中验证，锚点坍缩在工具使用、代码生成等其他领域是否类似未探讨。
3. **搜索后端漂移**：SERPER API随时间变化导致可重复性轻微波动（0.1-0.3个百分点），但本地索引部分可完全复现。
4. **判分器偏差**：使用GPT-4o-mini自动评估，可能存在偏见，但作者声称所有条件使用同一判分器，相对比较仍公平。
5. **模型能力下限**：1.7B模型收益有限，表明小模型可能无法有效利用多样化查询，限制了方法在超小模型上的适用性。
6. **候选池生成依赖单次调用**：虽然节省计算，但若LLM一次无法生成高质量多样化候选池（如复杂度极高的问题），可能限制效果。
7. **未评估真实推理成本**：590 GPU-hours仅部分反映训练/评估，实际应用中的延迟和吞吐量分析仅涉及首轮时间，全轨迹影响未深入。
