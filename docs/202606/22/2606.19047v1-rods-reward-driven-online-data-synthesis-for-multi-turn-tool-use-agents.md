## 摘要
多轮工具使用强化学习受限于静态数据集中信息丰富样本的快速消耗。我们观察到，GRPO中的梯度信号集中在具有最高展开奖励方差的任务上，这是Popoviciu上界的结果。因此，接近智能体能力边界（成功与失败大致平衡）的样本贡献了不成比例的大策略梯度。随着训练的进行，该边界不断移动，逐渐耗尽静态数据集中信息丰富的样本。我们提出RODS（奖励驱动在线数据合成）来解决这一消耗问题。RODS通过将进度奖励方差重新用作一种实用的零成本边界检测器（除了用于训练的已计算展开外，无需额外推理），在RL训练与数据生成之间形成闭环。它持续识别此类边界样本，通过技能对齐的重采样管道合成与其结构复杂度（如API拓扑和依赖深度）匹配的新多轮变体，并管理一个与策略共同演化的动态回放缓冲区。从400个人工种子开始，并维护约800个样本的活跃训练池，RODS在需要大约少20倍轨迹的情况下，达到了与17K样本离线管道相当的性能，并在我们的受控设置中优于固定数据RL和环境增强。

## Abstract
Multi-turn tool-use RL is bottlenecked by the rapid depletion of informative samples in static datasets. We observe that the gradient signal in GRPO concentrates on tasks with the highest rollout reward variance, a consequence of the Popoviciu upper bound. Consequently, samples near the agent's capability boundary -- where successes and failures are roughly balanced -- contribute disproportionately large policy gradients. As training progresses, this boundary continuously shifts, which gradually depletes the pool of informative samples in a static dataset. We propose RODS (Reward-driven Online Data Synthesis) to resolve this depletion. RODS closes the loop between RL training and data generation by repurposing the progress reward variance as a practical, zero-cost boundary detector that requires no extra inference beyond the rollouts already computed for training. It continuously identifies such boundary samples, synthesizes new multi-turn variants matching their structural complexity (e.g., API topology and dependency depth) via a skill-aligned resampling pipeline, and manages a dynamic replay buffer that co-evolves with the policy. Starting from 400 human seeds and maintaining an active training pool of ~800 samples, RODS achieves comparable performance to a 17K-sample offline pipeline while requiring roughly 20x fewer trajectories, and improves over fixed-data RL and environment augmentation in our controlled setting.

# 论文总结：RODS: 奖励驱动在线数据合成用于多轮工具使用智能体

## 1. 核心问题与整体含义

- **研究动机**：多轮工具使用智能体的强化学习（RL）训练面临一个关键瓶颈——**静态数据集**中**信息丰富样本的快速消耗**。随着智能体能力提升，原本有训练价值的样本逐渐被掌握或变得不可达，导致梯度信号稀疏甚至消失。
- **整体含义**：RODS 提出一种**在线数据合成框架**，将 RL 训练与数据生成耦合，利用训练过程中已有展开计算的统计量（奖励方差）作为零成本边界检测器，**动态在能力边界处合成新样本**，从而持续为策略提供高梯度信号，极大提升数据效率。

## 2. 方法论

### 核心思想
- 基于 Popoviciu 上界，连续进度奖励（$R_P\in[0,1]$）的方差在平均奖励 $\mu \approx 0.5$ 时理论上界最大。实证验证：能力边界样本的奖励方差是低/高奖励区域的 **2.0–2.2 倍**，梯度信号密度最高。
- 因此，**将平均进度奖励 $\bar{r}_i$ 作为边界探测器**，自然重用 GRPO 已计算的展开统计，无需额外推理成本。

### 关键技术细节

1. **奖励边界检测**
   - 根据 $\bar{r}_i$ 将任务分为三个区域：
     * $\bar{r} > 0.85$：已掌握（信号稀疏）
     * $0.20 \le \bar{r} \le 0.85$：**边界区域**（高方差，目标）
     * $\bar{r} < 0.20$：太难（信号稀疏）
   - 从边界区域按类型配额（基础/长上下文/缺失函数/缺失参数）选择最多 $M$ 个种子，优先选择最接近 $\bar{r}=0.5$ 的样本。

2. **技能对齐合成管道**
   - 不进行简单改写，而是**保持结构同构**：提取种子任务的结构复杂度（API 依赖有向无环图、参数流），生成变体使其$\Phi(x') \approx \Phi(x_{\text{seed}})$。
   - 五阶段多智能体管道：
     * **规划器**：基于种子设计执行计划和新叙事，避开已知失败历史。
     * **执行器**：在模拟环境中实例化，最多 3 次尝试，带错误修正。
     * **语义重写器**：基于统一叙事同时重写整个轨迹的所有自然语言查询，保证连贯性（解决 $C3$）。
     * **评判与细化**：规则检查 + LLM 评判，反馈循环修正。
     * **可选对抗增强**：注入缺失工具、模糊参数等，增加鲁棒性。

3. **动态回放缓冲区管理**
   - **注入控制**：新变体异步生成，在 epoch 边界分批注入，每 epoch 注入量不超过当前池大小的 20%。
   - **退休机制**：三层退出：
     * 烧入过滤：丢弃初始奖励过低的变体。
     * 边界漂移移出：任务达到已掌握或太难阈值后移除。
     * 方差优先剪枝：池超出最大容量 $P_{\max}=400$ 时，优先保留高方差样本。
   - 原始 400 个人工种子始终保留。

### 流程概述
$$
\text{RL训练} \xrightarrow{\text{进度奖励}} \text{边界检测} \xrightarrow{\text{种子}} \text{合成管道} \xrightarrow{\text{变体}} \text{动态回放缓冲区} \xrightarrow{\text{混合数据}} \text{继续RL训练}
$$

## 3. 实验设计

### 基准与数据集
- **分布内评估**：BFCL V3 多轮子集（800 样本），按 400 训练 / 400 测试划分，包含 Base、Missing Function、Missing Parameter、Long-Context 四个平衡子集。
- **分布外评估**：BFCL V4（Web Search、Memory）、τ2-bench（Retail、Airline、Telecom）、ACEBench Agent（Multi-turn、Multi-step），全部为未见过的 API 和交互模式。

### 对比方法

- **Tier 2（数据效率参考）**：FunReason-MT-4B（17K 离线轨迹）、GPT-4o、DeepSeek-V3.2-Exp 等 20 余个参考模型。

- 策略模型：Qwen3-4B-Instruct、Qwen2.5-7B-Instruct、Llama-3.1-8B-Instruct。

- 三阶段课程（所有方法共享）：Stage1 格式奖励、Stage2 基础推理、Stage3 全数据+扩展。

## 6. 主要结论与发现

1. **边界扩展优于固定数据 RL 和环境增强**：在三个模型上，RODS 均超过 Static 和 EnvTuning（例如 Qwen3-4B 达到 56.00%，高于 Static 的 50.00% 和 EnvTuning 的 50.50%）。
2. **数据效率卓越**：仅用 ~800 活跃样本，性能与 17K 离线管道的 FunReason-MT-4B 相当，**数据量减少约 20 倍**。
3. **机制验证**：合成变体奖励稳定落在边界区域（$R_P\in[0.25,0.75]$），边界区域方差是其他区域的 2.0–2.2 倍。
4. **OOD 泛化**：RODS 在 BFCL V4、τ2-bench、ACEBench 上均显著优于基线和 EnvTuning，表明结构同构合成学习到抽象推理模式而非记忆。
5. **数据效率随边界池规模提升但边际递减**：$P_{\max}=200$ 时已接近饱和，$P_{\max}=400$ 为实用点。
6. **组件重要性**：连贯性重写贡献最大（−5.13%），随机种子选择证明边界感知关键（−4.75%），退休机制必要（−3.38%），连续进度奖励优于二值（−3.25%）。

## 7. 优点

- **极高数据效率**：仅需数百种子即可匹敌大规模离线合成，大幅降低数据标注成本。
- **自适应动态课程**：通过奖励方差自动追踪能力边界，无需人工设计课程顺序。
- **无需额外推理**：边界检测完全复用 GRPO 已有展开统计，零额外开销。
- **合成质量保证**：多智能体管道 + 确定性模拟环境验证 + 质量评判，确保语义连贯性和执行正确性。
- **模型无关**：同样适用于 PPO（理论扩展），合成 LLM 替换影响小。
- **与现有方法互补**：可与环境增强（EnvTuning）结合获得进一步增益（附录 I）。

## 8. 不足与局限

- **依赖确定性模拟环境**：合成管道需要可执行 Python 对象来验证执行正确性。对于**不透明环境**（如远程 MCP 服务器）或无法获取内部状态的 API，当前仿真抽象难以适用。
- **计算开销仍存在**：虽然数据量减少 20 倍，但合成过程本身需要与训练对等的 GPU 资源（约为 2× 训练算力），整体成本对于资源有限的团队可能偏高。
- **实验统计性不足**：未报告多次运行的平均值和标准差，单次结果可能受随机性影响；结论的稳健性未充分量化。
- **泛化范围有限**：OOD 实验仅覆盖三个特定基准，更多复杂真实场景（如多域名混合、长尾 API）尚未验证。
- **超参数敏感**：边界阈值（0.20/0.85）、退休阈值（0.95/0.20）、注入比例（20%）等均为手动设定，缺乏理论指导或自动调节机制。
- **与更大模型的对比**：当前最佳结果（56.00%）仍低于部分闭源模型（如 Claude-Sonnet 61.38%、Grok-4 58.88%），RODS 在小模型上高效但未超越顶级大模型。
