## 摘要
外部技能可以在不改变模型权重的情况下改进面向行动的LLM智能体，但当技能从稀疏或噪声轨迹中提炼时，持续更新技能存在风险。一个看似合理的反思可能编码了有用的流程、虚假的捷径或目标执行器无法可靠遵循的规则。我们提出基于假设的技能优化（HDSO），这是一个无需训练的框架，其中技能管理器和智能体执行器都是冻结的推理端点。管理器观察执行器轨迹，提出带有明确验证计划的可证伪假设，将假设实例化为候选技能包，通过配对的控制/治疗执行来验证技能包，审查行为差异，并将仅受支持的候选技能整合到批准的库中。执行器通过渐进式披露使用批准的技能，在未选择技能时保留仅执行器路径。在ALFWorld上，HDSO将仅执行器基线的平均成功率分别提升了Qwen3-8B的+6.9个点和Qwen3.6-27B的+4.0个点。在技能发现和验证过程中，当成功/失败反馈有20%随机翻转时，HDSO保持了Qwen3-8B的+7.1个点增益。迁移和异类配对诊断进一步表明，经过验证的技能库在其生成运行之外也能发挥作用，但跨模型管理仅在管理器诊断、执行器能力和验证证据一致时成功。HDSO为冻结的动作智能体提供了可审计的技能生命周期，而非无约束的记忆累积过程。

## Abstract
External skills can improve action-oriented LLM agents without changing model weights, but persistent skill updates are risky when they are distilled from sparse or noisy trajectories. A plausible reflection may encode a useful procedure, a spurious shortcut, or a rule that the target executor cannot reliably follow. We propose Hypothesis-Driven Skill Optimization (HDSO), a train-free framework in which both the skill curator and the agent executor are frozen inference endpoints. The curator observes executor traces, proposes a falsifiable hypothesis with an explicit validation plan, instantiates the hypothesis as a candidate skill package, validates the package through paired control/treatment executions, reviews behavior differences, and consolidates only supported candidates into an approved repository. The executor consumes approved skills through progressive disclosure, preserving the executor-only path when no skill is selected. On ALFWorld, HDSO improves executor-only baselines by +6.9 Avg. SR points for Qwen3-8B and +4.0 points for Qwen3.6-27B. Under 20% randomly flipped success/failure feedback during skill discovery and validation, HDSO preserves a +7.1-point gain for Qwen3-8B. Transfer and heterogeneous-pair diagnostics further show that validated repositories can be useful beyond the run that produced them, but cross-model curation succeeds only when curator diagnosis, executor capability, and validation evidence align. HDSO provides an auditable skill lifecycle for frozen action agents rather than an unconstrained memory accumulation procedure.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- LLM 智能体在行动导向场景中常因程序性错误（如重复搜索、无效动作格式、前置条件未满足）而失败。外部技能（可复用的过程知识）可在不改变模型权重下改进智能体，但直接从稀疏或含噪的轨迹中提炼技能存在风险：一条看似合理的反思可能编码有用过程、虚假捷径或执行器无法可靠遵循的规则。
- 现有方法（如直接追加反思、无约束记忆累积）易导致技能库过泛、矛盾，且缺乏审计性。因此需要一种安全、可审计的技能更新机制，将每次更新视为假设，并通过前瞻性验证（paired control/treatment）来确认效果，只保留有证据支持的技能。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- **假设驱动技能优化（HDSO）**：将技能更新形式化为一个可证伪假设的生命周期，包括观察、提案、验证、评审、整合和拒绝。整个框架无需训练，策展人（curator）和执行器（executor）均为冻结的推理端点。
- 策展人观察执行器的轨迹，提出结构化假设并附带验证计划；通过配对执行（控制组使用当前库，治疗组加入候选技能）来评估假设；评审阶段根据配对结果和策展人判断决定是否将候选技能整合到批准库中。执行器通过渐进式披露使用技能：先看到紧凑的技能卡，需要时才请求完整技能体；无技能时保持基线行为。

### 关键技术细节
- **四个不变量**：①无技能的执行器路径与基线完全相同；②候选技能是有作用域和可证伪条件的结构化artifact；③验证是前瞻性且配对的（控制/治疗运行在相同任务索引上）；④被拒绝的候选保留在假设账本中，避免未来重复发现同一条失败规则。
- **执行器**：标准ReAct风格环境策略，增加可选技能接口。每步检索至多3个紧凑技能卡；执行器可请求其中一个技能的完整内容。请求被视为技能采纳事件并记录。
- **策展人优化循环**：维护批准库S、假设库H、证据账本L。每轮先收集发现轨迹（每个任务类型3条新轨迹，后续轮次补充不重叠轨迹）；策展人接收紧凑行为证据，输出结构化假设（包含行为声明、证据、作用域、预期变化、风险、回滚条件、技能artifact等）。候选技能验证分阶段：筛选小（4个目标任务+2个守卫任务）、筛选中（8+4）、确认（32+12）。每个阶段计算净胜数（治疗独赢-控制独赢）和ΔSR（成功率差）。守卫任务考察作用域外是否退化。通过阈值（净胜≥1且ΔSR≥0.05，筛选小阶段0守卫回归，后续≤1）和策展人评审决定是否批准。记录McNemar/binomial p值作为诊断，但不作为硬阈值。
- **评分公式**：
  ```
  net(s;V) = b - c
  ΔSR(s;V) = (1/|V|) Σ (y(τ^1_i) - y(τ^0_i))
  ```
  其中b为治疗独赢数，c为控制独赢数。p值=Pr[Binom(b+c,0.5)≥b]用于审计。

- **噪音鲁棒性**：在20%成功/失败标签翻转（仅优化时）条件下，策展人被告知可能存在噪音，系统依然能产出有用技能。

## 6. 主要结论与发现

- **HDSO能显著提升冻结LLM智能体性能**：同质设置下Qwen3-8B Avg. SR从40.5升至47.4（+6.9），Qwen3.6-27B从61.7升至65.7（+4.0）。异质设置（8B策展人→9B执行器）达到+7.8，是最大增益。
- **噪音反馈下保持鲁棒**：20%标签翻转时Qwen3-8B仍获得+7.1增益，且技能使用集中在目标任务类型，避免了泛泛的建议。
- **已批准技能库可跨模型迁移**：使用27B→27B产生的库通过渐进式披露交付给8B执行器，获得+5.7增益（而全技能上下文只获+1.6），表明技能携带可重用过程知识，且交付方式重要。
- **异质主动管理有边界**：强势策展人（如27B）为弱执行器（8B）写技能时，可能在筛选阶段有正效果但确认阶段逆转，因为执行器无法稳定执行高水平指令。Qwen3.5-9B作为策展人在所有组合中均未能产生稳定库，表明并非所有模型都适合作为策展人。
- **技能主要改善程序性失败**（如pick_heat_then_place任务中直接加热动作选择），对已稳定任务（如clean）无提升。

## 7. 优点

- **无需训练**：策展人和执行器均为冻结端点，适合生产环境（固定模型、成本、合规等约束）。
- **可审计**：每个批准的技能都有作用域、预期效果、验证证据、回滚条件；被拒绝的假设保留在账本中，形成可追溯的失败记录。
- **前瞻性配对验证**：通过控制/治疗运行在相同任务上评估候选技能，避免因随机性导致的虚假关联。分阶段筛选提高效率。
- **渐进式技能披露**：先展示紧凑卡片，仅当执行器请求时才揭示完整内容，减少上下文污染，保持基线行为。
- **噪音鲁棒**：在优化信号不可靠时仍能产生有用技能，且倾向产生窄而可验证的技能，而非泛泛的记忆。

## 8. 不足与局限

- **基准覆盖局限**：仅在ALFWorld（文本动作基准）上评估，未在问答、纯推理或更复杂的具身环境上测试。作者承认其技能接口可能无法直接迁移到单轮问答等场景。
