## 摘要
大型语言模型（LLM）多智能体系统越来越依赖于不仅单个智能体如何推理，还依赖于智能体之间的连接方式。本文介绍了QueenBee Planner，一个将智能体间通信拓扑视为可检索和自我改进的设计技能的框架。工作智能体池、任务适配器和评分函数是冻结的；只有外部LLM规划器学习生成时间通信DAG，指定谁在哪个轮次向谁发送信息，谁合并消息，以及谁发出最终答案。执行轨迹被提炼为有证据支持的设计规则，包含三个动作：保留、修改和避免。为了防止自我演化将偶然的成功或看似合理但错误的解释转变为策略，QueenBee使用了保留集接受门、方差感知信用、基元级别归因、迁移信任、洞察证伪和结构去重。我们在计数频率聚合和Silo-Bench风格的分布式协调任务上评估了该方法。在固定工作智能体的情况下，自我演化的图生成产生的通信结构优于固定拓扑和冷生成。在CF全测试设置中，最佳生成图将RMSE从最强固定拓扑的12.53降低到7.87，同时减少了消息、模型调用和token成本；Silo风格的结果显示了与冷生成和固定拓扑基线相同的改进方向。这些结果表明，多智能体系统可以学习可重用的架构设计知识，而不仅仅是记忆任务答案。

## Abstract
Large language model (LLM) multi-agent systems increasingly depend not only on how individual agents reason, but also on how agents are connected. This paper introduces QueenBee Planner, a framework that treats inter-agent communication topology as a retrievable and self-improving design skill. A pool of worker agents, the task adapter, and the scoring function are frozen; only an outer LLM planner learns to generate temporal communication DAGs specifying who sends information to whom, in which round, who merges messages, and who emits the final answer. Execution traces are distilled into evidence-backed design rules with three actions: \emph{Preserve}, \emph{Modify}, and \emph{Avoid}. To prevent self-evolution from turning lucky runs or plausible but false explanations into policy, QueenBee uses held-out acceptance gates, variance-aware credit, motif-level attribution, transfer trust, insight falsification, and structural deduplication. We evaluate the method on Count-Frequency aggregation and Silo-Bench-style distributed coordination tasks. With fixed workers, self-evolved graph generation produces communication structures that improve over fixed topologies and cold generation. In the CF fulltest setting, the best generated graph reduces RMSE from 12.53 for the strongest fixed topology to 7.87 while also reducing messages, model calls, and token cost; Silo-style results show the same direction of improvement over cold and fixed-topology baselines. These results suggest that multi-agent systems can learn reusable architectural design knowledge rather than merely memorizing task answers.

*   **研究动机**：现有的 LLM 多智能体系统通常使用固定的通信拓扑（如链式、星型、网状），缺乏根据任务自动调整路由的能力。虽然已有研究将通信图作为优化对象（如 GPTSwarm、G-Designer），但这些方法通常是一次性设计，无法积累并复用跨任务的设计知识。同时，一些基于经验库（skill library）的工作主要存储任务求解行为，而非智能体间连接方式的架构知识。
*   **核心问题**：能否将多智能体系统的通信拓扑本身作为一种可学习、可演进、可迁移的“设计技能”？
*   **整体含义**：论文提出 QueenBee Planner，通过一个外部 LLM 规划器冻结工作智能体，只让规划器从执行经验中学习生成时间通信 DAG，并蒸馏出 Preserve/Modify/Avoid 三类规则。这种“架构学习”表明系统可以学习的不仅是答案，更是如何组织智能体之间信息流动的知识。

## 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：
    *   将多智能体通信拓扑视为一个可检索、自改进的设计技能。
    *   冻结工作智能体池（worker pool）、任务适配器、评分函数，只让一个外部的 LLM 规划器（planner）学习生成**时间通信 DAG**（指定每个轮次谁向谁发送消息、谁合并消息、谁输出最终答案）。
    *   将执行轨迹蒸馏为带有证据的设计规则，动作包括：**Preserve**（保留）、**Modify**（修改）、**Avoid**（避免）。
*   **关键技术细节**：
    1.  **时间通信 DAG**：形式化为 G = (N, T, E, h, I, R)，其中 N 为智能体数，T 为轮次，E 为有向时间边。具有同步屏障语义、可行性约束（如最大轮次、最大边数、最大入度）。
    2.  **技能库 (Skill Bank)**：每条技能由（适用条件 z, 设计规则 r, 动作 a, 统计摘要 q, 证据 E）组成。Preserve 直接复用可执行 DAG；Modify 在保留核心信息流的前提下改写；Avoid 作为负约束。
    3.  **基元级信用分配**：将生成图分解为 8 个手工结构特征（如步数、消息数、扇入桶、是否有审计边等），每个基元获得带样本数惩罚的保守损失估计 \(\tilde{L}_k = \bar{L}_k + \kappa_m / \sqrt{n_k}\)。候选图的先验分数为共享基元的加权平均。
    4.  **自演化循环**（算法 2）：
        *   规划器生成候选 DAG → 执行收集证据 → 蒸馏候选规则。
        *   **保留集接受门**：候选规则必须在保留集（held-out）上使得规划器性能优于旧技能库，且达到显著性阈值 \(\epsilon\)。
        *   **方差感知信用**：采用 \(\tilde{L}_{risk} = \bar{L} + \kappa_s \sigma / \sqrt{n}\) 避免单次幸运运行主导。
        *   **洞察证伪**：LLM 提出的自然语言洞察需通过保留集检验才能入库。
        *   **迁移信任**：技能不能自动迁移到所有任务，需基于任务特征槽的证据强度决定使用 Preserve/Modify 或冷启动。
        *   **结构去重**：用拓扑等价哈希合并同构 DAG，避免技能库膨胀和信用分散。
    5.  **总目标**：最小化 \(J(G;x) = L(\hat{y}, y) + \lambda C(G)\)，更新仅改变技能库 S，不改变工人、适配器、评分函数。
