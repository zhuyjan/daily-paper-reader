## 摘要
基于知识的视觉问答（KB-VQA）要求模型将图像理解与外部知识相结合。以往大多数方法使用固定的检索-生成流水线，并预设检索器和静态top-k设置，在推理过程中缺乏自适应性。我们提出ProMSA，一种用于KB-VQA的渐进式多模态搜索智能体。给定图像-问题对，该智能体在明确的工具调用预算下，迭代选择图像搜索、文本搜索或停止，并通过去重操作避免冗余检索。在训练中，我们首先使用拒绝采样SFT学习有效的工具使用格式，然后通过TN-GSPO（一种序列级强化学习目标，通过生成长度和工具交互深度对更新进行归一化）优化智能体。在E-VQA和InfoSeek上的实验表明，相对于强RAG和智能体基线方法，本方法在检索和端到端准确率上均取得了一致提升。代码已开源：https://github.com/DingWu1021/Promsa。

## Abstract
Knowledge-based Visual Question Answering (KB-VQA) requires models to combine image understanding with external knowledge. Most prior methods use a fixed retrieve-then-generate pipeline with a pre-selected retriever and a static top-k setting, which is not adaptive during reasoning. We propose ProMSA, a progressive multimodal search agent for KB-VQA. Given an image-question pair, the agent iteratively chooses image search, text search, or stop, under explicit tool-call budgets and with deduplication to avoid redundant retrieval. For training, we first use rejection-sampling SFT to learn valid tool-use formats, then optimize the agent with TN-GSPO, a sequence-level RL objective that normalizes updates by both generation length and tool-interaction depth. Experiments on E-VQA and InfoSeek show consistent gains over strong RAG and agent baselines, and improved retrieval and end-to-end accuracy. The code is available at https://github.com/DingWu1021/Promsa.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：基于知识的视觉问答（KB-VQA）要求模型同时理解图像内容并利用外部知识（如Wikipedia）来回答关于长尾实体或细粒度属性的问题。
- **现有方法局限**：主流方法采用固定的“检索-生成”流水线（RAG），使用预选的检索器和静态top-k设置。该范式存在三个主要不足：
  - **工具与检索策略非自适应**：无法根据当前推理状态灵活选择检索模态（图像/文本）或是否检索。
  - **缺乏失败恢复机制**：当首次检索结果不可靠时，系统无法修正，只能强行基于错误证据回答。
  - **难以支持多跳证据链**：静态top-k注入无法逐步扩充推理所需证据。
- **核心动机**：KB-VQA需要一个将检索与推理紧密耦合的闭环系统，使模型像“预算感知的研究员”一样：不确定时检索，证据不足时继续搜索，证据充分时停止。

## 2. 方法论

### 2.1 核心思想
- 将KB-VQA建模为**预算约束下的渐进式搜索与推理问题**。智能体在有限次数内自主决定下一个动作：图像搜索、文本搜索或停止并给出答案。
- 使用**去重机制**避免重复检索已访问页面。
- 引入**显式纠正机制**：当实体识别不确定时，可排除前序候选再次触发图像检索；当实体已知但属性缺失时，可重写查询触发文本检索。

### 2.2 关键技术细节
- **动作空间**：\( u_t \in \{ \text{img\_search}, \text{text\_search}, \text{stop} \} \)，分别对应图像检索、文本检索和停止回答。
- **预算约束**：\( B_{\text{img}} \) 和 \( B_{\text{text}} \) 分别为图像搜索和文本搜索的最大调用次数（实验中设为3次）。
- **状态更新**：每步接收到检索结果后，压缩为问题相关的简短摘要，追加到上下文形成新状态 \( s_{t+1} = \text{Update}(s_t, o_{t+1}) \)。
- **去重**：维护已访问页面集合 \( P_t \)，每次工具调用时传入排除列表，避免重复检索。
- **统一生成式策略**：使用单个多模态大模型参数化策略 \( \pi_\theta \)，每一步生成推理文本、动作token和参数args。

### 2.3 训练流程（两阶段）
- **阶段I：冷启动SFT**（Rejection-Sampling SFT）
  - 从初始策略采样 \( N \) 条轨迹，过滤出格式正确、可执行、答案正确的轨迹，最多每问题保留1条。
  - 在构建的SFT数据集上做最大似然训练，使模型学会基本工具调用格式和执行模式。
- **阶段II：强化学习（RL）**
  - 采用序列级策略优化，奖励函数考虑答案正确性、格式正确性和工具调用成本（\( R(\tau) = r_{\text{ans}} + r_{\text{fmt}} - \lambda C(\tau) \)，成本正则化）。
  - 提出**TN-GSPO**（Tool-Normalized Group Sequence Policy Optimization）：在GSPO基础上引入工具交互深度归一化，使更新尺度同时依赖于生成token数 \( L(\tau) \) 和工具调用深度 \( H(\tau) \) （\( D(\tau) = L(\tau) \cdot (1 + c H(\tau)^\alpha) \)），从而减少生成长度波动带来的偏置，稳定学习。
  - 使用非对称裁剪（asymmetric clipping）进一步稳定训练。

## 3. 实验设计

### 3.1 数据集与Benchmark
- **E-VQA**（Encyclopedic-VQA）：约221K问答对，覆盖约16.7K Wikipedia实体，分为单跳和多跳问题。提供约2M页的知识库。
- **InfoSeek**：约1.3M问答对，约11K不同Wikipedia页面，包含未见实体（unseen entities）的泛化测试。使用100K页子集作为检索语料库。
- **OK-VQA**（额外泛化测试）：常规外部知识VQA基准。
- 评估指标：E-VQA采用BERT Matching（BEM）衡量语义一致性；InfoSeek报告标准VQA准确率和Relaxed Accuracy。

### 3.2 对比方法
- **零样本MLLMs**：BLIP-2、GPT-4V、Qwen2.5-VL-7B、Qwen3-VL-2B/8B等。
- **搜索智能体模型**：MMSearch-R1、DeepEyesV2（均使用相同骨干Qwen2.5-VL-7B，检索器使用BGE+EVA-CLIP）。
- **检索增强模型**：EchoSight、CC-VQA、REAL、MaS-VQA等（使用相同检索器EVA-CLIP-8B或BGE+EVA-CLIP，骨干为Qwen2.5-VL-7B或Qwen3-VL-8B）。

### 3.3 主要实验结果
- 在E-VQA和InfoSeek上，ProMSA在所有指标上均超过所有对比方法（包括零样本、搜索智能体、RAG方法）。例如：
  - E-VQA All：Ours (Qwen3-VL-8B) 52.6 vs. BEST RAG (REAL) 44.1。
  - InfoSeek All：Ours 53.4 vs. CC-VQA 45.1。
- 将Qwen2.5-VL-7B作为骨干时，ProMSA也显著优于MMSearch-R1和DeepEyesV2。
- 在OK-VQA上，ProMSA (Qwen3-VL-8B) 达到85.6%，优于CC-VQA (78.8%)。

## 6. 主要结论与发现

1. **渐进式多模态搜索智能体能显著提升KB-VQA性能**：在E-VQA和InfoSeek上均达到SOTA，相比最强RAG基线提升约8-9个百分点。
2. **两阶段训练（SFT+RL）有效**：冷启动SFT确保基础执行能力，RL进一步优化搜索策略，结果逐阶段提升（E-VQA All：32.8 → 38.6 → 52.6）。
3. **TN-GSPO优于其他RL方法**：将工具交互深度纳入归一化，训练更稳定，最终准确率最高（E-VQA 52.6 vs GRPO 44.2）。
4. **工具协同重要**：同时使用图像和文本检索比单一模态大幅提升（两者均用时E-VQA 52.6 vs 仅图像27.6/仅文本34.7）。
5. **纠正机制有效**：在交互轮次分析中，第一轮检索失败后有较高概率继续检索（46.4%），第二轮检索准确率从39.1%提升到48.8%，表明模型能主动纠正错误。
6. **泛化能力**：在OK-VQA上同样取得显著增益，说明学到策略不局限于训练集。

- **RL目标设计**：TN-GSPO合理地将工具交互深度纳入归一化，解决了标准序列级RL中因生成长度波动导致的训练不稳定问题，是一个针对性强的改进。

## 8. 不足与局限

- **依赖检索质量**：检索器（EVA-CLIP/BGE）性能对最终结果有重要影响，文中未对比更换更强检索器（如基于LLM的检索）的增益。

- **多语言/多模态扩展**：当前仅针对英语Wikipedia，未验证对其他语言或非Wikipedia知识库的适应性。
- **总体风险**：方法复杂度增加（多轮交互、摘要压缩、RL训练）可能带来部署和维护挑战，实际应用需考虑稳定性。
