## 摘要
Web代理基准测试主要衡量深度——在一系列约束之后找出一个晦涩的答案——而广度，即穷举枚举一个封闭集合并填写每个项的属性，很少被评估，尤其是在英语之外。广度也难构建：验证一个黄金集合是否完整以及每个单元格是否正确，比检查单个答案要昂贵得多。我引入了Ko-WideSearch，一个通过自动合成与验证管道构建的韩语广度搜索基准。每个任务命名一个集合父实体——一个电视剧季、一个王朝、一个联盟、一个行政区域、一个选举——并要求其完整成员列表加上每个项目的属性表，通过项F1、列F1和行F1进行评分。它涵盖了190个实体的228张表，以及三个难度层级中的十六个类别，这些层级由我独立调整的两个结构旋钮设置——表宽度和二维复合键——因此交叉积成员从0%增加到100%。一个单一的正则化感知比较器在黄金构建和评分之间共享，这样稳定的日期和计数列不会仅仅因为格式而过度掉落。在二十个Web代理中，失败是一致的：代理恢复了集合但没有恢复行（例如，项F1为92.8，而行F1为53.7），随着旋钮变硬，准确率稳步下降，更多的搜索或更多的花费都没有缩小差距。按单元格细分，困难部分是找到正确的值，而不是格式化它：开放式自由文本单元格失败最多，而具有标准答案（如日期或名称）的单元格通常表现正确。

## Abstract
Web-agent benchmarks overwhelmingly measure depth -- pinning one obscure answer behind a chain of constraints -- while breadth, exhaustively enumerating a closed set and filling each item's attributes, is barely evaluated, especially outside English. Breadth is also hard to build: certifying that a gold set is complete and every cell correct is far costlier than checking a single answer. I introduce \textsc{Ko-WideSearch}, a Korean breadth-search benchmark built by an automated synthesize-and-verify pipeline. Each task names a set-parent entity -- a TV season, a dynasty, a league, an administrative region, an election -- and asks for its full membership plus a per-item attribute table, graded by Item-, Column-, and Row-F1. It spans 228 tables over 190 entities and sixteen categories across three difficulty tiers, set by two structural knobs I dial independently -- table width and a 2-D composite key -- so cross-product membership climbs from 0\% to 100\% across the tiers. A single normalization-aware comparator is shared between gold construction and grading, so stable date and count columns are not over-dropped on formatting alone. Across twenty web agents, the failure is consistent: agents recover the set but not the rows (e.g.\ Item-F1 92.8 against Row-F1 53.7), accuracy falls steadily as the knobs harden, and neither more search nor more spend closes the gap. Broken down by cell, the hard part is finding the right value, not formatting it: open-ended free-text cells fail most, while cells with a standard answer such as a date or a name usually come out right.

# 详细论文总结：Ko-WideSearch

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：现有的Web代理基准测试（如BrowseComp、K-BrowseComp）主要衡量“深度搜索”——通过多跳约束找到单个隐藏答案。而“广度搜索”，即穷举地枚举一个封闭集合（例如列出所有韩国廉价航空公司）并为每个成员填充跨页面的属性表格，却很少被评估，尤其在英语之外。
- **背景**：广度搜索需要代理同时做到两件事：完整恢复集合（不漏、不伪造），并为每一行跨多个页面填充正确属性。这构成一种不同于深度的独立能力挑战。
- **研究意义**：广度搜索能暴露代理在“保持结构化集合并跨页面完成填充”方面的系统性缺陷，而现有基准无法衡量这一点。此外，韩语环境下的广度搜索评估完全空白。因此本文构建了Ko-WideSearch，填补了这一缺口。

## 2. 论文提出的方法论

- **核心思想**：通过自动化“合成-验证”流水线构建韩语广度搜索基准，并用两个结构旋钮（表宽度和二维复合键）控制难度层次，最后用统一的类型感知比较器进行评分。
- **关键技术细节**：
  - **流水线**：
    - **构建阶段**：一个“构建代理”接收实体种子（如“韩国有哪些低成本航空公司”），通过搜索/打开/查找工具进行穷举搜索，自动生成边界明确的枚举问题和黄金表格。
    - **验证阶段**：三个独立门控确保质量：
      1. **不可记忆性**：闭卷模型不能从记忆复现黄金单元格（不仅限于集合成员名，还包括属性值）。
      2. **完整性**：另一个独立代理重新枚举集合，与黄金集合的Set-F1≥阈值，确保集合边界无歧义。
      3. **跨源属性验证**：独立重新查找每个属性的值，不一致的列会被视为“源脆弱”而丢弃。要求至少保留一个跨源验证通过的属性列。
  - **难度旋钮**：
    - **表宽度**：每行需要填充的属性列数量（影响跨源查找量）。
    - **二维复合键**：当集合本身是交叉积（如团队×赛季）时，成员枚举变为网格，一个遗漏破坏一整行。
    - 据此定义三个层次：
      - **EASY**：窄一维列表，中位3列，无复合键。
      - **MEDIUM**：宽一维表（约5列）或窄二维网格（30%为二维）。
      - **HARD**：宽二维交叉积网格，中位7列，100%为二维复合键。
  - **正则化感知比较器**：兼顾日期精度统一、去除数字分隔符和单位、回退到归一化文本匹配，用于黄金构建的跨源验证和评分，避免格式差异导致误判。
  - **Web实源标签**：不同于LLM对页面结构的猜测（仅72%准确），代理实际打开最佳列表页验证所有请求列是否在同一页面，得到准确的“单页穷举”vs“跨源”标签。

## 6. 主要结论与发现

- **集合可恢复但行无法完整**：最强模型GPT-5.5的Item-F1达92.8，但Row-F1仅53.7，Table Success仅有19.3%。几乎所有模型都呈现巨大差距——即代理能找到所有成员，但会在一行中填错一个单元格导致整行错误。
- **难度梯度陡峭**：随着表宽度和二维复合键加入（EASY→MEDIUM→HARD），Row-F1和Table Success持续下降。但Item-F1基本稳定或略升（HARD的体育赛季网格系统性可枚举），因此难度增长并非源于检索失败，而是行完成失败。
- **韩语专门模型未缩小差距**：A.X-4.0（Row-F1 24.2）、Solar-Open-2-preview（24.4）、K-EXAONE-236B（17.5）位于开源权重的底层，远低于前沿模型（53.7）和最佳开源模型DeepSeek-V4-Pro（45.0）。韩语流利度本身不足以克服广度搜索的核心挑战。
- **更多搜索或更高花费无法闭合差距**：搜索最努力的模型（Qwen3.6平均每任务66次调用）得分最低；而顶级模型（GPT-5.5、Claude-Opus-4.8）搜索适中但仍领先。图7显示准确率随花费上升后迅速饱和，开源DeepSeek-V4-Pro以1/4的成本达到约85%的顶级Row-F1。
- **失败集中于开放文本单元格**：按单元格类型分析，自由文本值填充最差（49%），而日期（58%）、名称（56%）、数字（55%）、枚举（51%）更好。所以难点在于找到并规范化开放文本值，而非格式化。
- **组织集合大小不影响难度**：Row-F1在8-15、16-30、>30行的大小区间几乎持平，说明驱动困难的是宽度和二维结构，而非集合行数。
- **语义评判器仅小幅提升分数**：对严格字符串匹配判定错误的细胞使用LLM检查后，Row-F1提升0.8~4.9个百分点，且强大模型增益更大。说明强模型的剩余错误多为表面变体，而弱模型的错误是实质性错误（不同实体、不同值）。

## 7. 优点

- **填补空白**：首位系统构建韩语广度搜索基准，解决英语外广度评估的缺失。
- **自动化与可扩展**：合成-验证流水线使基准可自动构建和定期更新，降低人工标注成本。
- **细粒度难度控制**：通过表宽度和二维复合键两个结构旋钮定义三档难度，可揭示不同层次的能力瓶颈。
- **评分公平统一**：使用单一的正则化感知比较器进行黄金构建和模型评分，避免格式差异导致的误判；且不使用LLM作为主要评判，保证确定性和可复现性。
- **多重质量保障**：三阶段验证（不可记忆性、完整性、跨源验证）、去污染、人工抽查，保证数据质量。
- **实验结果深入**：按难度、单元格类型、搜索努力、花费、失败阶段等多维度分析，结论扎实，揭示了广度搜索中的关键瓶颈。
- **开源与隐私保护**：开源管线和评分器（MIT），评估数据按请求发放，防止污染。

## 8. 不足与局限

- **类别偏差**：HARD层（二维复合键）约67%为体育赛季任务，因为体育表格最易可靠构建，非体育二维数据（如选举）覆盖有限。
- **源标签的时变性**：跨源/单页标签反映标注时的页面结构，网页可能随时间变化，需要定期重新验证。
- **单一搜索后端和预算**：所有实验使用同一搜索工具和30轮迭代预算，未探索不同搜索引擎或预算对结果的影响。
- **人工验证有限**：只有母语者抽查，无正式标注者间一致性研究，基准被定位为“流水线验证”而非外部金标准。
- **韩语专门模型覆盖不全**：HyperCLOVA X、Kanana等模型未评估（标注为“待定”）。
- **未评估多模态能力**：基准仅依搜索文本，未涉及图像或结构化数据推断能力。
- **单个确定性评分器的局限**：尽管经正则化处理，严格字符串匹配仍会漏掉同义表达（如“서울, 대한민국” vs “서울”），论文额外通过语义评判器补充，但该评判器本身引入其他风险（自家模型评判自家）。
- **成本与可复现性**：专有模型API成本随模型和查询次数波动；自行托管模型的部署细节未说明，可能难以完全复现。
