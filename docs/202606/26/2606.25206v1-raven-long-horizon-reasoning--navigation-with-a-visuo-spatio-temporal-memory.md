## 摘要
长期机器人部署需要一种紧凑且可扩展的记忆系统，该系统能够保留细粒度的视觉语义，在空间和时间上锚定观测，并支持高效的存储与检索。在本文中，我们提出RAVEN，一种用于长视距机器人问答与导航的智能体记忆系统。RAVEN将视觉嵌入及其位姿和时间信息存储于向量数据库中，并在空间地图中定位检索，以回答查询并导航至目标。通过直接操作视觉嵌入，RAVEN避免了有损的图像到文本描述过程，并能在大规模场景下实现准确的语义、空间和时间检索。在多个模拟和真实世界视频问答基准测试中，RAVEN持续超越基于文本描述的记忆系统，并在长视距任务中以低10倍的检索成本达到前沿VLM的性能。最后，我们在Unitree Go1机器人上实例化RAVEN，用于自然语言目标到达的长视距导航任务，并在多个大型室内环境中成功部署。

## Abstract
Long-term robot deployment requires a compact and scalable memory that preserves fine-grained visual semantics, grounds observations in space and time, and enables efficient storage and retrieval. In this paper, we propose RAVEN, an agentic memory system for long-horizon robotic question answering and navigation. RAVEN stores visual embeddings with pose and time in a vector database, and grounds retrieval in a spatial map to answer queries and navigate to goals. By operating directly on visual embeddings, RAVEN avoids lossy image-to-text captioning and enables accurate semantic, spatial, and temporal retrieval at scale. Across several simulated and real-world video question-answering benchmarks, RAVEN consistently surpasses caption-based memory systems and matches frontier VLMs on long-horizon tasks at 10$\times$ lower retrieval cost. Finally, we instantiate RAVEN on a Unitree Go1 robot for the task of long-horizon navigation for natural language goal-reaching, and show successful deployment over several large indoor environments.

## 1. 核心问题与整体含义（研究动机和背景）

- **动机**：长期机器人部署需要一种紧凑且可扩展的记忆系统，能够保留细粒度视觉语义，并在空间和时间上锚定观测。现有方法存在明显局限：
  - 语义地图（如类别地图、3D点云）使用封闭词汇，无法处理开放词汇查询（如“蓝色帽子上的粉色蝴蝶结”）。
  - 原始视频流数据量过大，无法直接作为记忆。
  - 基于文本描述（caption）的方法引入了“描述瓶颈”**：图像→文本的有损压缩会丢失形状、空间布局、细微纹理等视觉细节，导致检索失败。例如，描述“肉”无法匹配查询“鸡大腿”。
- **整体含义**：探索能否超越基于文本的记忆，直接使用视觉嵌入作为记忆基本单元，从而在保持高信息密度的同时实现高效检索和推理。RAVEN验证了这种思路的有效性，为机器人长期部署提供了一个新的范式。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：利用多模态嵌入模型直接存储视觉嵌入（而非文本描述），与位姿（pose）、时间戳一起构成“视觉-空间-时间记忆”三元组，存储于向量数据库中。检索时，一个VLM智能体通过工具调用的方式迭代检索相关记忆，并完成推理与回答。

- **关键技术细节**：
  - **记忆构建（Memory Building）**：机器人通过主动探索采集RGB图像、深度图、位姿和时间戳。对每帧图像用预训练多模态编码器（如CLIP、SigLIP、Seed1.6-Embedding、QQMM-v2）提取视觉嵌入 \(z_i\)，与位姿 \(p_i\) 和时间 \(t_i\) 一同存入向量数据库（FAISS/Milvus），构成记忆条目 \(m_i = (p_i, t_i, z_i, o_i)\)。原始图像 \(o_i\) 也保留以备后续VLM查看。
  - **检索与推理（Querying & Execution）**：VLM智能体（如Gemini、GPT）作为一个有限状态机，维护工作记忆上下文。它可以通过四种工具检索记忆：
    - **文本检索**：将用户查询映射到嵌入空间，与视觉嵌入进行余弦相似度搜索。
    - **时间检索**：返回指定时间戳附近的连续帧。
    - **位置检索**：返回空间最近邻。
    - **图像检索**：支持用户以图像作为查询。
    每次检索返回Top-K个记忆条目（含图像、位姿、时间、相似度分数），追加到上下文中。VLM判断证据是否足够，若不足则继续调用工具，直至产生最终答案。输出可以是文本回答、空间坐标（用于导航）或时间引用。
  - **导航执行**：预测的目标坐标通过A*路径规划器驱动机器人到达。

- **算法流程**（文字说明）：
  1. **阶段I（探索与记忆构建）**：机器人执行前沿探索（frontier exploration），同时运行RGB-D SLAM建图，采集帧并编码嵌入，存储到向量数据库。
  2. **阶段II（查询与导航）**：给定自然语言查询，VLM智能体进入循环：解析查询→调用检索工具→获取相关记忆→推理→重复或输出。一旦确认为最终答案，输出目标坐标，机器人利用已建地图导航至该点。

6. **时序排序重要**：按时间顺序而非相似度排序能提升时间检索性能。

## 7. 优点

- **简洁而高效**：架构清晰，训练无关，即插即用，可直接融入现有机器人规划栈。
- **避免描述瓶颈**：直接利用视觉嵌入，保留更丰富的语义和空间信息。
- **灵活检索与推理**：提供多种检索工具（文本、时间、位置、图像），VLM可迭代推理，提高准确性。
- **可扩展性**：向量数据库支持亚线性时间检索，适合长运行时。
- **充分的实验验证**：涵盖模拟、真实、网络场景，多种模型组合，消融全面，结果可靠。
- **开源贡献**：公开代码和网站，促进社区研究。

## 8. 不足与局限

- **依赖预训练嵌入模型的质量**：检索能力受限于嵌入模型的对齐能力；若场景与训练分布差异大，性能可能下降。
- **静态记忆**：当前实现中记忆为离线构建的静态数据库，无法处理动态环境（物体移动、状态变化）。未来的工作需要更新和修剪机制。
- **计算资源要求**：虽然训练无关，但实时推理（特征提取、VLM调用）仍需要较强算力（如Jetson Orin NX），可能限制在低成本平台上的部署。
- **实验覆盖有限**：真实机器人实验仅在4个室内场景（实验室/办公室），缺乏复杂的户外、人群密集或长期（数天）的场景验证。
- **未完全开源所有模型**：部分嵌入模型（Seed1.6）为闭源，复现受限。
- **未见多机器人协作或实时记忆更新实验**，仅展示了单机器人离线记忆。
