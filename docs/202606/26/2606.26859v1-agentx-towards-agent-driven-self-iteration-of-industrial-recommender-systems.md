## 摘要
推荐算法的迭代正从一种工匠式的、依赖工程师的流程向工业化的研究循环转变，但这一转型仍受到结构性执行瓶颈的阻碍：从构思到上线的周期仍然依赖人类工程师生成假设、修改生产代码、启动A/B实验并归因在线结果。因此，创新规模与人员数量线性相关，而非随着证据、计算和积累的实验知识而复合增长。我们提出AgentX，一个生产部署的多智能体系统，从根本上重构了这一生产函数。AgentX作为一个自我演化的开发引擎运行：它以任何手动工作流都无法维持的规模和速度自主生成、实施、评估并从推荐实验中学习。该系统在闭环中编排四个紧密耦合的阶段。脑力激荡智能体综合历史实验、系统架构、数据分析和外部研究的证据，生成排序后的可执行提案。开发智能体通过基于仓库的生成和多维可靠性验证，将每个提案转化为生产就绪代码。评估智能体进行安全的在线部署，并带有护栏否决的A/B判断，将成功和失败都转化为结构化的知识资产。一个训练演化层（SGPO）随后将执行轨迹提炼为语义梯度更新，不断自我优化这些智能体本身——使系统不仅自动化，而且自我改进。

## Abstract
Recommendation algorithm iteration is moving from an artisanal, engineer-bound process toward an industrialized research loop, but this transition remains blocked by a structural execution bottleneck: the idea-to-launch cycle still depends on human engineers to generate hypotheses, modify production code, launch A/B experiments, and attribute online results. Innovation therefore scales linearly with headcount rather than compounding with evidence, compute, and accumulated experimental knowledge. We present AgentX, a production-deployed multi-agent system that fundamentally restructures this production function. AgentX operates as a self-evolving development engine: it autonomously generates, implements, evaluates, and learns from recommendation experiments at a scale and pace that no manual workflow can sustain.   The system orchestrates four tightly coupled stages in a closed loop. A Brainstorm Agent synthesizes evidence from historical experiments, system architecture, data analysis, and external research into ranked, executable proposals. A Developing Agent translates each proposal into production-ready code through repository-grounded generation and multi-dimensional reliability verification. An Evaluation Agent conducts safe online rollout with guardrail-vetoed A/B judgment, converting both successes and failures into structured knowledge assets. A Harness Evolution layer (SGPO) then distills execution trajectories into semantic-gradient updates that continuously sharpen the agents themselves -- making the system not merely automated, but self-improving.

## 2. 方法论：核心思想、关键技术细节、算法流程（文字说明）

- **核心思想**：将推荐算法迭代的生命周期分解为**四个紧耦合阶段**，每个阶段由专门智能体负责，形成闭环；额外增加**Harness Evolution 层**，利用执行轨迹持续优化智能体自身的提示、规则和工作流，实现系统的自我改进。
- **关键技术细节**：
  - **Brainstorm Agent（脑力激荡智能体）**：
    - 将模糊的优化意图转化为有界、可执行的实验提案。
    - 采用**证据加权生成**：结合四个知识源（实验知识库、系统知识库、数据分析、模型研究），每个候选方案获得加权证据分数，与目标对齐度、业务有效性、实现可行性等综合评分。
    - 提案分为三类：待实施、先探查、远期想法，避免过早投入。
    - 包含验证与材料化步骤：通过人类审核和自动化边界检查后生成结构化提案向下游传递。
  - **Developing Agent（开发智能体）**：
    - **双主线**：online strategy track（生产代码变更）和 offline model track（架构实验）。
    - 在线策略开发：**仓库导向代码生成**——通过项目知识库和案例工具箱（模式查询、DSL检查器、C++语法检查）将特征属性、API调用等转化为验证后的事实，而不是依赖LLM猜测；**以验证为导向的实现循环**——准确性循环与干运行管道，质量评分涵盖8个维度（如属性幻觉、合规循环迭代次数、人工干预次数）。
    - 模型开发：遵循 policy → (code ↔ verify) ∥ experts → exec → final_review 结构。强调**可证伪归因**——每个实验需声明因果机制和预期观测值，代码验证、专家投票（超2/3多数）、确定性AUC提取、因果链逐环核实。只有所有环节验证通过才记录为成功。失败分类策略：确定性错误直接放弃，临时故障重试一次。
  - **Evaluation Agent（评估智能体）**：
    - 负责安全部署（OpenAPI自动化、流量分配、灰度窗口）、在线A/B判断（使用CUPED/DiD等方差缩减）、**护栏否决**（业务范围定制的复合经济交换指标，阈值作为注意力信号而非绝对阻断）。
    - **负结果资产化**：每次失败实验记录根因、阶段、业务域、策略杠杆，供未来避免重复错误。
  - **Harness Evolution（训练演化层）**：
    - **SGPO-I（基于轨迹的语义梯度提示优化）**：从积累的轨迹池中采样→提取语义梯度（自然语言诊断）→生成候选提示更新→通过配对回放（相同任务上新旧对比）判定是否采纳（仅当 ∆score > ε 且通过安全检查）。
    - **SGPO-II（基于代码合并请求的演化）**：将历史 MR 转化为仅含需求的回放任务，评估五个维度（语义正确性、需求覆盖、文件覆盖、默认安全、代码风格），更新开发智能体的规则和模式记忆。
    - **模型研究管线演化**：三阶段自循环——再现（每个论文）、模块消融（隔离验证单个模块贡献）、跨论文组合（顶级模块嫁接+挑战者轮次），通过记忆引导剪枝（论文级、模块级、饱和度检测）和可伪造归因把关，将实验结论积累为可重用的知识（反模式库、剧本）。
- **算法流程（文字说明）**：
  - 外部输入（如业务目标）→ Brainstorm Agent 生成候选提案列表 → 验证与材料化 → Developing Agent (在线或模型路线) 生成验证过的代码/实验报告 → Evaluation Agent 上线A/B测试并给出结构化裁决（KEEP/EXTEND/DISCARD）→ 成功或失败的轨迹被记录 → Harness Evolution 层利用 SGPO 分析失败模式，生成局部智能体更新 → 配对回放验证后部署新版本智能体 → 进入下一轮循环。
