## 摘要
现有嵌入模型本质上是静态的：它们孤立地对文本片段进行编码，忽略其周围的上下文和时间顺序。本文提出EvoEmbedding，一种为检索生成可演化表示的新型嵌入模型。它专为长上下文场景设计，其中信息是动态的、顺序性的，并且需要持续的状态跟踪。我们的设计简单：EvoEmbedding在顺序处理输入时维护一个持续更新的潜在记忆，并将其与原始内容一起用于联合生成可演化嵌入。因此，对于相同的查询，我们的模型会调整其表示以基于演化的上下文检索不同的目标，超越了静态语义搜索。为了使模型具备这种能力，我们构建了EvoTrain-180K，一个用于潜在记忆和检索联合优化的多样化数据集。此外，我们引入了记忆队列以防止循环编码过程中的表示崩溃，以及处理显著长度变化并加速训练3.8倍的段分批技术。大量实验表明，我们的模型不仅在一系列长上下文检索基准测试中优于大规模专用模型（例如Qwen3-Embedding-8B和KaLM-Embedding-Gemma3-12B），还能很好地泛化到下游任务（例如个性化），其上下文长度比训练窗口长10倍。值得注意的是，EvoEmbedding无缝集成到主体工作流中以提升性能。例如，配备我们模型的简单RAG管道超越了专用的主体记忆系统。项目页面：https://clare-nie.github.io/EvoEmbedding/。

## Abstract
Existing embedding models are inherently static: they encode text segments in isolation, ignoring their surrounding context and temporal order. This paper introduces EvoEmbedding, a novel embedding model that generates evolvable representations for retrieval. It is tailored for long-context scenarios, where information is dynamic, sequential, and requires continuous state tracking. Our design is simple: EvoEmbedding maintains a continuously updated latent memory as it sequentially processes inputs, and uses it alongside the raw content to jointly generate evolvable embeddings. Consequently, for the same query, our model adapts its representation to retrieve distinct targets based on the evolving context, going beyond static semantic search. To equip the model with this capability, we construct EvoTrain-180K, a diverse dataset for the joint optimization of latent memory and retrieval. Furthermore, we introduce a memory queue to prevent representation collapse during recurrent encoding, alongside segment-batching techniques that tackle significant length variance and accelerate training by 3.8$\times$. Extensive experiments show that our model not only outperforms larger-scale specialists (e.g., Qwen3-Embedding-8B and KaLM-Embedding-Gemma3-12B) across a range of long-context retrieval benchmarks, but also generalizes well to downstream tasks (e.g., personalization) with contexts 10$\times$ longer than its training window. Notably, EvoEmbedding seamlessly integrates into agentic workflows to boost performance. For instance, a naive RAG pipeline equipped with our model surpasses dedicated agentic memory systems. - **背景**：检索增强生成（RAG）和智能体系统依赖可靠的长时记忆检索。但传统嵌入模型短于复杂上下文理解（如指代消解、时间推理），且现有优化方案（查询重写、重排序、构建结构化记忆等）带来额外计算与延迟。
- **整体含义**：本文提出**EvoEmbedding**，通过维护一个持续更新的“潜在记忆”来生成随上下文演化的嵌入表示，使同一查询在不同上下文中能检索到不同目标，从而实现动态、时序感知的检索，从根本上改进长上下文检索范式。

## 2. 论文提出的方法论

### 核心思想
- 顺序处理输入段（segment），同时维护一个固定容量的FIFO潜在记忆队列（Latent Memory Queue）。
- 对于当前段 \(x_t\) 和历史记忆 \(M_{t-1}\)，并行执行：
  - **记忆演化**：生成新的潜在记忆标记 \(\tilde{M}_t\)，更新队列。
  - **表示生成**：结合记忆与当前段，输出段嵌入 \(v_t\)。

### 关键技术细节
1. **潜在记忆队列**：容量 \(C = L \times K\)，存储最近 \(L\) 步生成的 \(K\) 个记忆标记。防止循环编码导致的表示崩溃，且限制计算复杂度。
2. **段分批（Segment-Batching）**：训练和推理时将 \(k\) 个连续段拼接后并行前向计算，总长度不超过阈值（如2048 token），加速训练约3.8倍。
3. **多LoRA设计**：对记忆演化和表示生成使用两个独立的LoRA适配器，冻结基础LLM，避免灾难性遗忘，并支持灵活切换。
4. **联合优化目标**：
   - 记忆生成损失 \(L_{mem}\)：使用交叉熵，以最终记忆和查询为条件预测目标答案（冻结骨干，通过逆向传播优化记忆）。
   - 对比损失 \(L_{con}\)：长度加权多正例对比损失，平衡不同长度样本的难度，使查询表示与相关段靠近。
5. **数据集EvoTrain-180K**：三阶段自动化流水线构建：
   - 原始语境构建（网页文本、合成对话、记忆）。
   - 动态QA生成（40+模板，覆盖语义匹配与推理）。
   - 检索标注与验证（用Gemini-3.1-Pro标注正负索引，过滤幻觉和上下文无关样本）。共184,137样本，最大12K tokens。

## 7. 优点

- **方法创新**：将潜在记忆引入嵌入生成，使表示随上下文演化，突破静态语义搜索局限。
- **训练高效**：段分批加速3.8倍，记忆队列防止表示崩溃，无需课程学习。
- **多LoRA解耦**：统一记忆、检索、生成于同一通用LLM，避免灾难性遗忘，灵活切换。
- **数据集设计**：EvoTrain-180K自动化合成，质量严格筛选（过滤幻觉、保证上下文依赖），支持复杂推理。
- **强兼容性**：可作为即插即用模块嵌入现有智能体系统，提升性能且不增加生成开销。
- **可解释性**：通过时间敏感性分析（相似度曲线、t-SNE）直观展示了模型对时序信息的捕获能力。

## 8. 不足与局限

- **计算资源与数据规模**：受限于训练数据量（180K）和计算资源，在极端域外场景性能可能下降。
- **不支持多模态**：当前仅支持文本，扩展至视觉/音频需更大记忆容量和维度对齐。
- **推理速度**：时序处理导致编码速度慢于并行静态模型（约22秒 vs 3-10秒），可能不适用于极低延迟场景。
- **记忆容量限制**：默认队列C=512 tokens，对超长上下文（>1M tokens）可能不足，需进一步扩展机制。
- **训练长度上限**：最大12K tokens，虽泛化到128K，但更极端长度下的表现未充分验证。
- **实验覆盖**：主要聚焦对话、文档检索；对非英语语言或异构数据（如代码、表格）未测试。
