## 摘要
大型语言模型（LLM）代理的长期记忆系统通常试图“记住一切”，通过均匀提取记忆来保留尽可能多的事实。然而，在生产环境中，推理成本和有限的上下文预算使得这种方法难以维持：除了将原始对话整合为记忆外，代理还必须施加“写入控制”，只高效地保留每个用户实际关心的信息。否则，长周期的个性化交互会遭遇“记忆膨胀”，即无关的琐事排挤有用信息，并逐渐侵蚀问答（QA）准确性。我们认为，值得记住的内容取决于角色，并提出AdaMem（自适应记忆）——一种从反馈中学习为每个用户记住什么的方法。AdaMem维护一个结构化的、角色特定的记忆策略，并通过轻量级的补丁式自我反思步骤（含失败回滚）根据每周的QA反馈来优化该策略。为研究这一场景，我们构建了AdaMem-Bench基准，该基准模拟了数周的交互，并包含逐周的QA。在两种提取模型和两种反馈模式下，与均匀的Mem0基线相比，AdaMem将QA准确率提升了高达+9.0%，同时将记忆体积缩小了9%。

## Abstract
Long-term memory systems for Large Language Model (LLM) agents typically try to \emph{remember everything}, extracting memories uniformly to retain as many facts as possible. In production, however, inference cost and finite context budgets make this untenable: beyond consolidating raw dialogue into memory, an agent must exert \emph{write control}, efficiently keeping only the information each user actually cares about. Otherwise, long-horizon personalized interactions suffer \emph{memory bloat}, where irrelevant trivia crowds out useful information and steadily erodes question-answering (QA) accuracy. We argue that what is worth remembering is role-dependent, and propose \textbf{AdaMem} (Adaptive Memory), a method that \emph{learns what to remember} for each user from feedback. AdaMem maintains a structured, role-specific Memory Policy and refines it from weekly QA feedback through a lightweight, patch-style self-reflection step with failure rollback. To study this setting, we build \textbf{AdaMem-Bench}, a benchmark that simulates weeks of interaction with week-by-week QA. Across two extraction models and two feedback modes, AdaMem improves QA accuracy by up to \textbf{+9.0\%} over the uniform Mem0 baseline while shrinking memory volume by \textbf{9\%}.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有 LLM 代理的长期记忆系统（如 Mem0、MemoryBank）试图“记住一切”，采用统一提取规则保留所有事实。但在实际部署中，受推理成本和有限上下文预算的约束，这种策略导致“记忆膨胀”（memory bloat）：无关琐事（如问候、微小日程）挤占有用信息，随着交互周期延长（数周至数月），问答（QA）准确率逐渐下降。
- **关键洞察**：值得记住的内容是**角色依赖**且**主观**的——例如，与管理者的对话应跟踪业务决策，而与健身伙伴的聊天则应关注训练计划。因此，记忆提取应由与用户偏好对齐的、可动态更新的“记忆策略”（Memory Policy）引导。
- **整体意义**：将长周期记忆问题从“尽可能多地存储”重新定义为“学习该存储什么”的**写控制问题**，为工业界在成本约束下实现高效个性化交互提供了新范式。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：AdaMem（Adaptive Memory）通过闭环的每周循环，从每周 QA 反馈中学习角色特定的记忆策略，并应用轻量级的**补丁式自反思**（patch-style self-reflection）与**失败回滚**来更新策略。

- **关键技术细节**：
  - **记忆策略结构**：`P = {C₁: T₁, ..., Cₙ: Tₙ}`，以 JSON 对象形式存储，将每个角色（字符）映射到其自然语言的提取偏好（例如，“只提取最终业务决策和截止日期，忽略日常闲聊”）。
  - **每周循环流程**（算法 1）：
    1. **日常提取**：根据当前策略 `P_t` 提取当天的偏好记忆，并在提取时进行**时间锚定**（Time Anchoring），将所有相对时间表达（如“明天”）解析为绝对日期后存储。
    2. **周评价**：通过每周 QA 测试代理，获取反馈（正确/错误）。
    3. **策略反思与更新**：对错误答案 `W_t` 诊断错误类型：
       - **回忆失败**（Missing Memory）：信息在对话中但未被提取（策略过于严格）。
       - **精度失败**（Noisy Memory）：无关记忆误导回答（策略过于宽松）。
       生成策略补丁 `ΔP_t`，得到候选新策略 `P'_{t+1} = P_t ⊕ ΔP_t`。
    4. **失败回滚**：在历史验证集上测试候选策略，若效果退化则拒绝更新，回滚至原策略。
    5. **结果写回**：用更新后的记忆重新生成 QA 答案并写入。

- **RQ4（策略学习瓶颈）**：
