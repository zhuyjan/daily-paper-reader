## 摘要
Agentic检索增强生成（RAG）系统结合了迭代推理循环、查询分解和自适应检索，以解决多跳问答问题。然而，每个组件的贡献仍然知之甚少，尤其是在仅使用本地语言模型的资源受限设置下。许多Agentic设计增加了自适应检索路由和更深的检索循环，假设增加的复杂性有助于提升性能。为了验证这一点，我们对一个完整的Agentic RAG流水线进行了受控消融研究，在HotpotQA干扰项开发集的5000个问题上使用本地7B参数模型（Qwen2.5-7B-Instruct）进行评估。我们的完整流水线实现了EM=53.2%和F1=61.6%，而单次稠密检索基线为EM=43.1%和F1=54.0%。在八种消融条件下，我们发现：（1）通过倒数秩融合的固定混合检索始终优于基于规则的自适应路由（+1.8 EM，+1.9 F1），因为路由启发式方法会因几乎出现在所有多跳子问题中的命名实体而过度路由到BM25；（2）对分解后的子问题进行两次检索迭代即可捕获五次迭代95%的收益，更深循环无显著益处；（3）查询分解和交叉编码器重排序各自贡献了统计显著但较小的增益（分别为p<0.01和p<0.001）。综上所述，在固定的本地模型预算下，更简单且固定的选择结果与自适应版本相比具有竞争力甚至更优：大部分收益来自短检索循环，而非自适应路由或多次迭代。我们未使用任何专有API或大规模计算。

## Abstract
Agentic retrieval-augmented generation (RAG) systems combine iterative reasoning loops, query decomposition, and adaptive retrieval to tackle multi-hop question answering. However, the contribution of each component remains poorly understood, particularly under resource-constrained settings using only local language models. Many agentic designs add adaptive retrieval routing and deeper retrieval loops on the assumption that the added complexity helps. To test whether it does, we run a controlled ablation study of a full agentic RAG pipeline evaluated on 5,000 questions from the HotpotQA distractor development set using a local 7B parameter model (Qwen2.5-7B-Instruct). Our full pipeline achieves EM=53.2% and F1=61.6%, compared to a single-pass dense-retrieval baseline of EM=43.1% and F1=54.0%. Across eight ablation conditions, we find that: (1) fixed hybrid retrieval via reciprocal rank fusion consistently outperforms rule-based adaptive routing (+1.8 EM, +1.9 F1), as the routing heuristic over-routes to BM25 by firing on named entities present in nearly all multi-hop sub-questions; (2) two retrieval iterations over the decomposed sub-questions capture 95% of the gains of five, with no meaningful benefit from deeper loops; and (3) query decomposition and cross-encoder reranking each contribute statistically significant but smaller gains (p<0.01 and p<0.001 respectively). Taken together, on a fixed local-model budget, the simpler and fixed choices turn out to be competitive with or better than their adaptive versions: most of the gain comes from running a short retrieval loop, not from adaptive routing or from many iterations. We use no proprietary APIs or large-scale compute.

## 2. 方法论

### 核心思想
通过在HotpotQA上对完整Agentic RAG流水线进行**八种受控消融实验**，量化每个组件（检索深度、分解、重排序、策略选择）对性能的独立贡献，并利用配对统计检验确保结论可靠。

### 关键技术细节
- **流水线结构**（规划-执行模式）：
  1. **查询分解**：LLM将原始问题拆分为一组可独立回答的子问题（一次性生成）。  
  2. **迭代检索循环**（默认 \(T=5\)）：按依赖顺序逐个子问题检索，每次循环：选择检索策略 → 检索 → 重排序 → 追加到共享上下文。**LLM不在步骤间调用**，仅最后调用一次合成。  
  3. **自适应检索**：基于规则的启发式路由：含命名实体/日期/数字→BM25；短或概念性→稠密；其他→混合。  
     - 稠密：BGE-small-en-v1.5 + Qdrant向量索引  
     - 稀疏：BM25  
     - 混合：倒数秩融合（RRF），等权合并稠密和稀疏排名  
  4. **重排序**：交叉编码器（ms-marco-MiniLM-L-6-v2）从top-20选出top-5。  
  5. **合成**：累积证据送LLM生成简短事实答案。

- **消融变体**（共8种，均保持其他默认）：
  - `steps-1/2/3`：检索深度1/2/3步（默认5步）  
  - `no-decomp`：禁用分解，原始问题直接检索  
  - `no-reranker`：禁用重排序，直接取top-5  
  - `dense-only / sparse-only / hybrid-only`：固定策略替代自适应路由

## 6. 主要结论与发现

1. **固定混合检索优于自适应路由**（+1.8 EM, +1.9 F1, p<0.001）  
   - 原因：路由启发式对命名实体过度触发（79.2%查询被路由到BM25），丢失互补的稠密信号。  
   - 表面上BM25胜率更高，但这是**选择偏差**（简单实体查询被分给BM25）。

2. **两次检索迭代即可获得绝大部分收益**  
   - 两步迭代（steps-2）捕获五步95%的增益（p<0.001 vs 一步）；三步后完全饱和。  
   - 主要驱动是“执行循环本身”，而非循环次数。

3. **查询分解和重排序贡献较小但显著**  
   - 分解+1.4 EM (p=0.004)；重排序+1.7 EM (p<0.001)。  
   - 重排序延迟可忽略；分解需额外LLM调用（延迟翻倍），是首个可牺牲的组件。

4. **个体策略（dense/sparse/hybrid）在迭代循环内表现接近**  
   - 差异≤0.4 EM，说明循环本身补偿了策略缺陷，策略选择比循环深度次要。

5. **最佳实践**：固定混合RRF + 2-3步检索循环 + 无条件保留重排序 + 按需保留分解（优先精度则保留，优先延迟则去掉）。

## 8. 不足与局限

- **数据集单一**：仅HotpotQA（两跳推理），更长/数值推理任务结果可能不同。  
- **模型局限**：仅Qwen2.5-7B，未验证其他架构或参数规模（如13B、Llama系列）。  
- **缺少多次运行**：单次greedy解码，无法评估生成随机性对结果的影响。  
- **消融耦合**：`no-decomp`同时丢失分解引导和多步检索（原因为子问题缺失使循环退化为单步），导致效果混淆（论文已指出）。  
- **评估基准偏差**：论文比较的“purpose-built系统”（如HGN）在完整测试集上更高，但本研究仅5000子集，直接对比不完整。  
- **硬件依赖**：延迟数据特定于Ollama+RTX A6000，不可推广。  
- **未探索动态停止条件**：仅固定步数，未测试基于置信度的早停。  
- **路由启发式简单**：仅基于词法特征（实体/日期），未测试更精细的规则或学习型路由。
