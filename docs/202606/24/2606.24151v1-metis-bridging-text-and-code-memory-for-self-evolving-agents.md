## 摘要
自演化智能体通过从过往执行中提炼经验并在未来任务中复用，随时间不断改进。现有系统将这些经验表示为注入到智能体上下文中的自然语言文本，或作为可调用工具暴露的代码。然而，这两种表示之间的选择通常在设计时而非根据经验本身的特征确定，导致对其间的权衡理解不足。我们首次在完全相同的经验集上开展了分离文本记忆与代码记忆的受控研究。结果表明，两种形式在构建成本、执行效率和可迁移性方面展现出互补的权衡，使得单一表示不足以应对。基于这些发现，我们提出了Metis，一种构建在分层双表示记忆上的自演化智能体系统。Metis将文本经验组织为执行计划、环境事实和常见陷阱，并有选择地将重复出现的计划固化为经过验证的可调用工具。这种设计结合了文本记忆的广泛适用性与代码记忆的执行效率，同时仅在重复复用合理时才产生工具生成成本。我们在面向交互式智能体的挑战性基准AppWorld上评估了Metis。结果显示，与ReAct相比，Metis将任务准确率提升了高达20.6%，同时将执行成本降低了高达22.8%。与代表性自演化智能体系统相比，Metis在准确性、执行效率和记忆构建成本之间始终实现了更好的平衡。

## Abstract
Self-evolving agents improve over time by distilling experience from past executions and reusing it in future tasks. Existing systems represent such experience either as natural-language text injected into the agent context or as code exposed as callable tools. However, the choice between these representations is typically made at design time rather than derived from the characteristics of the experience itself, leaving the trade-offs between them poorly understood. We present the first controlled study that isolates text memory and code memory over an identical set of experiences. Our results show that the two forms exhibit complementary trade-offs in construction cost, execution efficiency, and transferability, such that neither representation alone is sufficient. Guided by these findings, we propose Metis, a self-evolving agent system built on a hierarchical dual-representation memory. Metis organizes textual experience into execution plans, environment facts, and common pitfalls, and selectively crystallizes recurring plans into validated callable tools. This design combines the broad applicability of text memory with the execution efficiency of code memory while incurring tool-generation cost only when justified by repeated reuse. We evaluate Metis on AppWorld, a challenging benchmark for interactive agents. The results show that Metis improves task accuracy by up to 20.6% over ReAct while reducing execution cost by up to 22.8%. Compared with representative self-evolving agent systems, Metis consistently achieves a better balance between accuracy, execution efficiency, and memory-construction cost.

## 1. 论文的核心问题与整体含义（研究动机和背景）

现有的自演化智能体（self-evolving agents）通过从过往执行轨迹中提炼经验，并在未来任务中复用来提升性能。这些经验通常存储为两种形式：

- **代码记忆**：将过往流程封装为可调用的工具或MCP服务，智能体直接调用而无需重新推导过程。

然而，两种表示形式的选择通常是在系统设计时预先固定，而非根据经验本身的特征动态决定。这导致两方面的不足：
- 缺乏对两种表示形式各自优势与局限的系统性理解。
- 单一表示形式无法同时满足低成本构建、高效率执行和高泛化迁移的需求。

论文的**核心问题**是：如何根据经验的特性选择或组合表示形式，从而在构建成本、执行效率和可迁移性之间取得最佳平衡？

整体含义：该工作首次通过受控实验分离文本记忆和代码记忆的影响，揭示了它们在三个维度上的互补权衡，并据此设计了Metis——一个分层双表示记忆系统，能够自适应地决定经验的表示形式，从而实现更优的智能体持续进化。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 核心思想
Metis构建了一个分层双表示记忆体系：
- **底层**：结构化文本记忆（低承诺、高泛化），包含三种类型：执行计划（execution plans）、环境事实（environment facts）、常见陷阱（common pitfalls）。
- **上层**：代码记忆（高执行效率），仅在执行计划被**重复选择**（满足复用阈值）时，才将其“提升”（promote）为经过验证的可调用工具。

关键设计原则：文本记忆是默认的低成本载体；代码记忆只在模式稳定且可充分摊销构建成本时才创建，避免过拟合和浪费。

### 关键技术细节
1. **差异化的文本反射（Differentiated Text Reflection）**：
   - 反射器（reflector）是一个带推理循环的智能体，可以访问执行环境，对成功/失败的轨迹进行蒸馏。
   - 提取三种经验：执行计划（可重用子流程）、环境事实（环境无关属性）、常见陷阱（失败模式，表示为(触发条件, 错误, 后果)）。
   - 内存维护：通过合并、更新（无效化旧条目并追加新条目）来保持一致性。

2. **模式感知的代码生成（Pattern-Aware Code Generation）**：
   - 每个执行计划携带书签字段：候选缓冲区B(p)（记录哪些任务选择了该计划）、相关任务历史R(p)、工具依赖D(p)。
   - 当 |B(p)| ≥ 阈值θ时，触发代码化。代码化器接收计划体、候选查询、相关查询以及当前工具库，但**故意排除原始轨迹**，以避免过拟合。
   - 生成的小型可组合工具必须至少有两个结构不同的查询作为支持，且任务变化部分（如实体、时间范围等）必须暴露为参数。
   - 工具通过依赖闭合、编译检查后方可入库。

3. **内存管理器（Memory Manager）**：
   - 联合检索文本条目和代码工具：先嵌入相似度top-k初筛，再由轻量级LLM进行联合选择。
   - 允许文本计划自动拉取其关联工具，工具也能匹配对应的文本指导。

4. **反射支架（Reflection Harness）**：
   - 提供渐进式轨迹展示（保留头部和尾部，可请求展开中间步骤）。
   - 在工具准入时执行依赖闭合和编译检查，确保新工具在运行时完整可执行。
   - 反射器可以通过支架检查现有工具实现。

### 算法流程（文字说明）
整体执行-反射-复用循环：
1. 对于新任务q_i，内存管理器从当前记忆M中检索相关文本条目和工具，注入到执行器上下文中。
2. 执行器与环境交互完成任务，产生轨迹τ_i。
3. （可选）若选中的计划p满足|B(p)|≥θ，进入代码生成：通过查询集和计划体生成工具，验证后加入代码库M_code。
4. 差异化文本反射器从轨迹中蒸馏经验，更新文本记忆（新建/合并/更新条目）。
5. 更新后的记忆用于后续任务。

1. **文本记忆 vs. 代码记忆的互补性**：文本：便宜构建、可靠迁移，但执行效率低；代码：执行高效，但构建昂贵且迁移脆弱。
