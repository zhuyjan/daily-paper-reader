## 摘要
大型语言模型（LLM）智能体依赖长期记忆来支持超越有限上下文窗口的扩展交互和个性化辅助。现有的记忆智能体通过生成的写入、修改和删除操作主动更新外部记忆，但这些更新可能会遗漏重要信息、破坏现有记忆或引入不受支持的幻觉内容。一旦存储，此类错误会成为持续的系统状态故障，影响未来的推理和生成。在本文中，我们提出TrustMem，一个旨在提高记忆巩固可信度的框架。TrustMem依靠记忆转换验证器来评估记忆更新的转换过程，包括覆盖性、保留性和忠实性。它进一步在相同记忆状态下的候选更新之间构建偏好对，使偏好引导的强化学习能够直接优化记忆更新行为。大量实验表明，TrustMem提高了记忆的实用性和可靠性：它在MemoryAgentBench、HaluMem和Mem-alpha验证集上取得了最先进的结果，在HaluMem记忆提取上提高了12.14个F1点，并且与每种错误类型的最强基线相比，将转换级别的遗漏、破坏和幻觉分别减少了40.1%、79.1%和50.0%。

## Abstract
Large language model (LLM) agents rely on long-term memory to support extended interactions and personalized assistance beyond finite context windows. Existing memory agents actively update external memory through generated write, revise, and delete operations, but these updates may omit important information, corrupt existing memory, or introduce unsupported hallucinated content. Once stored, such errors become persistent system-state failures that can affect future reasoning and generation. In this paper, we propose TrustMem, a framework designed to improve the trustworthiness of memory consolidation. TrustMem relies on a Memory Transition Verifier to evaluate the transition process of memory updates in terms of coverage, preservation, and faithfulness. It further constructs preference pairs among candidate updates under the same memory state, enabling preference-guided reinforcement learning to directly optimize memory updating behaviors. Extensive experiments demonstrate that TrustMem improves both memory utility and reliability: it achieves state-of-the-art results across MemoryAgentBench, HaluMem, and the Mem-alpha validation set, improves HaluMem memory extraction by 12.14 F1 points, and reduces transition-level omission, corruption, and hallucination by 40.1\%, 79.1\%, and 50.0\%, respectively, compared with the strongest baseline for each error type.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **问题形式化**：将记忆巩固建模为持续的记忆更新过程。输入流 `C = {c1, c2, ..., cT}` 被分块处理，每个时间步 `t`，智能体观察当前块 `ct` 和历史记忆状态 `Mt-1`，生成结构化的记忆编辑动作（WRITE/REVISE/PRUNE），执行后得到新状态 `Mt`。每个更新定义为一个局部转换 `zt = (ct, Mt-1, at, Mt)`。
- **关键创新**：引入**记忆转换验证器（Memory Transition Verifier, MTV）**和**转换排序GRPO（Transition-Ranked GRPO）**，在转换层级提供细粒度监督。

### 关键技术细节
1. **记忆转换验证器（MTV）**：
   - 用冻结的LLM对每个局部转换 `zt` 沿三个维度评估：
     - **覆盖率（Coverage）**：当前块的重要信息是否被保留。
     - **保留性（Preservation）**：先前有效记忆是否未被不当删除或扭曲。
     - **忠实性（Faithfulness）**：新增或修改的内容是否得到当前块或先前记忆的支持。
   - 输出标量信任度分数 `vt ∈ [0,1]`，作为转换级验证奖励。

2. **多粒度奖励设计**：
   - **样本级奖励**：
     - 任务奖励 `R_task`：基于下游任务的准确率（F1或LLM评判）。
     - 效率奖励 `R_eff`：惩罚记忆膨胀，鼓励紧凑记忆。
   - **转换级奖励**：
     - 动作可执行性 `R_act`：动作是否能被记忆执行器解析执行。
     - 内容类型 `R_type`：确保修改内容语义具体且非空。
     - 验证器奖励 `R_ver`：来自MTV的信任度分数。
   - 总奖励 `R_i,t = R_task + λ_eff R_eff + λ_act R_act,t + λ_type R_type,t + λ_ver R_ver,t`。

3. **转换排序GRPO（Transition-Ranked GRPO）**：
   - 基于GRPO（Group Relative Policy Optimization）框架，对每个训练样本采样G个rollouts。
   - 对每个chunk步骤t，用组归一化计算优势 `Â`。
   - **偏好对构建**：在同一chunk和先前记忆状态下，从G个候选转换中选取验证器分数最高和最低的作为偏好对 `z+, z-`（需要分数差 > 边际δ）。
   - **排序损失**：使用DPO风格的损失函数，对比偏好的动作序列和拒绝的动作序列的长度归一化对数概率差，并加上KL正则化。
   - **联合目标**：`L = - J_GRPO + β_KL * L_KL + λ_rank * L_rank`。

4. **算法流程**（文字说明）：
   - 输入流分块 → 智能体基于当前块和先前记忆生成动作序列 → 记忆执行器更新状态 → MTV评估转换 → 多粒度奖励计算 → GRPO组内优势估计 → 基于验证器分数构建偏好对 → DPO风格排序损失 → 联合优化策略。

## 8. 不足与局限

1. **模态局限**：论文明确承认当前实验仅针对文本记忆。现实应用中的用户输入和记忆证据常常是多模态的（图像、视频、文档等），当前框架未涉及多模态记忆整合。
2. **验证器依赖**：MTV使用冻结的LLM作为评判器，其评估质量可能受限于LLM自身偏差，且需要额外调用成本。
3. **训练数据规模**：训练集仅562个实例，虽然展示了良好泛化，但更大规模训练可能带来进一步改进。
4. **计算开销**：需要32块H100训练2天，资源需求较高；推理时每次更新调用MTV也会增加计算成本。
5. **动作空间限制**：仅定义了WRITE/REVISE/PRUNE三种操作，可能无法覆盖更复杂的记忆编辑需求（如合并、重构）。
6. **下游任务依赖**：任务奖励（R_task）需要可访问的基准答案，限制了在没有标注反馈的真实环境中的适用性。
7. **安全评估范围**：虽然评估了三种错误类型，但未涉及对抗性攻击、记忆注入攻击等安全风险。
8. **记忆类型**：主要处理事实性知识记忆，对于个人偏好、情感记忆等主观信息的处理未深入探讨。
