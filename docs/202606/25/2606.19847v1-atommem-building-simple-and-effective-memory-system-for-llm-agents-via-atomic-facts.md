## 摘要
大型语言模型展现了强大的推理和生成能力，但其固定的上下文窗口限制了跨会话交互中的长期信息积累与重用。现有的记忆增强系统通常以粗糙且不稳定的方式构建记忆，依赖于低效的记忆表示或不稳定的无约束更新。为了解决这些挑战，我们提出了AtomMem，一种面向价值密集存储和稳定记忆演化的长期记忆系统。AtomMem引入了事实执行器，从长形式交互中选择性提取高价值原子事实，作为高效的记忆表示。随后，AtomMem将这些事实组织成层次化的事件结构与时序档案，捕捉连贯的情节上下文并随时间跟踪动态演化的用户属性。在检索过程中，系统激活关联记忆图以连接碎片化记忆。在LoCoMo基准上的实验证实，AtomMem在各种推理任务中达到了最先进的性能，为部署智能个性化智能体提供了可扩展且经济可行的解决方案。

## Abstract
Large language models (LLMs) demonstrate strong reasoning and generation abilities, but their fixed context windows limit long-term information accumulation and reuse across multi-session interactions. Existing memory-augmented systems often construct memory in a coarse and unstable manner, relying on inefficient memory representations or unstable unconstrained updates. To address these challenges, we propose AtomMem, a long-term memory system designed for value-dense storage and stable memory evolution. AtomMem introduces a Fact Executor, which selectively extracts high value atomic facts from long form interactions to serve as highly efficient memory representations. Subsequently, AtomMem organizes these facts into hierarchical event structures and temporal profiles, capturing coherent episodic contexts and tracking dynamically evolving user attributes over time. During retrieval, the system activates an associative memory graph to connect fragmented memories. Experiments on the LoCoMo benchmark confirm that AtomMem achieves state-of-the-art performance across various reasoning tasks, offering a scalable and economically viable solution for deploying intelligent personalized agents.

## 2. 方法：核心思想、关键技术细节

### 2.1 核心思想
- 以**原子事实（Atomic Facts）**作为记忆的基本语义单元。原子事实是**自包含、无歧义、高价值**的陈述句，通过解析原始对话中的指代、时间锚定等隐式依赖生成。
- 将原子事实组织为**层次化记忆结构**：
  - **事件记忆（Event Memory）**：将相关原子事实聚合为情节叙事块，捕获连续上下文。
  - **时序档案（Temporal Profile）**：基于累积的事实证据，增量式追踪用户的稳定属性与偏好变化，并保留历史版本。
- 检索时通过**记忆图（Memory Graph）** 的关联激活，连接分散的事实，增强跨会话的关联召回。

### 2.2 关键技术细节

#### 2.2.1 原子事实提取器（Fact Executor）
- 采用**监督微调（SFT）** 训练一个轻量级LLM（基于Qwen3-14B，LoRA微调），用于从原始对话中提取原子事实。
- 构建高质量训练数据集D（4352样本），通过两阶段流水线：教师模型（GPT-4o）初步提取 + 人工审查精炼。
- 提取时执行：去噪（过滤闲聊/无意义内容）、指代消解、时间锚定、第三人称重写、多模态信息融合。
- 输出结构化原子事实：`F = {id, c, v, P, K, T, E}`，其中c为自包含文本，v为嵌入向量，P为参与者，K为关键词集，T为时间信息，E为关联事件列表。

#### 2.2.2 事实验证与更新
- 存储前，先通过**混合相似度度量**（语义嵌入 + 关键词Jaccard相似度，见式2）检索候选事实集，然后由LLM判断新事实与候选集的关系，生成残余新内容`c'`和更新集`U`，避免冗余并维护一致性。

#### 2.2.3 事件记忆构建
- 动态将新事实吸收到已有事件中（更新摘要、参与者和时间跨度），或创建新事件。算法通过LLM语义路由决定吸收、创建新事件或独立存储。

#### 2.2.4 时序档案建模
- 采用**会话批量机制**：在原子事实提取时识别暗示长期特征的事实，加入等待队列；在会话结束时批量处理。
- 通过混合相似度检索用户已有档案，LLM决定是否冗余、更新当前档案、修改历史版本或创建新条目。保留历史状态`H`以支持时间回溯。

#### 2.2.5 关联记忆图激活
- 定义三种边：
  - **实体边**：共享关键词（IDF加权，见式3）
  - **事件边**：同属一个事件（大小惩罚，见式4）
  - **时域边**：相邻对话轮次（指数衰减，见式5）
- 使用**个性化PageRank（RWR）** 在局部子图上传播激活分数。先通过分层检索得到种子事实（主召回+补偿召回），然后扩展2跳以内邻居，经RWR收敛后取top-k事实作为检索结果。

#### 2.2.6 响应生成
- 查询意图分析：判断是否需要档案检索。
- 分层混合检索：主召回（直接事实相似度）+ 补偿召回（事件级扩展）+ 关联召回（图传播）。
- 档案增强：若意图需要，检索用户档案并选择时间版本。
- 最终将事实集和档案集拼接为上下文，输入LLM生成答案。
