## 摘要
显式的技能库使计算机使用智能体更易于检查，但目前尚不清楚是否可以通过交互数据挖掘的方式构建此类库，从而改进下游策略。我们通过一个三阶段流水线研究这个问题：分割GUI轨迹，将片段聚类为候选技能，并从生成的注释中训练技能感知策略。挖掘出的聚类在源基准上具有可读性：八个聚类中有五个与InteraSkill工作流标签的纯度至少达到0.95。然而，可读性并不意味着可迁移。GRPO仅将IW技能步进准确率从18.5%提升至20.5%，对BrowseComp+基本无影响，并且在关键源域指标上甚至不如简单的频率先验。因此，我们将该方法作为一项诊断研究呈现：轨迹挖掘可以揭示可检查的技能结构，但当前的边界检测器、无序的片段表示以及离线奖励模型不足以实现可靠的跨域策略改进。

## Abstract
Explicit skill libraries make computer-using agents easier to inspect, but it remains unclear whether such libraries can be mined from interaction data in a way that improves downstream policies. We study this question through a three-stage pipeline that segments GUI trajectories, clusters segments into candidate skills, and trains a skill-aware policy from the resulting annotations. The mined clusters are readable on the source benchmark: five of eight clusters have at least 0.95 purity against InteraSkill Workflows labels. However, readability does not imply transfer. GRPO improves IW skill-step accuracy only from 18.5\% to 20.5\%, leaves BrowseComp+ essentially unchanged, and underperforms trivial frequency priors on key source-domain metrics. We therefore present the method as a diagnostic study: trajectory mining can expose inspectable skill structure, but the current boundary detector, orderless segment representation, and offline reward model are insufficient for reliable cross-domain policy improvement.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：计算机使用智能体（CUA）在执行长尾GUI任务时，重复的操作模式（如搜索、复制、粘贴、切换应用）可被封装为显式技能（SKILL.md）。手工编写技能库存在瓶颈（命名、范围、文档、更新），且编码了设计者对可重用行为的假设。论文尝试从交互轨迹中自动挖掘显式技能库，并检验这些挖掘的技能是否能提升下游策略。
- **核心问题**：轨迹挖掘的难点不在于聚类本身，而在于证明挖掘出的技能类簇能够帮助策略在新任务上泛化。作者特别强调：一个聚类可以很一致（可读），但不一定有用；策略提升可能只因看到更多数据，而非学到了可迁移的技能词汇。因此核心问题是：**挖掘的技能中到底有什么能真正迁移？**

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

- **整体思想**：三阶段流水线（trajectory segmentation → skill clustering → policy training），将GUI轨迹分割为片段，聚类为候选技能，再训练技能感知策略。
- **阶段1：轨迹分割（技能边界检测）**
  - 采用简单的相邻动作欧氏距离作为变化点信号：  
    \[
    \Delta a_t = \|a_t - a_{t-1}\|_2,\quad t \in B \text{ if } \Delta a_t > \theta
    \]
    动作向量包含15维归一化特征（10维one-hot动作类型、屏幕坐标、时间戳、文本长度、滚动量）。阈值θ通过IW数据集上扫描百分位数、最大化边界F1选定。
  - 该规则简单但易过分割（例如点击到类型切换、坐标大幅跳跃都可能触发边界，即使仍属于同一技能）。
- **阶段2：技能嵌入与聚类（技能库构建）**
  - 步骤1：片段表示。对每个片段计算动作向量的均值和方差，得到长度不变的“动作包”摘要（顺序信息丢失）。
  - 步骤2：Wasserstein聚类。使用对角高斯之间的平方Bures距离进行层次聚类，簇数k从8到16扫描，最终选k=8。
  - 步骤3：伪标签对比学习。将聚类分配作为伪标签，训练一个MLP编码器（30→64→32→16维），使用监督对比损失（supervised contrastive loss）优化嵌入。使用类平衡采样器，训练200轮。
- **阶段3：技能感知GRPO训练**
  - 使用Qwen3-8B基模型，在1,275条包含任务上下文和挖掘技能名称的提示上进行GRPO训练。
  - 奖励模型：一个基于Qwen3-8B的序列分类头，在IW技能流偏好对上训练（使用启发式边界的成对排序损失），未在WebArena等目标域上训练。
  - 8个候选回复/提示，温度0.7，最大完成长度192，学习率5×10⁻⁶，梯度累积8，奖励裁剪5.0，训练1个周期。
  - 轻量基线：MLP（基于当前技能嵌入预测下一技能）、2层因果Transformer（技能序列建模）。

## 3. 实验设计：数据集/场景、benchmark、对比方法

- **数据集与场景**：
  - **IW**（InteraSkill Workflows）：2,000条合成企业风格轨迹，包含12种技能类型（如document_edit, search_navigate, send_message），8,290个地面真值片段，40,774个原始动作。作为**源域**用于分割、聚类和策略训练。

## 6. 主要结论与发现

- **积极发现**：挖掘的技能在源域可读。8个簇中有5个纯度≥0.95（对应document_edit, data_transfer, organize_files, export_publish, presentation_edit）。对比学习进一步提升了NMI（从0.650到0.862）。
- **消极发现（核心结果）**：
  - 策略提升微弱：GRPO在IW上准确率仅从18.5%提升至20.5%；WebArena从55.8%降至44.2%；BrowseComp+基本不变（43.5%→43.3%）。
  - **频率先验是最强基线**：在IW上频率先验的准确率（34.9%）高于MLP（23.3%）、GRPO（20.5%），甚至高于手工SKILL.md表的14.0%。在编辑距离上频率先验也是最优。
  - 自动生成的SKILL.md在部分数据量下优于手工表，但整体仍不如频率先验。
  - 边界检测高召回低精度（prec=0.419, rec=0.803），导致过分割；阈值不跨域稳定。
- **整体结论**：轨迹挖掘能产生可检查的技能结构，但当前流水线的边界检测器（基于简单欧氏距离）、无序段表示（bag-of-actions）和离线奖励模型（仅IW偏好）不足以实现可靠的跨域策略提升。论文将自身定位为诊断研究，而非成功声明。

## 8. 不足与局限

- **实验覆盖不完整**：未完成Mind2Web GRPO评估和实况WorkArena评估，限制了跨域迁移声明的范围。Live WorkArena被作者承认为更强的企事业转移测试，但论文仅使用文本诊断替代。
- **对比不公平**：MLP/Transformer与GRPO的输入模态和优化目标不同，无法单独归因奖励模型或策略优化。作者自己也承认这一点。
- **数据合成性**：IW是合成轨迹，可能无法反映真实企业环境中的复杂性和噪声。WebArena轨迹局限于地图导航，技能多样性低。
- **边界检测过于简单**：基于欧氏距离的变化点检测缺乏语义理解，容易过分割跨域不稳定。与基于动作预测误差的先进方法（如Open-World Skill Discovery）相比，可能处于劣势。
- **段表示丢失顺序信息**：均值/方差的无序总结忽略了关键的执行顺序（如先选择再复制、先打开菜单再选择项目），这对于可执行技能至关重要。这可能是导致策略迁移失败的主要原因。
- **奖励模型仅基于IW偏好**：未在目标域任务成功率上进行训练或微调，因此无法为跨域序列提供有效信号。
- **频率先验的胜利**：表明数据集存在严重的类不平衡或重复模式，挖掘的技能并未提供比简单统计更优的预测能力。这提示当前方法可能只是在学习数据分布偏差而非真正可迁移的技能结构。
- **缺乏人工验证**：虽然簇对齐了IW标签，但未扩展到其他域，也未有人工评估自动生成技能文件的实用性或可检查性。
