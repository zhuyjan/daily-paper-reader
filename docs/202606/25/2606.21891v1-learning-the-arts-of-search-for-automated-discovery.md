## 摘要
科学发现可以表述为在假设和实验空间上的迭代搜索过程。当代方法使用诸如MCTS之类的启发式算法来导航该空间。这些算法将假设的优劣与其实验执行的质量混为一谈。因此，一个有前景但执行初步的假设排名低于一个虽平庸但执行精细的假设。此外，随着搜索的进行，前期方法会修剪搜索日志，因为累积的历史记录超出上下文窗口。我们提出了树搜索的智能推理（ARTS），其中部署了一个推理语言模型来导航该空间。该模型检查先前的执行日志，诊断早期失败是由错误的实现还是糟糕的假设引起的，并选择接下来要构建的假设。为了缓解上下文长度的挑战，ARTS使用测试时训练将搜索树的知识注入模型权重。在来自MLGym和MLEBench的22个任务上，我们展示了ARTS优于领先算法，归一化分数相对提升超过15.3%。通过测试时训练，我们证明Qwen3-4B智能体可以匹配Gemini-3 Pro和GPT o3推理等闭源前沿模型的性能，同时推理成本降低多达5倍。我们进一步观察到，在部分可观察的强化学习任务中，经过测试时训练的Qwen3-4B科学家通过重新发现启发式方法修剪掉的人类最优递归记忆解决方案，超越了使用o3科学家的ARTS。

## Abstract
Scientific discovery can be formulated as an iterative search process over the space of hypotheses and experiments. Contemporary methods navigate this space using heuristics such as MCTS. These algorithms conflate the merit of a hypothesis with the quality of its experimental execution. A promising hypothesis with preliminary execution is therefore ranked below a modest hypothesis whose execution is refined. Moreover, prior methods prune the search logs as the search progresses because the accumulated history outgrows the context window. We propose Agentic Reasoning for Tree Search (ARTS), where we deploy a reasoning language model to navigate this space. The model inspects prior execution logs, diagnoses whether earlier failures arose from faulty implementations or bad hypotheses, and selects the hypothesis to build on next. To mitigate challenges with context length, ARTS uses test-time training to instill the knowledge of search tree in the model weights. Across 22 tasks from MLGym and MLEBench, we show that ARTS outperforms leading algorithms, with over 15.3% relative improvement in the normalized score. With test-time training we show that a Qwen3-4B agent can match performance with closed-source frontier models like Gemini-3 Pro and GPT o3-reasoning with upto 5x lower inference cost. We further observe that on partially observable RL tasks, the test-time trained Qwen3-4B scientist surpasses ARTS with the o3 scientist by rediscovering the human-best recurrent-memory solution that heuristic methods prune away.

### 论文方法论：核心思想、关键技术细节
- **核心思想**：将搜索过程分为“科学家”（推理模型）和“执行者”（编码代理）。科学家通过检查历史执行日志，区分失败是由“假设错误”还是“实现错误”导致，从而智能选择下一个扩展节点和假设。执行者则负责将假设转化为可运行代码。
- **关键组件**：
  - **节点检查（INSPECT）**：科学家可读取任意节点的代码、训练曲线、输出日志，以诊断低分原因。
  - **基线审计（Audit）**：初始阶段保留若干回合，科学家仅阅读数据、基线代码和评估脚本，发现任务的结构性偏差。
  - **假设采样（Verbalized Sampling）**：科学家枚举 K 个候选假设并分配概率，系统按概率采样而非贪心选择，以保持多样性。
  - **记忆模块**：每次扩展后写入一条短洞察，辅助后续决策，避免重复读取所有历史节点。
- **测试时训练（ARTS*）**：当搜索树超出上下文时，用 GRPO（组相对策略优化）微调科学家模型（Qwen3-4B），将搜索树知识注入 LoRA 适配器。奖励基于单步百分位得分（如前 30% 得+1），提供密集信号。
- **工作流程**（Algorithm 1 & 2）：
  1. 初始化树 G，进行基线审计。
  2. 每个搜索步：科学家检查节点 → 选择父节点并输出候选假设分布 → 执行者实现并验证假设 → 新节点加入树。
  3. 当树过大时，定期用当前树训练科学家（GRPO，组大小 B=8），更新 LoRA 权重。

### 论文主要结论与发现
1. **ARTS 显著优于现有方法**：在 22 个任务中，ARTS 在 16 个任务上取得最佳结果，归一化平均分数比最强基线（MLEvolve）提升 15.3%，IQM 达 0.93（对比 0.87）。
2. **失败归因是关键优势**：科学家通过检查日志区分“假设错误”与“实现错误”，保留有前景的方向。例如在 HMS 任务上，连续回归节点被正确归因为训练不足而非假设失败，最终达到最佳 KL 0.467。
3. **多样性更高**：ARTS 探索的假设轴数（4.43）和熵（1.73）均高于基线（AIRA: 2.20/0.43；MLEvolve: 2.43/0.58），且 45% 的扩展是完全新的假设方向。
4. **测试时训练提升小模型**：Qwen3-4B 经 TTT 后，在 12 个 MLGym 任务上归一化分数从 0.72 提升至 1.01，在部分任务（如 MetaMaze）超越 o3 科学家，匹配甚至超过闭源模型，且推理成本降低 5 倍。
5. **TTT 缓解上下文溢出**：在 MetaMaze 部分可观测 RL 任务中，TTT 训练的科学家成功找回并优化被启发式方法丢弃的 LSTM 递归记忆方案，达到超越人类最优的 53.0 分。

### 优点
- **新颖的智能代理搜索架构**：将科学家与执行者解耦，让推理模型专注于方向选择与失败归因，而非既写代码又做决策。
- **有效的失败归因机制**：显式要求科学家分类“假设错误” vs “实现错误”，避免有前途的想法因实现细节被丢弃，这是启发式方法无法做到的。
- **口头采样防止多样性崩溃**：要求模型列出多个候选并给出概率，系统采样，确保搜索覆盖更广的假设空间。
- **测试时训练解决长上下文问题**：将搜索树知识压缩到模型权重中，不仅消除修剪历史的信息丢失，还让轻量模型接近强推理模型性能，实际部署成本极低。
- **实验严谨**：覆盖多种任务和基线，多次重复，消融详尽，使用了可靠的聚合指标（IQM、最优差距）。

### 不足与局限
- **依赖 API 模型**：默认科学家为闭源 o3，虽可替换但实验成本高；即使 TTT 使用小模型，仍需 GPT/Claude 等外部模型进行初步审查或生成（论文附录提及使用 GPT-5.5 标记假设轴）。
- **单任务 8 小时预算可能不足**：对于更复杂的课题（如新材料发现），8 小时可能仅能探索少数方向；论文未讨论较大规模时间预算下的表现。
- **执行者与科学家捆绑**：方法性能依赖于执行者（Gemini 3 Flash）的代码生成能力，若执行者引入额外错误，会污染归因质量。
- **实验数量相对有限**：虽然 22 个任务多样，但每个任务仅 3 次重复，偶然性可能影响显著性的统计推断。消融实验仅在部分任务上进行，未在所有 22 个任务上展开。
- **TTT 的泛化性未充分验证**：论文只在 MLGym 的 12 个任务上验证了 TTT，未在 MLEBench 的 Kaggle 任务上应用，可能因计算限制。跨域泛化未知。
- **潜在的评估偏差**：论文通过恢复评估脚本来防止“作弊”，但未讨论智能体可能通过改变随机种子等方式间接影响结果；人工筛选失败案例时可能存在选择偏差。
