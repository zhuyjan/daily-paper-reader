## 摘要
我们提出了SimpleSearch-VL，一个高效、可靠且实用的多模态智能搜索框架。其核心思想是改进智能体自身的搜索与验证过程，而非扩展数据、工具或辅助模型组件。在效率方面，因子化自适应展开（FAR）通过形成更多信息量的训练组，同时利用冗余样本来缓解长尾延迟并暴露困难样本，从而提高了采样效率。在可靠性方面，SimpleSearch-VL执行基于证据的推理，明确使用思维链验证来评估检索到的视觉和文本线索与原始上下文的相关性。在实用性方面，SimpleSearch-VL保持轻量级的工具接口，并在智能体内部执行网页自我摘要，无需额外外部依赖。仅使用5K监督工具交织轨迹和2K强化学习数据，SimpleSearch-VL在Qwen3-VL智能基线上分别将8B和30B-A3B变体的平均得分提升了15.8和16.0分。SimpleSearch-VL-30B-A3B模型进一步实现了与智能体Gemini-3-Pro竞争的性能。

## Abstract
We present SimpleSearch-VL, an efficient, reliable, and practical framework for multimodal agentic search. Its core idea is to improve the agent's own search-and-verification process rather than scaling data, tools, or auxiliary model components. For efficiency, Factorized Adaptive Rollout (FAR) improves sampling efficiency by forming more informative training groups while using redundant samples to mitigate long-tail latency and expose hard samples. For reliability, SimpleSearch-VL performs evidence-verified reasoning, explicitly using chain-of-thought verification to assess the relevance of retrieved visual and textual cues to the original context. For practicality, SimpleSearch-VL keeps a lightweight tool interface and performs webpage self-summary within the agent, requiring no additional external dependencies. With only 5K supervised tool-interleaved trajectories and 2K RL data, SimpleSearch-VL improves Qwen3-VL agentic baselines by 15.8 and 16.0 average points for the 8B and 30B-A3B variants, respectively. The SimpleSearch-VL-30B-A3B model further achieves performance competitive with agentic Gemini-3-Pro.

## 1. 核心问题与整体含义（研究动机和背景）

*   **问题**：多模态智能搜索代理（agentic search）在面对知识密集型视觉问题时，需要主动收集和验证外部证据。然而，现有方法存在三大瓶颈：
    *   **效率低**：多轮工具调用的长尾轨迹导致 rollout 生成时间过长，固定采样策略无法根据信号质量自适应分配计算资源。
    *   **可靠性不足**：检索到的视觉证据（如反向图像搜索返回的缩略图、标题、URL）可能表面匹配但实际不相关，模型缺乏显式的视觉验证机制。
    *   **实用性差**：许多系统依赖复杂工具管道或外部网页摘要模型，增加了部署成本和模型依赖。
*   **整体含义**：作者主张通过改进智能体自身的**搜索与验证过程**来解决问题，而不是通过大规模扩展数据、工具或辅助组件。SimpleSearch-VL 旨在提供一个**高效、可靠、实用**的轻量级框架，降低训练和部署门槛。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心思想
*   将多模态搜索形式化为马尔可夫决策过程（MDP），智能体交替进行推理、工具调用、证据验证和答案生成。
*   三个关键设计：**因子化自适应展开（FAR）**、**证据验证推理**、**自摘要网页访问**。

### 2.2 关键技术细节
#### a) 因子化自适应展开（FAR）
*   **目的**：提升强化学习 rollout 的采样效率，将固定预算转化为信号感知分配。
*   **两个维度**：
    *   **Prompt 扩展**：当需要更多有用信号时，引入新的 prompt 组。
    *   **Rollout 分配**：为缺乏奖励对比的组增加额外尝试，一旦获得信号则跳过冗余尾部 rollout。
*   **信号评估**：使用答案正确性指示函数（0/1）判断组内是否有混合结果（部分正确部分错误），只有存在信号变化的组才用于策略更新。
*   **过滤规则**：“仅当所有 rollout 都错误时才继续扩展”，即“扩展只针对全错组”。一旦出现正确 rollout，后续扩展槽被屏蔽。
*   **停止规则**：当收集到足够数量的有效组，或已用掉约 99% 的预分配槽位时停止，避免等待慢轨迹。
*   **选择策略**：从有效组中选取最多 n_min 个轨迹，保持奖励多样性和粗粒度行为多样性（不同奖励、推理轮数、工具调用次数）。
*   **策略更新**：使用 RLOO 估计器，基于完整组计算留一基线，并采用裁剪的 PPO 目标优化。

#### b) 证据验证推理
*   **视觉证据验证**：`image_search` 工具返回匹配的缩略图、网页标题和 URL。模型被训练先比较缩略图与查询区域，确认视觉匹配后再使用该来源的信息。
*   **文本证据验证**：文本搜索返回片段、标题、URL，供模型进行页面级验证。
*   **区域级缓存**：为图像搜索建立 IoU 阈值的缓存，避免重复调用相同区域的搜索。

#### c) 自摘要网页访问
*   **机制**：`visit` 工具获取网页内容后，策略模型自身生成目标导向的摘要（不依赖外部摘要模型）。
*   **优势**：消除额外模型依赖，使摘要行为随智能体策略共同适应。

### 2.3 算法流程简述
1.  状态初始化：包含问题、输入图像及历史交互。
2.  推理：模型先输出 `<thinking>` 推理，然后选择一个工具或生成答案。
3.  工具调用：支持 `text_search`、`image_search`、`visit`，每个工具返回结构化观察。
4.  证据验证：在图像搜索后，检查缩略图一致性；在文本搜索后，结合片段和摘要判断。
5.  更新状态：将工具调用和观察追加到历史。
6.  答案生成：当认为证据充分时，输出 `<answer>`。
7.  训练：SFT 阶段使用 5K 证据感知轨迹；RL 阶段使用 FAR 进行 rollout 采样和策略更新。
