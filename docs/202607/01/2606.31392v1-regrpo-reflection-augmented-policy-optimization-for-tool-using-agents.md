## 摘要
工具增强的视觉语言模型（VLM）能够通过调用外部工具解决多模态、多步骤任务，但在实践中仍然脆弱。现有工作存在两个常见不足。监督微调（SFT）主要基于成功轨迹构建，在工具故障后提供的恢复信号很少，而稀疏的轨迹级强化学习奖励则对哪个步骤失败以及如何修复提供有限的指导。我们提出了ReGRPO（反射增强的组相对策略优化），这是一个在工具使用智能体中学习反射引导纠正的框架。ReGRPO从一个结构化的反射数据引擎开始：我们执行接近正确的动作以收集基于实际失败的观察，然后构建反射思维三元组（错误类型、证据、修复计划），并与纠正后的动作配对，用于热启动SFT。接着，我们使用组相对优势在局部轨迹内联合优化反射标记和纠正动作，并引入反射代价项以减少不必要的反射。在GTA和GAIA上的实验表明，在相同的骨干网络和工具套件下，ReGRPO持续优于强大的开源基线，并在所比较的开源控制器中取得了最佳结果。代码和RoT数据可在https://github.com/showlab/ReGRPO获取。

## Abstract
Tool-augmented vision-language models (VLMs) can solve multimodal, multi-step tasks by calling external tools, yet they remain fragile in practice. Existing works have two common gaps. Supervised fine-tuning (SFT) is built mostly on successful trajectories and offers little signal for recovery after tool failures, while sparse trajectory-level RL rewards provide limited guidance on which step failed and how to repair it. We introduce ReGRPO (Reflection-augmented Group Relative Policy Optimization), a framework that learns reflection-guided correction in tool-using agents. ReGRPO starts with a structured reflective data engine: we execute near-miss actions to collect grounded failure observations, then build Reflection-of-Thought triplets (ErrorType, Evidence, FixPlan) paired with corrected actions for warm-start SFT. We then optimize reflection tokens and corrective actions jointly within local trajectories using group-relative advantages, and include a reflection-cost term to reduce unnecessary reflection. Experiments on GTA and GAIA show that, under the same backbone and tool suite, ReGRPO consistently outperforms strong open-source baselines and achieves the best results among the compared open-source controllers. Code and RoT data are available at https://github.com/showlab/ReGRPO.

- **整体意义**：本文提出**ReGRPO**，首次将反射（Reflection）显式建模为可学习的决策变量，通过结构化反思数据引擎和组相对策略优化，使智能体在工具失败时能够自主诊断并纠正错误，从而提升整体鲁棒性。

## 2. 论文提出的方法论

### 2.1 核心思想
- **反射增强**：将失败动作后的反思过程（错误类型、证据、修复计划）视为可优化的轨迹部分，与纠正动作一起联合学习。
- **结构化反思数据引擎**：从成功轨迹中生成“接近失败”的动作，执行后获得真实失败观测，然后使用教师模型（GPT-4o）生成结构化的**反思三元组**（ErrorType, Evidence, FixPlan），并与纠正动作配对，构成“误差-反思-修正”的监督数据。
- **反射增强的组相对策略优化（ReGRPO）**：在标准 GRPO 框架下，将反思令牌和纠正动作纳入局部轨迹的优化目标，使用组相对优势信号进行更新，同时引入**反射代价项**（reflection cost）以抑制不必要的反思。

### 2.2 关键技术细节
- **局部轨迹定义**：对于状态 \(x_i\)，智能体先执行初始动作 \(a_i^{(0)}\) 并得到观测 \(o_i^{(0)}\)；若成功则轨迹结束；若失败（硬错误或低置信度），则生成反思 \(z_i\) 和纠正动作 \(a_i^{(1)}\)，形成局部恢复段。
- **复合奖励函数**：
  \[
  R(\tau) = \lambda_{\text{exec}} \cdot \mathbb{1}\{\text{success}\} - \eta C(\tau) + \lambda_{\text{val}} V(\tau)
  \]
  - \(\mathbb{1}\{\text{success}\}\)：环境反馈的任务成功标志；
  - \(C(\tau)\)：反思长度惩罚，仅在有反思时按令牌数量计算；
  - \(V(\tau)\)：可选的训练时验证器分数（由元数据确定性计算，不调用LLM）；
  - 默认系数：\(\lambda_{\text{exec}}=1.0,\ \eta=0.1,\ \lambda_{\text{val}}=0\)（无验证器训练，推理时完全零验证器）。
- **优势估计**：每个 sampled group 内计算组均值作为基线，优势 \(A_i^{(k)} = R(\tau_i^{(k)}) - \bar{R}\)。
- **优化目标**：
  \[
  \mathcal{L}_{\text{ReGRPO}} = -\mathbb{E}_{x_i,\tau\sim\pi_\theta}\left[\frac{1}{K}\sum_{k=1}^K A_i^{(k)}\log\pi_\theta(\tau_i^{(k)}|x_i)\right] + \beta D_{\text{KL}}(\pi_\theta\|\pi_{\text{ref}})
  \]
- **推理阶段**：单路径执行，使用确定性触发器（工具错误/空观测/低置信度）决定是否开启最多一个局部反射-纠正块。

### 2.3 训练流程
1. **SFT 热启动**：在反思数据上最大化条件似然 \(P(z_i, a_i^* | x_i, a_i^{\text{fail}}, o_i^{\text{fail}})\)，使模型学会诊断和修复。
2. **ReGRPO 强化学习**：自我探索，生成包含反射和纠正的轨迹组，使用组相对优势更新策略参数。
